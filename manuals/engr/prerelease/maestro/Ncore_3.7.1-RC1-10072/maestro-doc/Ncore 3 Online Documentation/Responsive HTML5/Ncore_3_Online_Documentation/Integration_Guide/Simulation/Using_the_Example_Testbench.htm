5.3 Using the Example Testbench

The example testbench environment includes a set of tests to test the coherent subsystem. The environment is based on the UVM environment and leverages the commercial VIPs. Each one of the interfaces' names is based on the name assigned to the structure in the Transport Definition perspective. The example testbench is constructed using a standard makefile. The make targets are described as follows:

make build

Builds and compiles the testbench for the coherent subsystem portion of the design.

make <test_name>

Runs a single test for the coherent subsystem portion of the design.

make clean

Removes the results of previous calls of make.

make all

Builds the testbench and runs ncore_sys_test, ncore_ral_reset_value_test, and ncore_connectivity_test.

Run make from <output_dir>/tb/<SIM_TOOL_dir>.

The build outputs are in the <output_dir>/tb/<SIM_TOOL_dir>/build directory.

Test outputs are in the <output_dir>/tb/<SIM_TOOL_dir>/run/<test_name> directory.
