1.3 Design Configuration Workflow

The Ncore design workflow is driven by the Maestro GUI. Maestro comprises a client component and a server component.

• The client includes all designer-visible functionality from specification through to refinement and design checking.

• The server component, which can be run at Arteris or on-premises depending on the license agreements, is invoked during the design workflow to generate RTL and collaterals as listed above.

1.3.1 Ncore Workflow Phases

The Ncore workflow encompasses a sequence of design actions known as phases as shown in the following figure.

Maestro assists you in performing the phase design actions:

1. Phase 1 - The SoC Specification is the phase of the design flow that defines a chip, power domains and regions, clock domains, and subdomains. See “SoC Specification (Phase 1)”. Setting the safety ASIL value is also part of this phase.

2. Phase 2 - The System Assembly phase of the design flow defines sockets, memory maps, and connectivity requirements. See “System Assembly (Phase 2)”.

3. Phase 3 - The Structural (Architectural) Design phase of the design flow is where the topology solution is created to meet the performance requirements of the design and includes creation and configuration of Ncore caches, and inserting the Resiliency and CSR networks. Users can manually select and configure each network and create the memory map. See “Structural Design (Architecture, Phase 3)”.

4. Phase 4 - Architectural Refinement is the phase of the design flow during which the designer can fine-tune parameters to achieve specific PPA objectives, for example, choosing buffer sizes, tweaking arbitration policies, deploying pipe stages to achieve timing closure, etc. See “Architectural Refinement (Phase 4)”.

5. Phase 5 - Export is the phase of the design flow that invokes the server component of Maestro to generate RTL and collaterals. This process is only allowed if the design is clear of any outstanding errors. See “Export Design (Phase 5)”.

Throughout the design flow, Maestro provides guidance to the designer about the current and next steps. Actions are automated to minimize the iteration time and the possibility of misconfiguration. Additionally, checks are performed in the flow to prevent incorrect parameterizations.
