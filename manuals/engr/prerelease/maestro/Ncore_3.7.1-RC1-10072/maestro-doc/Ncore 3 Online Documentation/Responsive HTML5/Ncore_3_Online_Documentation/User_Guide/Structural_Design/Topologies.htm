5.3  Topologies

The following topologies are supported:

• “Crossbar topology”

• “Mesh topology”

5.3.1 Crossbar topology

This topology consists of a single switch. All ports of all units (of the same network) will be connected to this switch.

Circles indicate switches, squares indicate the Ncore units

5.3.2 Mesh topology

In this topology, the switches are arranged as a mesh and the routes are generated based on X, then Y routing from source to the destination. The switches with 1 input and 1 output port are deleted after the generation of the routes and replaced with a wire in-place.

The size of the mesh is defined by the parameters 'meshx' and 'meshy'. The maximum number of nodes supported is 25. The bandwidth for routing the data in the data network is specified by the parameter 'datawidth'. The frequency for routing is defined by the clock subdomain passed in the parameter 'clock'.

Circles indicate switches, squares indicate the Ncore units

XY ordering can be used to maximize tiling options for the CSR networks by routing one with X-Y ordering and the second with Y-X ordering.

Copy Location: After assigning Ncore Units to a mesh network, the remaining networks can be created by copying the node location information for a previously created network. This ensures that nodes of the same mesh connect to the same Ncore Units assisting in maximizing tiling of the design.

Note the following:

• Prior to generating routes, each Ncore unit needs to set a position on how the Ncore unit is attached to the mesh.

• Mesh supports CSR networks.
