4

self-checking atomic test (additionaly include write, read, data compare in the test to make sure atomic goes well) for IOAIUp i/f to DMI with SMC

  1. Write to addressed location with directed data

  2. Initiate one of atomic operation (mentioned below) to same addressed location with directed data. And also obtain the data stored in addressed location returned by atomic.

  3. Issue read to same addressed location.

  4. Make sure in sequence atomic operation functions well by checking

    1. Initial Data stored in addressed location returned in atomic : It must match with directed write data in #1

    2. Data written to addressed location after atomic operation. : It should match with expected value in the sequence based on atomic operation.

FSC registers mapping change

  • This design update in FSC accommodates 160 error inputs (vs 128 error inputs in ncore3.6) in automotive configuration with FSC registers mapping change (3 new registers - FSCLF4, FSCMF4, FSCCETHF4 with new mapping of mission fault, latent fault, CERR fault inputs) captured in snapshots below. Thus increased support on DMI error inputs from “8” in ncore3.6 to “32” in ncore3.7.

    • DCE - 16 error inputs

    • DVE - 1 error inputs

    • DMI - 32 (vs 8 in ncore3.6) error inputs

    • DII - 32 error inputs

    • IOAIU - 32 error inputs

    • CAIU - 32 error inputs

  • New FSC registers mapping as per Ncore_system_architecture_3.7_0.70

  • Old FSC registers mapping as per Ncore36_systarch_v1.0.8

  • Therefore no “stimulus” update in DV is needed to verify this design update.

  • “Functional coverage and check” DV update is needed to align with register mapping change effective in all FSC tests, with no additional checks.

Mesh Topology support

Mesh topology is enabled as part of Ncore3.6.4 release for the hw_cfg_ventana_mesh*_mpf, hw_cfg_17_mesh_mpf and the hw_cfg_resiltech_mesh_mpf configs and we will continue to run these configs on Ncore 3.7

Increase RBR Count

Maximum number of RbCredits for DCE and DMI are increased from 32 to 64

We need to have configs setting the parameters nDceRbCredits and nDmiRbCredits to values > 32. No new tests required

Config Details TBD

Increase SMC Tag bank to 4

Maximum number of Tagbanks in DMI SMC and the IOAIU Proxy cache is increased from 2 to 4

We need to have configs setting the parameter nTagBanks to 4. No new tests required

Config Details TBD

Larger STT

Total number of STT entries are not user settable. The value is derived from nAIUSnpCredits + nDVMSnpCredits

nAIUSnpCredits can range from 2 to 16

nDVMSnpCredits depends on systemlevel parameter NoDVM and the number of DVM agents

nDVMSnpCredits = NoDVM ? 0 : max{8, 2*(no of DVM agents +1)}

No new tests needed. We already have configs with NoDVM set to 0 and 1.

Increase Skid Buffer Size

The maximum skid buffer size has been increased from 320 to 768 for both the cmd skid buffer {DCE, DMI and DII } and the mrd skid buffer {DMI}

We need to have configs setting the parameters nCmdSkidBufSize and nMrdSkidBufSize to values > 320

No new tests needed.

Config details TBD

Add SRAM implementation to Skid Buffer

When the Overflow buffer size exceeds 256, Ncore3.7 architecture recommends to implement the buffer with SRAMs rather than flops

Overflow buffer size = n*SkidBufSize - n*SkidBufArb

The boolean parameters CMDOverflowBufInSRAM and MRDOverflowBufInSRAM needs to be set to True in case the overflow buffer size > 256

When the parameters CMDOverflowBufInSRAM and MRDOverflowBufInSRAM are set to true, the memType should be set to SRAM and the memProtectionType should not be set to NONE

We need to have configs ensuring the overflow buffer size > 256 and also the memProtectiontype set to SECDED and Parity

Error tests for SRAM error injection is covered at DCE / DII and DMI Unit level benches.
