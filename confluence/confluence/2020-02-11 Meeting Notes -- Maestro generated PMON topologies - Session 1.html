Summary of discussion.

1. How do we configure the blocks for given features, so that verif have this information. Standard configuration setup can be done using Darshan’s script and Maestro can reuse that. Darshan to talk with Mokhtar and Fredric. Standard configuration will only involve packet count. Latency binning, bandwidth measurement will be verified in a stand alone configuration but won’t be part of standard configuration hence Maestro won’t be generating networks involving PMON with bandwidth measurement and Latency binning features.

2. Provide mokhtar with parameters DV needs from confluence page. Add link to NodeID connectivity information.

    https://confluence.arteris.com/display/ENGR/Symphony+Block+Common+Characteristics
    Please look at Interrupt Bit Description section.

3. Maestro needs to provide ID of internal ATUT (hidden from testbench) that config network is driven from.
4. For DV to have information about Interrupt network, Aria needs to parse IP-XACT register descriptions and provide the interrupt block connectivity structure. Rather than getting these information from Maestro dumped JSON datastructures in order to avoid copies.
5. Symphony can have multiple top interrupt signals coming out of system. For Revision 1 we will have only 1 interrupt signal coming out of the system.
6. Maestro should generate the pmonConnectivity parameter in the top_rtl.json. Please look into Software Parameters Requirement for DV for more information. This parameter includes details of PMON stats, request and response probes associated with each ATUI.
7. For every register description will have “BlockID” : “Number”, that specify which register is associated with which Block. And in addition, for the top-level interrupt blocks, there will be interrupt bit Map descriptions as proposed in the ‘Symphony Block Common Characteristics’ confluence page.
8. IPXACT description for that register block should match value of read-only register BlockID. Which will be verified by RAL gen bit-bash test.

9. Aria needs to search through PAM tables of ATUI to find ID of ATUT connected to the config network. Only those ATUIs will have access to the config space.
10. Configuration network should be highest priority in terms of Maestro generated PMON configurations.
