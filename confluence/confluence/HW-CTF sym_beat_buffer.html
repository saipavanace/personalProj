Description

The sym_beat_buffer accumulates partial beats and combines them to full beats. The main purpose is to transform an interface with data byte enables to one without.

This block assumes the data will be packed meaning:

  • The first beat will always start at byte 0. 
  • There will be no bubbles in the data. (As indicated by the Byte Enables)

When a transaction comes in and its a read to a narrower target a context entry will be allocated to it. Each context entry has a corresponding HW-CTF sym_data_packer which will pack any partial beats from that target.

Data may be dropped if be's are all 0, and may bypass the buffer if be indicates all 1. 

The sym_beat_buffer will bit bucket any ndp associated with a partial dp.

Response Path Capture

The response dp needs top be able to lookup its sequence number so it will hold until an ndp arrives will its corresponding sequence number. This syncs the ndp and dp channels.

User Field

The sym_beat_buffer will also pack the user field if its present.
