Overview

For version 3.7.1, the IOAIU testing strategy is to reuse all tests from version 3.4, while also introducing additional configurations, performing cleanup of the 3.4 tests, and improving overall coverage.

The checking is performed using the PMON verification library, which was already integrated in version 3.4 for latency validation.

Both bring-up tests and random tests will be executed for each AIU type in the single-core IOAIU configuration, including:

  • ace

  • ace_lite

  • ace_lite_e

  • axi

  • axi_proxy

In addition, these tests will also be run on the newly added configurations in 3.7.1, such as:

  • ace_lite_owo_256b

  • ace5_lite_owo_512b

  • axi_owo_512b

Set Master Count Enable (bit 31) to 1 to globally enable all counters.

xCNTCR – Counter Control Register

  • Count Enable (bit[0])
    Set to 1 to enable local counting. Can be overridden by Master Count Enable.

  • Count Clear (bit[1])
    Write 1 (W1C) to clear both xCNTVR and xCNTSR.

  • Interrupt Enable (bit[2])
    Set to 1 to enable interrupts on counter overflow (via ORed path of xCNTVR/xCNTSR overflow).

  • Rollover/Overflow Status (bit[3])
    Read-only. Indicates overflow event. Cleared when counters are cleared.

  • Counter Control (bits[6:4])
    Selects event operating mode:

    • 000:Normal coun

    • 001: AND count

    • 010: XOR count

    • 011: Instantaneous count (used for multi bit events like Active OTT entries)

    • 100: use as 32-bit counter (count event first

    • 100 to 111 – reserved

  • SSR Count (bits[9:7])
    Selects mode for xCNTSR:

    • 000: SSR disabled

    • 001: Upper 32-bit capture (for 64-bit counting)

    • 010: Capture xCNTVR snapshot into xCNTSR

    • 011: IIR Filter mode

    • 100: Saturation/max limit

    • 101: 32-bit second event counter

  • Filter Select (bits[12:10])
    Valid only when SSR Count is 011 (filter mode):

    • 000 = 1/1

    • 001 = 1/2

    • 010 = 1/4

    • 011 = 1/8

    • 100 = 1/16

    • 101 = 1/32

    • 110 = 1/64

    • 111 = 1/128

  • Minimum Stall Period (bits[15:13])
    Sets delay in cycles before a new event is accepted (0 to 7 clock cycles).

    • Use 000 for zero stall.

  • Count Event Second (bits[21:16])
    Selects secondary event. Must be different from first event and not 0.

  • Reserved (bits[22:23], [30:31])
    Always write zero.

  • Count Event First (bits[29:24])
    Selects primary event source. Must not be zero.
