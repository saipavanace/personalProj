In your workarea, proceed with the following steps:

  1. Go to a browser and download the latest version of the ProtocolTransactionTable spreadsheet from the following link: Base Packet Parameters & Relationships
  2. Copy the spreadsheet to <workarea>/hw/rt/lib/bringup. And rename it to 'ProtocolTransactionTable.xlsx'
  3. cd <workarea>/hw/rtl/lib/bringup
  4. Run the following shell script: ./gen_if_param_map_from_xlsx.js. This should update the interface_param_map.json and interface_param_map_new.json which is the same as ( or JSON-ified version of) the Mapping tab in the spreadsheet - every row in the tab is an entry in the 'ndp' and 'dp' arrays in the JSON object.
  5. To generate just the BasePktATP.constraints.json, use this command: $WORK_TOP/../rtl/lib/bringup/gen_base_pkt_constraints.js -i $WORK_TOP/../rtl/lib/bringup/interface_param_map_new.json -o $WORK_TOP/../cpr/packet/BasePacketATP.constraints.json
  6. Run the following shell script: ./update_sym_cprs . This should update the relevant CPR files in the <workarea>/hw/cpr folder.
  7. Run the following shell script: ./gen_smi_lib.js. This should update the tables inside  <workarea>/hw-sym/js/pkt2Smi.js.
  8. Run Symphony (One for All Simulation - Regression Tool - "runsim") and Ncore (Ncore Fabric Generator) regressions to ensure the CPR changes don't break either of them.
  9. If the regressions are clean, you can push in the updated CPR files.

The above script update_sym_cprs is used to update only the CPR files. In order to update the interface_param_map.json, and JS libraries in <workarea>/hw/rtl/lib/bringup/js, as well along with the CPR files -Steps 5,6,7 in one step, you can run the following command in the same area as the other script:
./update_sym_ifmap_cprs_libs_from_xlsx
