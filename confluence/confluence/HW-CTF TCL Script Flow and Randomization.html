1 Create power domain

  1. Create Power Regions
    1. Create 1 or 2 power Regions
    2. Randomly assign one of two voltages to each region 
  2. Create Power Domains
    1. Create 1 or 2 power domains per region
    2. Randomly define a regions as being switchable or not.

2 Create clock domain

  1. For each power regions create a clock region
    1. For the regions, randomly assign a frequency of 2 GHz, 1 GHz,  600 MHz, 500 MHz, 400 MHz, or 200 MHz
  2. For each power domain, create one or two clock domains
    1. If the power domain is switchable, make all enclosed clock domains switchable
    2. If the power domain is not switchable, randomly make enclosed clock domains switchable or not.

Order Issues for Randomization:

In the script we can work around the order issues described below by doing it in the proper order in the script with intermediate parameters, but applying it in the order required by Maestro. The order issues are only related to when we are trying to randomize things, and customers won't even notice.

The order needed for randomization is:

  1. Create Target Sockets
  2. Create Memory Map
    1. The memory map is a function of the targets.
  3. Create Number of Initiators
  4. Create Flows between Initiators and Targets
  5. Create Initiator Sockets
    1. The number of address bits required by an initiator is a function of the memory map and flows
