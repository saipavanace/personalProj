Description

This page contains the micro-architectural specification of the AXI Native Layer inside Symphony. This layer along with two other layers -CTL and the Packet Layer comprise the AXI-ATU. There are two variants of this - one on the initiator ATU and the other on the target ATU. The Native Layer is the only one of the three layers that is expected to vary in microarchitecture from one native layer protocol to the other.

ID remap and context

The request IDs need to be remapped with following rules:

-          Requests from same Initiator and same ID, must use same remapped ID – that is the external slave should see them with same ID to preserve ordering.

-          Requests from different initiators can get remapped to same ID if there is ID compression.

-          The remapped ID bits can be less or more than the request ID bits, and also unrelated to number of context entries (when remapped ID bits are less than the log2(nContextEntry), it is called ID compression).
