Description

The block puts down a HW-CTF sym_firewall_adapter per VC, using a HW-CTF sym_vc_demux for the inInterface, and a HW-CTF sym_vc_mux for the outInterface. For a description of what the parameters do, refer to HW-CTF sym_firewall_adapter.

This block also puts down an interrupt accumulator module to accumulate each interrupt from the HW-CTF sym_firewall_adapter per VC into one output interrupt. 

This block also puts down an apb_demux to drive the APB interface of the HW-CTF sym_firewall_adapter per VC and the interrupt accumulator module, by forwarding the incoming APB transaction based on the APB address decode.

This block writes out a csr object that has an array of spaceBlock entries, each entry is a complete description for a set of registers per device, accessible via the device's APB interface. Each spaceBlock entry (i.e. each device) takes 256 bytes out of 4 K bytes of address space that's assigned to this block. As a result, this block supports maximum nVC = (4K / 256) - 1 = 15.

For nVC = 2:

  csr.spaceBlock[0] is for sym_firewall_adapter@VC=0

  csr.spaceBlock[1] is for sym_firewall_adapter@VC=1

  csr.spaceBlock[2] is for interrupt accumulator.
