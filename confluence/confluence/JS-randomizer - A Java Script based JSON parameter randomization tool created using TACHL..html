Presentation

With the high number of parameters used in the Symphony Design, there is need to generate  JSON parameter for all corresponding RTL modules in a hierarchical, dependent and constraint random way.

The main initiative behind the genesis of the tool is to enable DV team to automatically generate the JSON parameters resolving the complex inter dependencies, while randomizing the parameter within their legal ranges.

This tool can be extended to created random configurations for any given topology for all the rtl modules present.

 

 

The tool bases on a set of random function written in Java Script. For every rtl module with JSON parameters , there needs to be corresponding templatized  JSON which is written in the form readable by  TACHL.

All parameters with integer values are create by a random function which accepts min,max, resolution and a list of corresponding aliases or dependent parameters.

Parameters which are objects or arrays are created by specific function written in the library ,which has the base parameters passed as argument.

Also for dependent variables, there is a way to create an expression. Eg log(n) or sum(header_parameters) etc.

There is top level parameter tachl file which defines the sequence of individual module tachls to be preprocessed . Once a set of parameters are randomized upstream , the downstream variables can query if the dependent

parameter are defined or can be randomized within the range specified.

A top level param.json  is used to define topological, route maps and constraints for the module JSON files.
