2. Game Plan

To help with ease of migration and to ensure we don't step on other updates, we agreed to create a separate base test for Synopsys VIP bringup. This shall use the Synopsys VIP agents and sequences. The first step would be to enable the env to use the existing scoreboard. The easiest way to get to this is by instantiating in-house monitor in the env and have the scoreboard subscribe to this. The only downside I can see is, if the in-house VIP is not sampling some events, then those would get missed in the scoreboard checks. However, the chances of that happening is minimal and as a first step in the migration, we agreed to overlook this aspect. The flow is described in the diagram below.

Once we get this working, my recommendation would be to invert this approach, where in in-house BFM drives the native interface and have a passive Synopsys VIP monitor hooked to double check the protocol driven by our in-house VIP and close the Synopsys VIP functional coverage. This gives confidence in our in-house VIP for 3.6, which in turn shall promote for any future purpose if required.
