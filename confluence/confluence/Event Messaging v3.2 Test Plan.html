3.2 Stimulus

DCE Block Level Stimulus:

  • Send Exclusive Store/Load Commands which triggers a DCE exclusive monitor state changes
    • This also includes sending multiple exclusive store/load commands back-to-back to verify the DCE buffer and event accumulation
  • Enable DCE event sending so that DCE can send SysReq.Event message on the SMI
  • Program the DCE Snoop Enable Register, DCE_USER0 to 32'FFFF_FFFF (this is just temporary, until we start the verification of Sys coherency)
    • If a configuration has more than 32 AIUs, program DCE_USER1 to 32'FFFF_FFFF
  • After receiving the SysReq.Event message on the SMI, wait for a random delay and issue the appropriate SysRsp packet.
  • If an AIU event sending is disabled (like for the IOAIU), send the appropriate SysRsp packet (cmstatus = 00_000_xx0)

CHI Block Level Stimulus:

  • Enable SMI BFM event sending so that CHI can receive SysReq.Event message.
  • After receiving the SysReq.Event message from the SMI, wait for a random delay and issue the eventOut.Req and eventOut.Ack on the CHI Interface.
  • If an AIU event receiving is disabled, make sure no eventOut.req & eventOut.ack are issued and receive the appropriate SysRsp packet (cmstatus = 00_000_xx0) from CHI

4 Event Messaging 3.2 RC Release

4.1 SCOPE (spec reference : SysCmd 0.52 Architecture Spec)

CHI-AIU:

       What will be verified:

  • Only Event Receiver Block is present (verified as a part of previous release)
  • Need to verify Event Timeout in CHI-AIU Blocks

      What is outside the scope:

  • There is no event sender in CHI-AIU
  • Error scenarios apart from "event timeout" error will NOT be verified

IO-AIU:

      What will be verified:

  • Only the ACE and AXI with proxy cache will have Event Receiver Block which receives the event messages from DCE  (Need to look at Maestro passed in object)
  • Need to verify Event Timeout for ACE and AXI4 with proxy cache Blocks

      What is outside the scope:

  • There is no event sender blocks in IOAIUs
  • ACE-Lite-E, ACE-Lite and AXI4 (without proxy cache) will not participate in SysReq Events
  • Error scenarios apart from "event timeout" error will NOT be verified

DCE:

      What will be verified:

  • DCE can receive a "event timeout response" from the AIUs
  • DCE can generate a "protocol timeout" error and log them in it's registers 

      What is outside the scope:

  • Error scenarios apart from "event and protocol timeout" error will NOT be verified
