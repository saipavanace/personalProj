  • Normal response - report the error to ATUI

  • Make sure system recovers after the interrupt event as expected.
  • Check error is reported for normal response by reporting to ATUI.
  • Check error is reported for early response by writing CSR registers for early response error.
  • If the late responses receive contain error, and corresponding early response has already been generated then interrupt should be raised
  • Raise an error if the response isn't received from all or specific targets in the specified amount of time. Verified by checker inside ATUi which has timer.
