hw-lib/js/lib_utils.js

Base type

None

Composition

None

Description

This is the topmost factory function in the hierarchy of factories that constitute the Verilog expression generator. The function returns an object with both data and functions (behavior) also known as the module pattern. The object is used to represent a Verilog expression with behavior common to all objects created using the Verilog expression generator. The functions returned can be called like methods that perform operations on the associated object. This is because the functions are closures that retain state of the outer-scope variables when the object is created.

Usage

The factory function can be called using the following call signature:

vlogExpr( width, expr,  lastOp )

where width is a positive integer that represents the width of the expression in bits, expr is the string that holds the Verilog expression, and lastOp is a string that specifies the last operation performed to compute the expr string.

Interface

Data

The returned object will have the following data-properties:

  1. width. type: number. Represents the width of the expression in bits. Read-only.
  2. expr. type: string. Holds the computed Verilog expression string.

Methods

The returned object will provide the following methods: 

replicate ( n )

Returns a new object of type vlogExpr which is the original expression replicated by n times. n must be positive. When n = 1, the returned expression will be identical to original expression on which this method is called.

cat ( expr2, expr3, expr4, ... )

Returns a new object of type vlogExpr which is the original expression concatenated with expr2, expr3, expr4 and so on. expr2, expr3, expr4, ... are all expected to be of type vlogExpr. Zero-width expressions are removed from the concatenated result.

pad0 ( n )

Returns a new object of type vlogExpr, which is the original expression concatenated with n zeros on the MSB side. n is zero or a positive integer. If n === 0, the result is identical to the original expression.

pad1( n )

Same as pad0 except pads with ones instead of zeros.

pad0lower ( n )

Same as pad0 except pads zeros on the LSB side.

pad1lower ( n )

Same as pad1 except pads ones on the LSB side.

unaryOp ( op )

Returns a new object of type vlogExpr, which is the result of performing the specified unary operation - op. op is a single-character string and supports all unary Verilog operators including the bitwise complement operator (~).

bitwise ( op, expr )

Returns a new object of type vlogExpr, which is the result of performing the specified binary operation, op on the input expression, expr. op is a single-character string and supports all bitwise binary Verilog operators.

ternaryOp ( condition, expr )

Return type: vlogExpr. Returns a conditional expression with the expression on which this is called being the default expression ( i.e. when condition is false ). This is equivalent to condition ? expr : this, where condition and expr are expected to be of type vlogExpr, and this is the expression on which the method is called (context).

assign ( lhs, options )

Return type: string. Returns a string containing a Verilog statement expression assigning the expression, expr to the expression on which this method is called. The parameter options is optional. It provides options for:

  1. Extension: Optionally zero extend the RHS expression if its width is less than LHS, by setting extend = true. An exception is thrown if LHS width < RHS width or if extend is false and LHS width != RHS width. Default false.
  2. Indentation: Optionally specify the number of  'tabs' between LHS expression and the assignment operator. Default 2 tabs.
  3. Assignment type: Field 'type' is a string. Legal values: "procedural", "continuous". Default "continuous".
  4. Assignment operator: Optionally specify the type of assignment operator: "=" (Blocking), "<=" (Non-blocking). Default "=" (Blocking).
