Hi All,


As we are branching to multiple projects, I’ve updated the SOURCEME and pushq/pushtest setup to support various projects. Below are the steps to setup database and check in files for the various projects.


Ncore3.2.0 (Project completed):

To checkout and setup a database for Ncore3.2.0 do the following steps:

  1. setup_concerto
  2. source SOURCEME ncore3.2.0 or source SOURCEME.csh ncore3.2.0 (depending on shell you are using)

To check in any changes to hw-ncr repo for ncore3.2.0 project do normal git add, git commit and git push (Please qualify your changes locally atleast by running $WORK_TOP/dv/scripts/checkin_test -n -l -o -p). The push_ncr script will not work for this project. Since this project is already completed any check in to be made to this repo needs approval from Chien/Akarsh.

 

Ncore3.2.x:

To checkout and setup a database for Ncore3.2.x do the following steps:

  1. setup_concerto
  2. source SOURCEME ncore3.2.x or source SOURCEME.csh ncore3.2.x (depending on shell you are using)


To check in any changes to the hw-ncr repo for Ncore3.2.x project run push_ncr script and can check the pushq/pushtest job status in the below link:

All [NCore_3.2.x] [Jenkins] (arteris.com)

Note: For time being source SOURCEME ncore3.0.1 or SOURCEME.csh ncore3.0.1 would still work and will have the same effect as source SOURCEME ncore3.2.x or source SOURCEME.csh ncore3.2.x. Eventually ncore3.0.1 option will be removed from SOURCEME/SOURCEM.csh. The pushq and pushtest jobs in All [NCore_3.0.1] [Jenkins] (arteris.com) have been disabled.

To checkout and setup a database for Ncore3.4 do the following steps:

  1. setup_concerto
  2. source SOURCEME ncore3.4 or source SOURCEME.csh ncore3.4 (depending on shell you are using)


To check in any changes to the hw-ncr repo for Ncore3.4 project run push_ncr script and can check the pushq/pushtest job status in the below link:

All [NCore_3.4] [Jenkins] (arteris.com)

 

There are couple limitations to keep in mind:

  1. Once you checkout and setup a database for a particular project do not try to use/switch the same database for another project.
  2. Don’t try to switch between projects in the same terminal.

I’ll work on getting around these limitations and send out a separate email once it is fixed.


Please let me know if you face any issues or if something is not clear.
