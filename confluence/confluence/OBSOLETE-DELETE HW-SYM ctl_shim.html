Description

The CTL Shim is used as a Native layer in both the atui_ctl and atut_ctl. It maps one Common Layer Interface to another using the HW-SYM Generic Assignment Proposal. By default, it will be a direct translation from in to out CTL interfaces. The added parameter inToOutMap will override any mapping. For example, if only user needs to be modified the inToOutMap will only need to have the user signal defined.

This block will use the drivePadOrError function meaning it will drive every output signal with a direct or padded input signal.

  • If the input signal doesnt exit the output signal will be driven to 0.
  • If the output is thinner in the input a Warning will be printed
  • If an input has no output destination then a Warning will be printed.

Each atut_ctl and atui_ctl will have two shims, one for the request path and one for the response path.
