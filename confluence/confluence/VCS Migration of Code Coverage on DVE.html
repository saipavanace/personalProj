Code coverage on each ncore unit is currently achieved in multiple steps.

  1. A known regression is run on multiple iterations. These tests have code coverage enabled on them and each test generates a coverage database that are then all merged together into one database.

  2. A ‘formal’ check is run on the design that was compiled, looking for unreachable parts of the code for that configuration. (CoverCheck for Siemens & VCFormal for Synopsys). This will yield a list of exclusions that applied to the database obtained in step #1 above.

  3. A designer will examine the database from step #2 and begin to write a list of exclusions & justifications that will help close code coverage for that ncore unit. (Attaining a certain goal in percentage for a unit) This list of exclusions can be a list of modules that can be waived due to coverage on another block, uncovered wrappers like memory cells or special circumstances of untestable logic for a particular configuration.
