Release Notes

This is NCore 3.4 Beta#1 release of the performance model. 

This release add the following 4 features in addition to 3.2 release

  • Proxy Cache NCAIU interleaving only @ 64B boundaries
  • Proxy Cache Owner Transfer
  • Proxy Cache Sharer Promotion
  • NCAIU with Proxy Cache non-coherent access support
Running the model

Recommended way to run the model is by specifying a scenario file

Examples:


${NCORE_HOME}/bin/ncore3_archsim -s cpu_coherent_traffic.scn


Here is an example of scenario file (above called cpu_coherent_traffic.scn):


maestro-config=../../top.level.rtl.csr.json
input-file=cpu0:../../stl/bringup/ncore3_bringup_rd0ch.stl
input-file=acc0:../../stl/bringup/ncore3_bringup_rd0ch.stl
csr-config=../../csr_config/default.json


top.level.rtl.csr.json is the file generated by maestro in folder output/models/standalone/TLM. This file configures the performance model with the NCore3.4 configuration.

It is necessary in this release to update the version number to 3.4.0 instead of 3.2.0 manually. If this change is not done, none of the above 4 features would be activated.


Each AIU can be provided with a stimulus file. This can be in either stl format, or prf format (details in integration guide).

The model also requires a csr-config file (default.json above). This file provides the memory map and related attributes to the performance model. Following is an example


{

  "DecodedAddresBits": 33,

  "IGVId": 0,

  "CSRs": [

    {

      "regionBaseAddr": "0x0",

      "regionHui": 0,

      "regionSize": "0x10000000",

      "regionHut": "DMI",

      "NC": false

    }

  ],

  "EnableUpdtoDCE": {

    "cpu0": true

  }

}


CSRs is a vector specifying the memory map

Please refer to https://confluence.arteris.com/display/ENGR/Maestro+Note for syntax

There are 2 new parameters added with 3.4 release

NC : Set this to true to treat all matching accesses to the specified memory region as non-coherent

EnableUpdtoDCE: Specify a map of initiator name to a Boolean. Setting this to true will enable/disable update commands to DCE for accesses from that AIU. This bit affects update commands in general irrespective of the original allocating transaction is coherent or non-coherent.
