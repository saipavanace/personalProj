Native_Trace_Match

An incoming transaction on the native interface is compared with

the trace CSR settings, if there is a match the transaction is marked to be traced.

If there are multiple sets of registers (nTraceRegisters>1), then each set of triggers will be OR’d with each

other.

Native_Address_Match


Program belowe register
TBALR{n}_base_address_lo_out : Lower order bits 43:12 of the base address of the region
TBALR{n}_base_address_hi_out : Higher order bits 51:44 of the base address of the region
TCTRLR{n}_size_out : Address range size, this field indicates a binary number from 0 to 31 from which the region's Size is calculated as (Size of IG) * 2 ^(Size +12) bytes

Native_Memory_Attribute_Match

Program belowe register
TCTRLR{n}_memory_attribute_out:
Trace if there is a match on the memory attribute specified
0 : Disable memory attribute match trigger.
1 : Enable memory attribute match trigger.

TCTRLR{n}_memattr_out:
[31:28] AxCache field
[27:26] for AR and AW respectively, if both are set match is done on both AR and AW
[25:24] reserved

Target_Type_Match

Program belowe register
TCTRLR{n}_target_type_out
Trace if there is a match on the target type
0 : Disable target type match trigger.
1 : Enable target type match trigger.

TCTRLR{n}_hut_out
Specifies target type DMI or DII
0 : DMI target selected.
1 : DII target selected.

TCTRLR{n}_hui_out
Specifies the HUI (NunitID for DII and MIG
number of DMI)
