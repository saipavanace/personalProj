Directory

  • workspaces for jenkins runs are created in /scratch/jenkins/angelo_ncore3_<type>/<date>_<time>. 
  • nightly regression scripts will automatically remove the directory if everything passes, or remove everything older than 7 days.
  • Note that Jenkins will automatically remove things in it's local workspace after a while, but since that local space is not accessible to the grid, you can't run on the queue if you need to access that space

Check In Regression

  • http://jenkins.arteris.com:8080/job/NCore3/angelo_ncore3/
  • Runs after every commit and once per night
  • If any fail, email is sent to whoever did the checkin
  • Tests run:
    • ../perf-vc/tests/bin/run_list.pl --model 'bin/ncore3_archsim --csr-config ../perf-vc/tests/csr_config/default.json' --testlist ../perf-vc/tests/regress/testlist/checkin.list --queue
    • ctest -j8

Nightly Random

  • http://jenkins.arteris.com:8080/job/NCore3/job/NightlyRandom
  • Runs once per night
  • If any fail, email is sent to Eyck Jentzsch and all developers which contributed changes
  • List of configuration can be found in http://gitlab.arteris.com/maestro-dev/packages/angelo/blob/integration/contrib/Jenkins/Random, variables REPO_CFG and EX_CFG
    • this list is compiled based on the DV used configurations. The state of them can be found at http://dev.arteris.com/dv_reg_ncore/reports/

Merging Integration into Master

  • http://jenkins.arteris.com:8080/job/NCore3/job/NightlyMerge2Master/
  • Runs once per night
  • If any fail, email is sent to Eyck Jentzsch, and nothing is pushed into Master.

Generate Latest Jsons from hw-test-projects

  • http://jenkins.arteris.com:8080/job/NCore3/job/UpdateMaestroJsons/
  • Runs once per night
  • If any fail, email is sent to Steve Kromer (Deactivated), and nothing is pushed into Integration perf-vc
  • If all pass, new jsons are pushed to perf-vc, and the link to hw-test-projects is updated.
  • Need to relook at whether these are worth generating directly at compile time, for now, these provide a stable json for at least a day to do debug on.

Testing Jenkins Changes

When making modifications to commands used by the Jenkins jobs, I usually try them out here first:

http://jenkins.arteris.com:8080/job/Testing/job/stevek_ncore3/

That way, the results while changing and debugging don't clutter the statistics of the normal runs.
