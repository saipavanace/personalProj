Description

apb_pipe is a simple single synchronous pipe stage that can be placed in the path of an APB bus. apb_pipe uses old-school interfaces, while apb_pipe_adapter uses the newer interface definition. apb_pipe_adapter instantiates apb_pipe.

The prdata and pwdata buses share a common data register in order to save flops. 

A two stage shift register is used to gate the psel andpenable to the master in order to account for the round-trip impact of master pready -> slave pready -> slave psel/penable -> master psel/penable.

Only address and data are power-gated.

pstrb, pprot, and pslverr are optional, and can be omitted by setting their corresponding parameter widths to zero.

There is currently no option to send in protection information to the flops.
