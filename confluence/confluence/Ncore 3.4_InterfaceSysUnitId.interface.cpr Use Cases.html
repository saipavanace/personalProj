The InterfaceSysUnitId.interface.cpr file for Ncore 3.4 is now added with a field called “connectivity”.

So, there are two fields in the InterfaceSysUnitId :

-“f_unit_id” which carries an array of f_unit_id that is flattened and concatenated as a bit vector

-“connectivity” which carries an array of connection bit that is flattened and concatenated as a bit vector


For Ncore 3.4, InterfaceSysUnitId.interface.cpr has three use cases:

-System Connectivity

-System Event and Coherency Messaging

-Snoop Filter Caching Agents

Use Case 1: System Connectivity


AIU to DMI Connectivity

  assign caiu0_uSysIdInt_dmi_connectivity = 3'b111;

 assign caiu0_uSysIdInt_dmi_f_unit_id = {4'h9,4'h8,4'h7};


AIU to DII Connectivity

  assign caiu0_uSysIdInt_dii_connectivity = 2'b11;

  assign caiu0_uSysIdInt_dii_f_unit_id = {4'hb,4'ha};


AIU to DCE Connectivity

  assign caiu0_uSysIdInt_dce_connectivity = 2'b11;

  assign caiu0_uSysIdInt_dce_f_unit_id = {4'h6,4'h5};


DCE to DMI Connectivity

-for Recall snoop request issue to snooping AIUs


Use Case 2: System Event and Coherency Messaging

NOTE: InterfaceSysNodeUnitId.interface.cpr is used in conjunction with InterfaceSysUnitId.interface.cpr for SystemEvent and Coherency Messaging

NOTE: InterfaceSysNodeUnitId.interface.cpr is used to carry the NUnitId array that corresponds to the FUnitId array in InterfaceSysUnitId.interface.cpr.


AIU SysCoh Sender needs a Target FUnitId array for sending SysCo.Attach/Detach message to DCE.

  assign caiu0_uSysIdInt_dce_f_unit_id = {4'h6,4'h5};


AIU SysCoh Sender needs a Target FUnitId array for sending SysCo.Attach/Detach message to DVE.

  assign caiu0_uSysIdInt_dve_f_unit_id = {4'hc};


DCE SysEvent Sender needs a Target FUnitId array for sending exclusive monitor SysEvent message to AIU.

  assign dce0_uSysIdInt_ca_f_unit_id = {4'h2,4'h3,4'h0,4'h1};


Use Case 3: Snoop Filter Caching Agents

  assign dce0_uSysIdInt_ca_f_unit_id = {4'h2,4'h3,4'h0,4'h1};
