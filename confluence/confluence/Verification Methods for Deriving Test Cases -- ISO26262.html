Description

DV Application

1a – Analysis of requirements

DV team members will analyze the requirement document which also includes Ncore architecture specification & micro-architecture specification.

1b – Analysis of internal and external interfaces

DV team members will analyze all external (standard) specifications for example, AXI, ACE, CHI… etc.

In addition DV team will also analyze internal interface and protocols specific to Ncore product line, for example, CCP, SMI etc.

1c – Generation and analysis of equivalence classes

An equivalence class is the name that we give to the subset of S which includes all elements that are equivalent to each other. “Equivalent” is dependent on a specified relationship, called an equivalence relation. If there’s an equivalence relation between any two elements, they’re called equivalent. The equivalence class of a consists of the set of all x, such that x = a’.

1d – Analysis of boundary values.

DV team will analyze all possible corner cases, boundary values of address, data and control buses.

1e – Knowledge or experience based error guessing.

Analyze all possible error cases which are defined in specification and which are not defined. Which will also includes hang scenarios.

1f – Analysis of functional dependencies.

1g – Analysis of common limit conditions, sequences and sources of dependent failures.
