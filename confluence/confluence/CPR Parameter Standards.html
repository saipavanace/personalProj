Parameter Naming Conventions

Within a block

Hierarchical name.

Example

@interface.inInterface.param.nVc

Example with Array:

@interface.inInterfaces[].param.nVc

Between Blocks, Packets and Interfaces

blockName(id).hierarchical name

packetName.hierarchical name

interfaceName.hierarical name

Example

@sym_switch(24).interface.inInterfaces[].param.nVc

Between Sections

Names stay hierarchical.

For blocks, packets, interfaces remains the same.

top.hierarchical name

network.hierarchical name

request.hierarchical name

response.hierarchical name

Constraint Format

Use C++ format with tagged variables. It's in the form of a function that returns a boolean. True means the constraint is met.
Need to be able to refer to the number of entries in an array, the values in arrays and correlate sets of arrays by index.

Use .size to refer to the length of an array:

example: return @interfaces.inInterfaces.size() == @interfaces.outInterfaces.size();

Mapping Equations

Use C++ format with tagged variables

@postMapped
#preMapped 

example:

 @interfaces.inInterface.params.nVc = #network.nVC
