Description

SRAM-based FIFO with Memory Protection.

The SRAM has 1 Read Port and 1 Write Port.

Memory Protection can be configured as NONE, PARITYENTRY, or SECDED.

When Memory Protection is configured as PARITYENTRY or SECDED, the SRAM address is included in the protection, together with the data.

When the FIFO is empty, input valid/ready/data is bypassed to output valid/ready/data.

The control module “sym_fifo_mem_2p_ctl” interfaces with the input valid/ready and the output valid/ready, and generates SRAM write and SRAM read, and interfaces with the “self_correcting_fifo” module’s input valid/ready and output valid/ready. The SRAM read data is directly written to the “self_correcting_fifo”.

Write data forwarding is provided in the clock cycle when SRAM Is being read and the read data is not available yet. This can happen when there is back-to-back writes coming in when the SRAM-based FIFO is empty. Write data forwarding ensures that the output valid remains high with forwarded write data.
