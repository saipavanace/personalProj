Maestro’s Ncore3 branch has renamed the packet route names to a more concise name (MAES-4224).

The new packet route name format is <network>_<fromUnit>_<outPort>_<toUnit>_<inPort>


For example: ndn1_caiu0_O0_dmi0_I0

where <network> = ndn1; <fromUnit> = caiu0; <outPort> = O0; <toUnit> = dmi0; <inPort> = I0


In addition, the initiator and target packet port names have also been renamed. As a result, any manual routing designs will need to be updated with the new packet port name.


Here is an example of how to convert from the old packet port name to the new names.


Packet_port name:

<unitName>_pkt<N>Z0  becomes O<N> ; the new name is uppercase letter ‘O’

<unitNmae>_dpkt<N>I0 becomes I<N> ; the new name is uppercase letter ‘I’


For example:

Original name

New Name

<unitName>_pkt0Z0
O0
<unitName>_pkt1Z0
O1
<unitName>_pkt2Z0
O2
<unitName>_pkt3Z0
O3
<unitName>_dpkt0I0
I0
<unitName>_dpkt1I0
I1
<unitName>_dpkt2I0
I2
<unitName>_dpkt3I0
I3




Old way to create packet routes:

update_object -model route -add [list $ts_cni_caiu0/caiu0_pkt2Z0 SW1 $ts_cni_ncaiu6/ncaiu6_dpkt2I0 ]

New way:

update_object -model route -add [list $ts_cni_caiu0/O2 SW1 $ts_cni_ncaiu8/I2]
