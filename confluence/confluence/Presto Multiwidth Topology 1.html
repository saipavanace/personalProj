The design is located at:

$WORK_TOP/tb/top/jcodd_multi_width_cfg1_ni

Create a regression area: In that area type the command:

./hw/rtl/lib/bringup/run_multiwidth_ni

This will create the RTL and put it in a directory at the same level called:

multi_width_cfg1_ni

To do a quick lint check on the code type the following:

cd multi_width_cfg1_ni

verilator --lint-only ./gen_wrapper.v -I. --error-limit 0 -Wall -Wno-UNUSED

You can compile it with cadence vsim by doing the following:

vsim &

Once vsim has started type in the following commands:

vlib work
vmap work work
vlog -reportprogress 300 -work work /home/jcodd/work/regress/multi_width_cfg1_ni/*.v

Once you have the above clean you can move to the next step: 

vopt +acc dut -o dut_opt

You can get more errors that need to be cleaned up with the above. Once that's clean:

vsim dut_opt

Plot all the I/O

run 100

Look for Zs and undrivens. 

If the above verilator command returns any warnings or errors, the code needs to be cleaned up until nothing prints out. This still won't guarantee  a clean synthesis compile, but it's pretty close.
