Overview

Ncore only takes Odd_Parity_Byte_ALL option once Check_Type is enabled, and this is applicable to all the following interfaces an IOAIU supports:

  • AXI5

  • ACE5

  • ACE5_Lite

  • ACE5_LiteDVM

CHK signals should have correct value based on Odd parity. This means that check signals are added to groups of signals on the interface and driven such that there
is always an odd number of asserted bits in that group.

Use snps vip to drive correct parity signals.
