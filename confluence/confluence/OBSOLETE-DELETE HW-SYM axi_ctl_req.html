module_name

axi_ctl_req

JavaScript parameters

userMap: A parameter that overwrites default mappings in AXI2CTL library used mainly for user field mapping.

Interfaces

clkInterface: A slave interface of type InterfaceCLK.

axiArInterface: A slave interface that is the AR sub-interface of InterfaceAXI.

axiAwInterface: A slave interface that is the AW sub-interface of InterfaceAXI.

ctlReqInterface: A master interface that is the InterfaceCTLReq sub-interface (ctl_req_) of HW-SYM InterfaceCTL.

Description

This component translates requests on the AW and AR sub-interfaces of the AXI interface to CTL request interface (ctl_req_). The read and write requests on AXI are multiplexed onto a single CTL request channel.

The block employes a simple 2-state round-robin arbiter which keeps track of the last request type (read or write). When both read and write requests are valid on AXI, read wins if the last request type granted was a write, otherwise write wins.

The mapping between AXI to CTL signals is done using HW-SYM class AXI2CTL (InterfaceMapLibs.js) libraries and tables and the userMap JSON parameter. The length translation logic determines the length of the transaction in bytes for CTL using request address and length of transaction in beats (on AXI).
