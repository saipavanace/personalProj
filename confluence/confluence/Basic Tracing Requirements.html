• Ability to filter transactions for tracing
    • Examples of filtering criteria: Source, destination, type of transaction (read, write, exclusives, etc.), user bits, address ranges, VN, VC, other transactional attributes
    • Good to have: can have multiple filters, and perform a LUT between filters to determine final filter hit
    • raise an interrupt when a trace event is triggered by a filter hit
  • Ability to trace header, or both header and data
  • Flow control - no need for flow control; monitor packets only: i.e. drop packets if end point cannot sustain bandwidth.  Upon dropping of packets, there needs to be an overflow packet to inform system that packets are being dropped, and raise an interrupt
    • Flow control mode (ability to backpressure data path) – not required in basic mode
  • CTI triggering - not required in basic mode
  • Output format
    • ATB, with optional STPv2 for timestamping.  Timestamps should indicate the time that the trace happened.
    • RAW format - not required in basic mode
