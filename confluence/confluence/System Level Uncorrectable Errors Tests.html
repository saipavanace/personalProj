This document lists the uncorrectable error scenarios described in the Concerto System Architecture Specification (Revision B, Version 0.1). The specification lists these major categories of uncorrectable error scenarios :

1) Data Errors

2) Transport Errors

3) Propagated Errors

 

Update from Chirag: Dont add checking for error logging and interrupts yet.

Additional details of each scenario as described in the specification, are tabulated here :

 OTT Data error

(a) configure for parity; single bit corrupted

(b) configure for SECDED ECC; multi bit corruption

For each (a) and (b) configurations :

1) Agent read response – verify each corrupted data beat has data corruption error indicated

IO Cache Data error

(a) configure for parity; single bit corrupted

(b) configure for SECDED ECC; multi bit corruption

For each (a) and (b) configurations :

run one scenario with,

1) native agent read request – verify that NCB marks each data beat in the native agent read response with data corruption error if that data beat contains a corrupted byte.

2) native agent write request – verify that NCB indicates a data corruption error in the write response

3) DTWreq message due to IO cache eviction – verify that NCB deasserts the byte enable for each corrupted byte

4) Protocol snoop message – verify that NCB completes the snoop message and marks each corrupt data beat in the DTRreq message with a data corruption error. In the snoop response message -  verify data corruption error is NOT indicated

DCE :

Transport Power Management Error

 SNPreq message, HNTreq message, MRDreq message, STRreq message is completed with a disconnected power domain error

Verify upon receiving a SNPrsp message with this error, the DCE interprets the corresponding snoop result {RV, DN, DC, DT} as {0, 0, 0, 0} and logs a transport error in the corresponding ATT-Ctrl resource.

Verify upon receiving a HNTrsp message with this error, the DCE logs a transport error in the corresponding ATT-Ctrl resource.

In case of a protocol transaction, once the required SNPrsp messages and HNTrsp messages have been received, the DCE terminates the protocol transaction and signals a transport error to the requesting AIU/NCB in the STRreq message.

Verify upon receiving a MRDrsp message with this error, the DCE signals a transport error to the requesting AIU/NCB in a control DTRreq message.

Verify upon receiving a STRrsp message with this error, the DCE completes the protocol transaction, including performing the directory commit.

AIU :

Propagated Data Corruption Error (data-propagated)

 AIU receives a data corruption error in a data transfer DTRreq message.

Verify the requesting AIU completes the protocol transaction and marks each data beat in the native agent read response with a data corruption error if that data beat contains a corrupted data byte.

Verify AIU indicates a data corruption error in the native agent write response.

AIU :

Native Snoop Response Data Error 

Snooping AIU issues a native snoop request and verify for each of these scenarios :

-          The native snoop response indicates an error and indicates a data transfer

-          The native snoop response indicates a data transfer and the snooping AIU issues a DTWreq message that completes with a data corruption error

Veify that if the snoop response indicates an error and a DTWreq message is required in response to the SNPreq message, the snooping AIU does not issue the DTWreq message.

Verify that the snooping AIU issues SNPrsp message with data corruption error indicated

AIU :

Poisoned Data Error

Native agent transaction or a snoop message results in the propagation of poisoned data from the IO cache.

Verify that in the case of an IO cache hit due to a native agent transaction, the NCB completes the transaction and marks each data beat in the native agent read response with a data corruption error if that data beat contains a corrupted data byte or indicates a data corruption error in the native agent write response.

Verify that in the case of an IO cache miss due to a native agent transaction, the NCB performs the update transaction and marks each data beat in the DTWreq message with a data corruption error if that data beat contains a corrupted byte
