Each defined individual hardware unit shall have its own CSR definition file. There shall be a master CSR file to describe commonly used register format like error reporting registers. The master file shall reside in the library repository. 

Example for an Ncore individual hardware unit is AIU/DCe/DMI ....etc

The registers shall be specifiable using Parameters. 

The golden source shall be checked into GIT repository with source tACHL

The golden source shall be used to generate IPXACT, Documentation and base tACHL code

Unit level CSR file: csr_unit.md

 

Register Fields:

  • name: Name of the register field 
  • bitoffset:  
  • width: width of the field 
  • SW Access: software access type
    • RO Read only
    • WO Write only
    • RW Read write
    • W1C Write 1 to clear
  • HW Access: Hardware access type
    • RO Read only
    • WO Write only
    • RW Read write
  • reset: field reset value
  • link: Link to another register filed that this field is related to. If not related to any other register field set to 'NIL' 
  • link_op: Operation on the linked register 
    • SOR set 1 on read
    • COR clear on read
    • SOW set 1 on write
    • COW clear on write
    • NIL 
  • op_order: operation order priority in case the register field is simultaneously being accessed by either software or hardware
    • SW (link accesses is considered as software access)
    • HW
  • scope: visibility of the scope of the register
    • ENG: Not visible to customers, i.e. not documented in reference manual and IPXACT  
    • GUI: Visible to customers
  • description: Description of the register field.
