Flow implementation of reproduction of TCL cfg fail using gen cfg flow with following steps:

1. Check-in run.tcl file which includes parameters with random values.
  Eg: maestro_4x4_32_AXI4_128_36_AXI4_128_36_AXI4_128_40_AXI4_128_36_32_AXI4_128_40_AXI4_128_32_AXI4_128_40_AXI4_128_32_07Dec20_same_width_striping_random_params.tcl

has parameters maxPduSz, fixedSupported, wrapSupported, narrowSupported, readInterleaveSupported set to more than 1 value.
 
2. Include the run.tcl as part of testlist, in above case it has been included in top_axi_axi.json and tests_nl.json and associated with only "axi4_single_beat_wr_rd" flavors of tests.

2. Run regression with above run.tcl file and debug the fall out.

3. Copy rtl.json, dv.json and associated cpr files into gen_cfg directory, add new config with bug number into testlist.
   gen_cfg directory name and config name: <tcl_cfg_name>_<Bug_number>

Eg: maestro_4x4_32_AXI4_128_36_AXI4_128_36_AXI4_128_40_AXI4_128_36_32_AXI4_128_40_AXI4_128_32_AXI4_128_40_AXI4_128_32_07Dec20_same_width_striping_random_params_<bug_number>
  
4. Add config to corresponding config and test files Eg: In this case into top_axi_axi.json and tests_nl.json

5. Add following flavor to the config in tests_nl.json 
    "flavors":

       {"<bug_no.>": {"iter": 1, "label": ["striping"]}}

6. Provide runsim command for newly created gen_cfg in the bug for reproducability. Make sure the new config-test pair creates only 1 test.
Eg: runsim -c maestro_4x4_32_AXI4_128_36_AXI4_128_36_AXI4_128_40_AXI4_128_36_32_AXI4_128_40_AXI4_128_32_AXI4_128_40_AXI4_128_32_07Dec20_same_width_striping_random_params -l striping -h, lists only 1 config-test pair.
