1 - Library Presentation :

Pmon files are under: /hw-ncr/dv/common/lib_tb/perf_counter/

Files:

  • event_list.json : To define event, it’s used to generate different files.

  • latency_counters_scoreboard.svh  : latency scorebord

  • latency_if.sv  : latency interface used to spy and generate alloc and dealloc signals : It’s used by latency_scoreboard

  • perf_cnt_pkg.sv : Pmon package

  • perf_cnt_test.sv : Pmon test

  • perf_cnt_unit_cfg_seq.sv : Pmon configuration sequence

  • perf_cnt_unit_defines.svh : Used to define Pmon objects types.

  • perf_cnt_unit_macros.svh : Used to define Marcos

  • perf_cnt_units.sv : Pmon sequence item

  • perf_counters_scoreboard.svh : Pmon scoreboard

  • stall_if.sv an interface  used to collect informations and signals from dut and blocks scoreboard. It’s used also to generate different Pmon events.

2- BW counter integration

To integrate DV updates related to PMON 3.4 features you should :

  1. Add new BW events to “hw-ncr/dv/common/lib_tb/perf_counter/event_list.json” for your block as DII example below : 

    {"evt_idx":17,"name":"DtwReq","itf_name":"dtw_req","width":1,"type":"bw","comment":"Count all DtwReqs that come into DII"},
{"evt_idx":18,"name":"DtrReq","itf_name":"dtr_req","width":1,"type":"bw","comment":"Count all DtrReqs that are issued by DII"},

    Please choose name of the event and itf_name to be used into stall_if as significant name

  2. Grep for ‘//Pmon 3.4 feature’ in /hw-ncr/dv/common/lib_tb/perf_counter folder and add your block in the Javascript if statements after each comment having as text : //Pmon 3.4 feature

    //Pmon 3.4 feature
<% if (obj.BlockId.includes("dii") || obj.BlockId.includes("<your_block>")) {  %>

  3. Define BW filter value constraints in perf_cnt_units.sv : BW filter value can be user bits or Funit ID
    NB : Search for // add your bw filter constraint here //
    You should protect your code because constraints depends on each block
    Noe that below constraints are block defined according to constraint in your block sequence.

    example of DII : 

        constraint c_bw_filter_value{
        
        foreach(bw_filter_value[i]) {
            if (bw_filter_select[i]) { bw_filter_value[i] inside {[0:<%=obj.DveInfo[0].nAius%>]};}
            else  { bw_filter_value[i] inside {[5:15]};} 
        }
    }
    
    constraint c_bw_filter_mask{
        
        foreach(bw_filter_mask[i]) {
            if (bw_filter_select[i]) { 
                ($urandom_range(1,<%=obj.DveInfo[0].nAius%>) & bw_filter_mask[i]) inside {[1:<%=obj.DveInfo[0].nAius%>]};
            } // dii funit id values
            else  { 
                ($urandom_range(5,15) & bw_filter_mask[i]) inside {[5:15]};
            } // dii user bits values
        }
    }

  4. Connect BW signals in your block_tp_top.sv file :
    NB : Connect signals with generated names in stall_if : <event.itf_name> is defined in event_list.json and it will be generated in stall_if

    1. <event.itf_name>_valid : 

      assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_stall_if.dtr_req_valid = dut.u_dii_unit.dtr_req_valid;

    2. <event.itf_name>_ready : 

      assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_stall_if.dtr_req_ready = dut.u_dii_unit.dtr_req_ready;

    3. <event.itf_name>__funit_id_if : 

      assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_stall_if.dtr_req_funit_id_if = dut.u_dii_unit.dtr_req_target_id;

    4. <event.itf_name>__user_bits_if : 

      <%if (wArUser > 0){%>assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_stall_if.dtr_req_user_bits_if = dut.u_dii_unit.pmon_dtr_req_user_bits; //dut.u_dii_unit.dtr_req_aux;<% } %>

  5. Add Pmon BW tests to your runsim test list :
    Try to do like old Pmon tests of your block and add args below to define your requirements test for BW

"+perfmon_32bit_mode":null, ==> to enable  32bit mode

  "+event_first":"18", ==> define first event as your bw event

  "+event_second":"30", ==> define second event as div16 event

    "+ssr_count":"4", ==> use as 32 bit counter

    “+counter_control":"4", ==> use as 32 bit counter

    "+min_stall_period":"0",

"+pmon_bw_test":null, ==> to enable bw test

To enable and define specific bw filter attributes you should use  :

"+filter_enable":"1", ==> to enable bw filter

     "+bw_filter_select":"1", ==> to set filter select (default is a random value)

      "+bw_filter_value":"04", ==> to set filter value (default is a random value)

"+bw_filter_mask":"FFFFF" ==> to set filter mask (default is a random value)

NB : Make sure that you use "+bw_filter_mask":"FFFFF" (value should be FFFFF) in your BW filter bring-up test.

3- Latency counter integration

To integrate DV updates related to Latency feature you should :

  1. Make sure that your have already added your block after each comment //Pmon 3.4 feature as explained in BW integration part.

  2. Grep for “//Pmon 3.4 latency” in /hw-ncr/dv/common/lib_tb/perf_counter folder and add your block in the Javascript if statements after each comment having as text : “//Pmon 3.4 latency” 

  //Pmon 3.4 latency
  <% if (obj.BlockId.includes("dii")) {  %>
  `include "<%=obj.strRtlNamePrefix%>_latency_counters_scoreboard.svh"
  <% } %>

3. You need to define params below in latency_if.sv file for your block :

if (obj.BlockId.includes("dii")) {
          nWttEntries = obj.DiiInfo[obj.Id].cmpInfo.nWttCtrlEntries;
          nRttEntries = obj.DiiInfo[obj.Id].cmpInfo.nRttCtrlEntries;

}

for IOAIU or CHI nWttEntries and nRttEntries will take the same value which is nOttEntries or you can just define “nXttEntries” in same file latency_if.sv

4. You need to do below steps in your <block>_tb_top.sv :

a. Instantiate latency_if

b. implement uvm_config_db set and make sure that field_name in set method has as suffix “_m_top_latency_if”

// Latency if
<%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_latency_if <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if(); // PERF_CNT Latency_IF
initial uvm_config_db#(virtual <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_latency_if)::set(null, "", "<%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_m_top_latency_if",       <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if);

c. connect below signals of latency_if : 

/////////////////////// PERF_CNT CONNECT DUT to LATENCY LATENCY_IF////////////////////////////////////////////////////

assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if.clk              =   dut_clk;
assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if.rst_n            =   soft_rstn;
assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if.alloc_if         =   dut.u_dii_unit.u_ncr_pmon.latency_counter_in_alloc;
assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if.dealloc_if       =   dut.u_dii_unit.u_ncr_pmon.latency_counter_in_dealloc;
assign <%=obj.DiiInfo[obj.Id].strRtlNamePrefix%>_sb_latency_if.dut_latency_bins =   dut.u_dii_unit.u_ncr_pmon.latency_bins;

/////////////////////// PERF_CNT CONNECT STALL_IF TO DUT////////////////////////////////////////////////////

4. Add Pmon Latency tests to your runsim test list :
Try to do like old Pmon tests of your block and add args below to define your requirements test for latency :

"+perfmon_local_count_enable":null, ==> to set local count enable register to 1
"+perfmon_32bit_mode":null, ==> to enable 32 bit mode
"+perfmon_local_count_clear":null, ==> to set local clear register to 1

"+latency_count_enable":null, ==> to enable latency count enable register

"+pmon_latency_test":null, ==> to enable latency test (enabling Pmon latency sequence and scoreboard check)

"+lct_bin_offset":"00", ==> to set latency bin offset value (default is a random value)

"+lct_pre_scale":"0", ==> to set latency pre_scale value (default is a random value)

"+lct_type":"1", ==> to set Latency type (rd/wr)
