This table is required for Ncore testbench automation. It's written from Aria in a form of a json file, here is an example 

{
      "rtlSignal": "dmi0__mst_ar_len",
      "dvSignal": "ioaiu_if0.arlen[7:0]",
      "info": "Interface dmi0__mst_ (InterfaceAXI), connected to instance /myChip/dmi0 (dmi)"
    },

rtlSignal consists of one or more prefixes which are interface instance names  (here dmi0_mst_) and a signal name inside the interface (ar_len)

dvSignal is constructed by genNcoreTbConnectivityMap as shown below in the snippet of Javascript code

 

// 1. Require Aria as a library
let {genNcoreTbConnectivityMap,ncoreTbNamingMap} = require( '/engr/dev/releases/utils/aria/latest/aria-linux-x64/resources/app/index'); 
 
// 2. Initialize Ncore signal TB naming rule mapping table
let nMap = new ncoreTbNamingMap('ncore','fsys');

// 3. Define names of interfaces and ncore units for which mapping rules may be defined
nMap.legalInterfaces = ['InterfaceAXI','InterfaceCTL','InterfaceSMI','InterfaceATP','InterfaceAPB','InterfaceCHI','InrefaceCLK'];
nMap.legalNcoreUnits = ['aiu_top','ioaiu_top','dve','dce','dmi','dii_top','grb'];

// 4. Add prefix lookup rules
// a. If both interfaceDef and unitDef are specified then prefix is only applicable for 
//  the combination of interface and unit. So here m_chi_if prefix will only appear for InterfaceCHI 
//  which connect to instances of aiu_top 
nMap.addPrefixRule({interfaceDef:'InterfaceCHI',unitDef:'aiu_top',tbInstanceName:'m_chi_if'}); 
// b. If only interfaceDef is specified, then rule becomes default for this interface
//  definition for all unit types. Here ioaiu_if prefix will be used for all instances 
// of InterfaceAXI
nMap.addPrefixRule({interfaceDef:'InterfaceAXI',tbInstanceName:'ioaiu_if'});

// If system can't find appropriate name in the lookup table, then tb name simply falls 
// back to the rtl signal name  
// 5. Add suffix naming changing (typically regsub) rules
nMap.addRegsubRule({
  interfaceDef:'InterfaceAXI',
  regsubFunc: function(str) {return str.replace(/^ar_/,'ar')}
})

// 6. Now Generate signal mapping table
genNcoreTbConnectivityMap({
  // Mandatory parameters
  rtlJsonFile: '/home/bennyw/testcases/rtl_json/hw_config_two.bad.json',   // RTL DUT file in genwrapper format
  outputFile: 'tb_signal_map_hw_config_two.json',                          // File which will contain signal mapping table
  tbNamingMap: nMap,                                                       // Previously defined naming rules
  // Optional parameters
  checkDut: {run:true, exitOnError:false}, // Check validity of of RTL json and abort if errors are found
  msgPrefix: 'ARIA', // Add prefix to all message printed by Aria
  verbose: true      // Run in verbose mode, useful for debug
});

When checkDut is enabled a checker procedure is launched right after loading RTL json. It reports individual errors and a summary table at the end 

if user checkDut: {run:true, exitOnError:false}   and errors are detected in RTL json, assertion is raised thus aborting the entire process

Checker procedure can also be launched from interactive Aria as ncoreChecker.run() command
