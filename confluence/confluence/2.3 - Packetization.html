As mentioned in the Overview section, Symphony uses a packetized transport. Native layer transactions are converted into packets and transported across the fabric by the Initiator ATU (ATU-I). The Target ATU (ATU-T) converts the packets back into packets. Symphony offers three different styles of packetization mechanism. 

These different styles are listed below: 

    • Header parallel mode 
    • Header serial mode
    • Fixed width mode

The Packet Style is the way data is packed in the packet . Header Parallel means the first beat of a packet contains both header and data information and if subsequent data beats are needed, the following beats contain no header. The width of the ATP link in the header parallel style is set by the width of the header and width of the Data Phit.  Nextra packetization latency added in this style.

In the Header serial mode, the header is sent in the first beat and the subsequent beats carry any data if present. The width of the APT link the maximum of the width of the Header or the data PHIT. The Header serial style results in narrower ATP links but adds a single cycle packetization latency over the Header parallel mode. In the Header serial mode, the header is carried in a single beat. 

The Fixed Width mode, is a variant of the Header Serial mode, where the header can span multiple beats. The data beats follow the header. The width of the ATP link set by the maximum of the widest header field except the address and the Data PHIT. Note that the restriction is that a field in the header has to be contained in a single Flit, except the address. 

The ATP link is defined by the following parameters: 

  1. Maximum packet size.
  2. Flit Data Width
  3. Packet Style
    1. Header parallel (referred to as parallel)
    2. Header serial (referred to as serial)
    3. Fixed width (referred to as fixed)

The maximum packet size is the maximum data load in bytes a packet can carry for a given fabric.

The Data PHIT is the with of the data sent in a beat of data on the ATP link.
