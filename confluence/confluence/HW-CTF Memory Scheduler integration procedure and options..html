Here are all the options for address generation at the initiator

Address_mode 0 : randomizes within a page for a giver initiator and flow.

Address_mode 1 : totally random

Address_mode 2 : sequential addressing


Burst option can be added per memory scheduler as

{ "name": "m0", "kind": "mem_sch", "attrb": { "num_txn": 10000, "buff_size"  : 40, "buff_dly"   : 0, "buff_drain" : 1, "level": 11, "rr_wt" : "1", "bursts" : 1, "sram_drain" : 20 }}

Burst = 1  sends no_of_phits*4 sizes of bursts onto the schedulers.


And sram_drain option will drain once every n clocks as specified. sram_drain_start,sram_drain_end options would set the drain rate only in that window.
