This is a JS class that can be found in the following path and file:
hw-lib/js/lib_utils.js

Base Class

None

Composition

An instance of the CreateCSRObj class consists of ("has a") objects of the following class:  

HW-SYM class CreateRegCSRObj

Purpose

This class was created to work with the HW-CTF sym_apb_csr ( sames as apb_csr) component. It effectively converts the csr into an object that can still access the original properties of the csr data-structure but is also augmented with methods to parse and extract useful data from the csr data-structure that the instance of the class is initialized with.

The class is meant to be used both by the apb_csr component as well as any TACHL component that instantiates it. 

Interface (Methods)

The class provides the following interface methods: 

constructor( CSR )

This is the constructor function used to initialize the instance of the class. It accepts only one parameter or data-structure which follows the CSR specification described here: csr. The constructor does two things:

  1. Adds methods at the same level as the properties of the csr at the top-most hierarchical level.
  2. Replaces every entry in the registers property inside every spaceBlock with an instance of the HW-SYM class CreateRegCSRObj.


getHWPortName( regName, fieldName, type )

This method provides a standard way to construct port names for the apb_csr component. TACHL components that instantiate the apb_csr are expected to use this if they need to create the port names individually instead of using the getHwPorts( ) method. This guarantees that the port names used are the same as what the apb_csr component generates inside it. When port-names are generated only on a per-register basis, then fieldName is set to empty string "". The type can take the following values: "in", "out", "wr", "wr_en", "rd_en", "sw_wr", "sw_rd", and "tz_en".


getHWPortsIn(  params )

This method returns an object with input port names for apb_csr module as keys and their signal widths (in bits) as values. This does not include the names of ports that are part of "Interfaces" described in apb_csr. The params parameter is optional and maybe skipped. 


getHWPortsOut(  params )

This method returns an object with input port names for apb_csr module as keys and their signal widths (in bits) as values. This does not include the names of ports that are part of "Interfaces" described in apb_csr. The params parameter is optional and maybe skipped. Or it can include the following parameters as keys:

  1. enSwWritePorts: An integer that is either one or zero. Setting to 1 creates software write ports (See under the section "ports" in apb_csr).
  2. enSwReadPorts: An integer that is either one or zero. Setting to 1 creates software read ports (See under the section "ports" in apb_csr).


getHWPorts( params )

This method returns an object with all the hardware ports mentioned in apb_csr as keys and signals widths as values. This does not include the names of ports that are part of interfaces. 

The params parameter is optional and maybe skipped. Or it can include the following parameters as keys:

  1. enSwWritePorts: An integer that is either one or zero. Setting to 1 creates software write ports (See under the section "ports" in apb_csr).
  2. enSwReadPorts: An integer that is either one or zero. Setting to 1 creates software read ports (See under the section "ports" in apb_csr).
