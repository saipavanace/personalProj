Ncore3.4/proxy_cache_promotion

Sharer Promotion and Owner Transfer

no parameter control, internal RTL tie-off

Set the following internal variable “useSharerPromotion” to 1 to enable the feature.

 

For DCE:

rtl/dce/src/dce_att_entry.tachl:let useSharerPromotion = 1;

rtl/chi_aiu/src/snp_dat_gen.tachl:var useSharerPromotion = 1;

rtl/chi_aiu/src/snp_resp_gen.tachl:var useSharerPromotion = 1;

rtl/chi_aiu/src/stt_entry.tachl:var useSharerPromotion = 1;

For IOIAU: Set:

ioaiu_top.tachl: useOwnerTransfer = 1;
ioaiu_top.tachl: useSharerPromotion = 1;

Commands for setting up workspace to use a feature branch:

  • STEP 1: setup_concerto

  • STEP 2: source SOURCEME ncore3.4

  • STEP 3: cd $WORK_TOP

  • STEP 4: git config --add remote.origin.fetch +refs/heads/Ncore3.4/credit_skid_buffer:refs/remotes/origin/Ncore3.4/credit_skid_buffer

  • STEP 5: git fetch

  • STEP 6: git checkout Ncore3.4/credit_skid_buffer

Note: The example above is shown for Ncore3.4/credit_skid_buffer branch. The steps are exactly same for the other feature branches as well. Just change the names.

Steps for building the local maestro image:

These branches would require you to build local maestro image as the RTL tachl files will be updated in the feature branch. Use the regular refresh_rtl flow to build local maestro image.

  • STEP 1: cd $WORK_TOP

  • STEP 2: $WORK_TOP/dv/scripts/refresh_rtl

  • STEP 3: source $WORK_TOP/SOURCEME_image or $WORSOURCEME_image.tcsh (depending on whether you are using bash or csh)

Push files to feature branch:

Files can be pushed to feature branch by doing git add <file names>; git commit; git push.

Note: When you do git push it will automatically run the check-in tests for all blocks and FSYS. If check-in tests passes, your files will be pushed to the repo. If it fails, the push will be aborted. Need to debug the failures.
