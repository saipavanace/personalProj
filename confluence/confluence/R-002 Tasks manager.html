Feature description

The main aim of this feature is to help user go through the flow. During project creation user goes throw different stages in state machine. For different states there are different tasks will be displayed in the Task Manager. As soon as user completed all tasks he will see the green checkmark. 

Use cases

UC1. User has uncompleted tasks and can see unmarked tasks in the panel. 

UC2. User can see completed tasks in the panel. 

UC3. User can open Help console by clicking the task on the panel. 

UC4. User can open Issues console by clicking on the issues task on the panel. 

Requirements

R-002-001. By default the Task manager window is hidden. 

R-002-002. User can open/close Task manager from Preferences menu. 

R-002-003. For different states there are different tasks. The system should validate the current state of state machine and show only suitable tasks. 

R-002-004. User can click on the task and see description of needed changes in Help console. 

R-002-005. After every user's action tasks are updated. 

R-002-006. As soon as user has completed all needed actions for the particular task, the green checkmark should appear near it. 

R-002-007. The first version of available tasks for different states are displayed in the table 1 R-002-007 Tasks description.
