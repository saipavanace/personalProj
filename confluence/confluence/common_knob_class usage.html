Common_knob_class provide command line arguments to be collected from sim args and can be accessed directly into testbench environment. To provide access the value from command line arg one should create instance of knobs using common_knob_class in testbench environment. 

E.g.  k_num_requests in sequence class using sim args.

const int             m_weights_for_k_num_requests[3]   = {10, 85, 5};

const t_minmax_range  m_minmax_for_k_num_requests[3]    = {{100, 200}, {1000, 2000}, {10000,20000}};

common_knob_class k_num_requests = new ("k_num_requests", this, m_weights_for_k_num_requests, m_minmax_for_k_num_requests);

k_num_requests is instance created which can return values from randomization with different weights assigned array, here it is 10% with range from 100 - 200 and 85% of values return within range of 1000 - 2000 and 5% values return from 10000 - 20000.

Earlier these knobs are defined as var (int) inside axi agent config class hence needed to provide values using hierarchical paths from base test or env class. But now we won't require to do so instead if particular values to be provide these class based on test configuration one should use set_value method resides inside common_knob_class. 

e.g.  In base test or arg class:

if ((arg_value == "dii_arready_backpressure_test1") || (arg_value == "dii_arready_backpressure_test2")) begin

    m_env_cfg.m_axi_slave_agent_cfg.k_ace_slave_read_addr_chnl_delay_min.set_value(50);

In existing test-bench, if one used to provide values to these knobs in axi agent config class using hierarchical path from base test or arg class, now it should be removed for above knobs. If values to be set particularly then please do set_value as shown here in example code else it will use default value.
