conn/Current problems:

  1. Too many tests in IOAIU nightly regression.

  2. There are many missing config/test pairs (test that should be running on certain configs but are not.)

    1. The regr and regr_cov labels are not fully updated.

  3. Some features do not have feature labels to assist users in finding or running all tests for a particular feature.

  4. Some base tests are different from other related tests for no good reason, i.e. number of transactions.

  5. Testlist size of ~10,000 lines is too large. Making it hard to read or keep up-to-date.

Solution elements:

Note: some of this work can be done by one person working on the entire testlist, but other parts need to involve the expertise of the feature owner.

  1. Each feature needs at least one feature label and all tests for that feature must be labeled with it.

  2. Related tests should in most test be converted into flavors. This will simplify and shorten the testlist.

  3. Once flavors are used where possible, the testlist should be analyzed and optimized per feature.

    1. Examine where random tests can replace a set of directed tests. If a random test does not exist, note the need below.

    2. Examine the feature regr, regr_cov, and short_regr labels. Does each label need to be added or removed for each test.

    3. Once the list of code and functional coverage configs has been finalized, make sure that regr_cov has the right tests for each config.

    4. If a particular feature has too many test iterations, work with the feature owner to reduce this.
