Notes
  1. We started our discussion with Ncore 3.0 requirements for power aware. Sanjay send out the requirements via email as follows,

    1. The following goals are expected for power management in Ncore 3.0 (only). (The goals should be expected to be expanded further for 3.1 and beyond.)

      1. All of Ncore domain will be synchronous with single clock. This means that all of the Ncore units and Legato fabric will run on a common clock.
      2. The Ncore units will not embody clock crossings within them. Any frequency accommodation will be beyond their own functional interfaces (AXI, ACE, CHI, APB, etc.)
      3. AIUs will need to talk to asynchronous FIFOs that will allow processor clusters to run at independent frequencies, which could change independently and dynamically via full DVFS. These AIU interfaces must be equipped with appropriate signaling with the Ncore platform to allow the system to dynamically power down the cores at will.
      4. The Ncore configuration network will communicate with the Ncore units via APB interfaces. The network itself will be based on Presto and since it is not expected to be performance-critical, would likely run at a significantly different (lower) frequency. This will require the insertion of an asynchronous boundary either on the APB bus or just prior to the APB bus in the Presto network.
