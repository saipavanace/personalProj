1hrPerformance modelingManadher Kharroubi (Deactivated)

Project Angelo

  • aim for cycle accurate modeling; for modules that cannot achieve that, we will do approximation
  • composition and scheduling of combinatorial logic
  • directed acyclic graph
  • divide modules to processes
  • calculate processes until you hit a flip flop
  • every flip flop has to evaluate all its inputs
  • push method: calculate from the ponit with no dependency, schedule the functions that are dependent, and then schedule the next level
    • event driven
    • SW is responsible for the scheduling
  • pull method: start by looking at the end point, and calculate backwards on its dependencies
    • recursive function call
    • C++ itself handles the scheduling
    • assume there is no combinatorial loop (including the outside world)
  • how about control flip flops modeling? (e.g. inside arbiter) should be handled by this mechanism
  • will not derive the dependency graph from TACHL (it will be slow)
  • need to solve asynchronous clock domains problems
    • can have a global clock tracker
  • FIFO can be generalized as a flip flop
  • how about connection to outside? need a virtual flip flop at the edge of the model (if there is none in the configuration)
  • Decision is to go full steam on this approach; need verification on speed
