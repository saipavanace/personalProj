Purpose:

Store a histogram on the number of instances a particular value of a parameter, or a particular set of a values for a cross of parameters was hit. Used to determine if all desired ranges were hit.

Syntax

"parameterCoverage" : 
 {"indexRange" : [],
  "valueRange": [], 
  "crossValueWith": [],
  "crossIndexWith": [], 
  "whiteList": boolean, 
  "blackList": boolean}

In order for coverage to be done on a parameter,

"parameterCoverage" : {}

Must be specified in the CPR for that parameter.

"noCoverage" : boolean

When true indicates to not collect coverage info on parameter. Undefined is the same as false. Used for CPR checking and ignore during parameter coverage processing.

"noEntCoverage": boolean

When true indicates to not collect coverage info on the entry values in a parameter that is an array. Undefined is the same as false. Used in both CPR checking and during parameter processing

"skipObjCoverage": boolean

Only meaningful on type objects or arrayType objects. When true indicates to collect no coverage info on the referenced object.

indexRange (optional)

The indexRange is used to specify the number of entries in an array to cover and will always be non negative integers. 0 is a valid entry indicating an empty array. It has no meaning and is ignored for non-arrays.

If the index range is not specified it will use the minimum and maximum values specified in the CPR.

valueRange (optional)

The valueRange is an enumerated list of the values you want covered. When specified in the array that is not objects, this is range you want each entry in the array to cover, but for reporting purposes the entries are lumped into one pool. It has no meaning for an array of objects and is ignored.

If the value range is not specified it will use the values specified in the constraints parameters in the CPR.

Range Shortcuts

The following shortcuts can be used in indexRange and valueRange

I to J:

Used to specify a set of integers that increment by 1 from I to J.

I po2 J:

Used to specify power of 2s that exist between I and J, inclusive of 0.

0 po2 12 → 0,1,2,4,8

1 po2 64 → 1,2,4,8,16,32,64

I spo2 J:

0 spo2 12 → 0,1,2,3,4,5,7,8,9

1 spo2 65 → 1,2,3,4,5,7,8,9,15,16,17,31,32,33,63,64,65

17 spo2 63 → 17,31,32,33,63

Used to specify surrounded power of 2s that exist between I and J, inclusive of 0.

crossValueWith and crossIndexWith (optional)

If you don't specify a cross*With, no crossing will be done.

When you are crossing and array you have to indicate which of the two things in an array you are going to cross the other guy with, values of the array or number of indexes into an array.

The cross*With refers to other variables in the object that you want to cross with. The variable you cross with is given as a hierarchical name separated by "." for each level in the hierarchy. If you are crossing with an array, you have to indicate if you are crossing with the value, _v_, or the number of indexes, _i_. If you are and array, you must indicate which thing,

You cannot cross to values inside an object that is an element of an array.

You cannot cross unless you are have an indexRange or coverRange specified.

You can cross to as many variables as you like.

whiteList:

When true indicates the parameter can be randomized in the TCL DV flow. When false cannot be randomized. undefined is equivalent to false.

blackList:

When true indicates the parameter cannot be randomized in the TCL DV flow. When false can be randomized based on the definition of whiteList. undefined is equivalent to false.
