I/O:

output credit; 
input credit_rtrn; 
if ((params.maxTime > 0) && (params.minPkts > 1)) {output credit_cnt;}

Description

This is the interface that passes credits back an forth in the sym_rate_adapter when the mode is slow2fast.

Description of operation

When credit and credit_rtrn are the same value, this indicates that the slave has not credits. When the Master flips the value of credit (from 0->1 or 1->0), this tells the slave that it has received credits. The amount of credit is a known fixed value between the slave and the master. If optional signal credit_cnt exists, this indicates that the value can be either the known value (credit_cnt = 0) or the just 1 credit (credit_cnt = 1).

When the slave flips credit_rtrn to match credit, that indicates the given credit is being returned.
