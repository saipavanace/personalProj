Introduction

To determine what parameter values are set in a design can be difficult if the parameter is hidden or non-settable. Also, users would like to see what parameters were set in the design for documentation purposes. Maestro does not have a convenient command to report these parameters. The current method is to write out a mpf (Maestro project file) and decrypt it to view the design. This is inconvenient and not user friendly.

This document is a proposal to allow Maestro to query ADM (Arteris Data Model) data such as parameters, references, and child objects. Some of the data may not be visible (hidden) to the user. However, with the appropriate license, the parameters shall be accessible to view. With this capability, debugging a design can be done interactively from the console.

Intended implementation version: Ncore 3.6 and later.

Current Usage

Currently, several tcl commands are being used for this purpose: get_parameter, query_object, report_object, generate_output_param_report, generate_input_param_report, get_objects, get_connected.

Tcl Command

Current Testcase Usage

get_parameter

IMPLEMENTED in 3.6

Frequently used. Returns visible parameters or references and their values. Hidden parameters or references are returned with -hidden for holders of a development license. -details provides constraint information.

query_object

Seldom used. It can be replaced by other commands when this project is completed.

report_object

Seldom used. It reports parameters and other data on certain, but not all, objects. Hidden parameters are not reported. Attempts have been made to give specialize reports based on object type.

get_objects

Heavily used. It returns child objects and other objects (references) indirectly related to the given object.

generate_output_param_report

Seldom used internally for comparing design versions.

generate_input_param_report

Seldom used internally for comparing design versions.

get_connected

Used as a relative of get_objects with the specific goal of mining topology connectivity.

 

Maestro has no means to query for child object types available for a given object to explore and debug a design.

 

 

Proposed Usage

To show the hidden data, the design must have “Arteris/Dev” license enabled. The code can query for this license using isDebugLicenseAvailable() API.

Since query_object is seldom used, this command shall be deprecated and removed. The current usage of this command shall be provided by other commands. Some of the current usages are:

  • getting the clock of a ncoreunit,

  • determine the type of network,

  • check if an object exists without erroring out.

Replacement functionality will be implemented with new tcl commands below.

report

The report_object command shall be replaced by the report command.

The new report command shall be implemented to show all parameters, references, and child objects under different command options.

report $obj shall return a report of the object. The report shall contain all user settable/viewable parameters and references of the object in a user readable format.

If the -hidden option is added to the command and the design has a debug license, all parameters (including hidden ones) shall be reported. If the design does not have a debug license, this option is ignored.

If the -file $filename is added to the command, the output shall go into the file.

get_objects

get_objects command was implemented when the database APIs were not as richly developed. Many of the commands have specialized code to retrieve the requested data. Now, Maestro has a richer set of database APIs; the retrieval can be done generically. Therefore, code clean-up shall be done.

This command shall return a list of child objects base on the -type $child_type option. In the current ADM API, each object has a list of user “gettable” (visible and editable) child objects which this command shall use to determine the gettable child objects. If the child_type is a hidden child object type and if the design does not have the debug license, the command shall return an empty list and an error message shall be displayed. If the child_type is a hidden child object type and if the design does have the debug license, the command shall return the list of hidden objects.

In the current implementation, the command sometimes returns direct children, and sometimes performs a recursive descent. The implementation is not fully consistent. It is recommended to add a -recursive argument-less flag to handle recursive descent and eliminate the ambiguity.

For some applications, the command performs a dedicated traversal, e.g., asking for the ClockSubDomain of a parent object may return the clock of that object - rather than a hierarchical descendant. This usage overlaps with the ability to get references via get_parameter and should be slowly deprecated in the interest of consistency and predictability.

Additional Tcl Commands

Some additional tcl commands shall be created to replace the query_object functionality.

get_clock_subdomain

get_clock_subdomain $object shall return the clock subdomain that is attached to the object. If no clock subdomain is attached, the command shall return an empty name. If the object does not support a clock subdomain, an error message shall be displayed.

get_type

get_type $object shall return the type of object. This command shall work on hidden objects too. If the object does not exist, the command shall return an empty type and no error message shall be displayed.

If the -child_object_type is added to the command, a tcl list of user visible object types shall be returned. The object type may then be used in the get_objects command to query the child objects.

If the -hidden option (applicable with -child_object_type option) and debug license is available, all child object types except user visible types shall be returned; otherwise, the command shall return an empty list.

get_network_type

get_network_type $network shall return type of network. Valid return values are: nondata, data, request, response, unknown. An error message shall be displayed if the object is not a network and the return value will be an empty name.

get_network_type $network -data | -nondata | -request | -response shall return “true” if the network matches the option, otherwise returns “false”. Only one option is allowed. An error message shall be displayed if the object is not a network. (Check with David’s tepam files for ‘data’, ‘nondata’, etc. Should match GUI.)

get_parameter

IMPLEMENTED in 3.6

get_parameter -object $object , without options, returns a tcl list of name-value pairs of the user visible and settable parameters and references.

If the -hidden option is added to the command, all parameters and references (including hidden ones) become accessible. If the user does not have a debug license, this option is not available.

If the -name $name option is added to the command, the value of the given parameter or reference shall be returned. If the parameter is hidden, -hidden must be specified as well; if not, the operation results in an error, but if the user has access to a debug license, a reminder will be printed to retry with -hidden.

If the -details option is added to the command, the command will also show additional available information, such as the min, max, enum and default values of the parameter, if any, and its visibility flags.

The legacy command query_attribute is allowed as a synonym, but prints a Deprecated: message reminding the user to use the new command name.

set_parameter

MAES-6002 - Getting issue details... STATUS

set_parameter $object -name $name -value $value shall the name of the parameter with the value on the object. Hidden and non-visible parameters are settable using this command, only subject to access to the development license, and after printing a Warning or Deprecated. Write-protected parameters are never settable with this command or any other command.

Some parameters requires an array of values, which shall also be accepted by the -value option.

set_parameter $object -param_list [list name1 val1 name2 val2 …] shall set a list of name and value pairs of parameters to the object. If -name or -value options are present, the command shall error.

generate_param_report

generate_output_param_report and generate_input_param_report shall be replaced by the generate_param_report command.

generate_param_report command shall report the parameters used in the user in the design. The output shall begin at the project level and recursively report the child objects. The report shall be in the form of a table for each object.

If the -file $filename is added to the command, the output shall go into the file.

The -format option may be included. The valid formats are html and text.

If the -all is added to the command, it shall report all visible and settable parameters and references.

If the -hidden is added to the command and the debug license is available, it shall report all (including non-visible) parameters and references. If debug license is not available, only the visible and settable ones are reported.

 

Additional Commands to be Reviewed

  • create_interface looks like it was added without peer review and may be redundant.

  • report_clock should be reviewed for consistency with report_object.

  • get_cells was also added with custom terminology and poorly specified use cases. It may be redundant.

  • get_context and set_context are probably ready for removal or rescoping.

  • query_custom_attribute may be renamed to get_custom_attribute for consistency.

  • get_connected is a command for topology traversal with inconsistent implementation that we probably want to make more systematic.

Additional Infrastructure Changes

Currently a hard-coded table of user object type name to Adm class is in “TclTypeName.cc”, which is used by create_objects and get_objects commands to determine which type of objects to create or retrieve. Ideally, this table should be incorporated into the type file of each object and removed from “TclTypeName.cc”. This project shall add object type name into the type file of each object. MAES-6097 has been filed for this enhancement.
