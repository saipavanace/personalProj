Motivation

Currently, the amount of design validation functions is relatively small and they are distributed by different components (ADM, Ddg)

libchecker component may provide the uniform validation functions API from different components.

Existence implementation contains

  • Checker - generic checker interface
  • Checker Manager - The storage class for checkers
  • Deadlock checker implementation

The idea is to use the checker interface for all design validation functionality and use Checker Manager as an entry point to use it.

Cyclic-dependency issue

Existence implementation creates checkers (actually only one DeadLockChecker) right into its code with checkerFactory functions.

In the case of ADM debugObject checker, it creates a dependency between libchecker and libdata. (libchecker knows about libdata).

If Adm states will use libchecker API the backward dependency will created (libdata will know about libchecker)

Implementation road-map

  1. Clean-up existence libchecker component code from unused files and functions
  2. Implement checker registering API to avoid cyclic dependencies
  3. Implement ADM debug object checker
  4. Update ADM states code to use libchecker API
  5. Implement other checkers (like Ddg)
