Bug Fixes

CONC-10000: CONC-10000

The issue is that two Posted Writes could possibly leave the DII not in order if the IOAIU is set to Write Order.

The DII added a dependency for this ordering mode where all Writes from an Initiator will be put in order.

CONC-9936: CONC-9936

This effects partial cache line WRAP Write and Stash transactions from the IOAIU to the DMI/CHI
Without this fix DWID is incorrect and the data might be corrupted to the CHI or DMI based on address alignment

For IOAIU configurations with 64-bit or 128-bit wide databases, on multi-beat wrap bursts which are not address aligned to the cacheline size, the IOAIU can incorrectly organize the data-beats going to the DMI on writes or CHIAIU on stashes, leading to data corruption.

CONC-9798: CONC-9798

Erratum: A correctable tag ecc error can result in a transaction being sent to the smi with incorrect memory attributes.

CONC-9669: CONC-9669

A deadlock could occur when transactions are constantly being woken up and not given the chance to advance because they have no credits.

This stream of woken up transactions can block new transactions which would release credits.

The fix is that a transaction must have credits to be eligible to be woken up.

CONC-9790: CONC-9790

Erratum: A correctable tag ecc error can result in a transaction looking up the cache multiple times for a single access.

CONC-9758: CONC-9758

CHI-AIU is monitoring the Transaction ID on the CHI RX RSP channel and CHI RX DAT channel to match the Transaction ID that CHI-AIU sent on CHI TX SNP channel for the stashing snoop request. The RTL bug is that CHI-AIU does not decode the Opcode on the CHI RX RSP channel and CHI RX DAT channel to make sure itâ€™s a snoop response packet. As a result, CHI-AIU wound send a data packet with wrong Transaction ID on the CHI TX DAT channel for the stashing snoop request.

CONC-9944: CONC-9944

The DII packed data incorrectly for single beat narrow transactions on the DII native interface.

This bug resulted from the DII AxSize fix bug CONC-9871

CONC-9871: CONC-9871

This bug effects all non-modifiable single beat narrow transactions to the DII from IOAIU
Without this fix all single beat narrow transactions may be modified at the DII to be non narrow.
With this fix the DII will keep nonmodifiable single beat narrow transactions as narrow.

CONC-9736: CONC-9736

Erratum: uncorrectable errors (parity or ecc) in the concerto mux on inbound smi transactions can cause unpredictable behavior in the ioaiu resulting in hangs, incorrect cache behavior, or incorrect interface behavior. The uncorrectable error is properly forwarded to the fault controller.

CONC-9674: CONC-9674

Errata fixed since 3.2.0
Inside the IOAIU the address dependency chains were removed for normal transactions in non proxy cache configurations.

This was needed so that Reads could not block writes inside the IOAIU under no condition.

In addition a DII fix needed to be added where all OR==0 transactions needed to be write ordered.
