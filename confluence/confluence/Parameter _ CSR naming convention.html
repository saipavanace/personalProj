Parameter Data types 

  • Basic data types:
    • Integer (Int)
    • Boolean (Bool)
    • Enumerated (Enum)
    • String (Str)
  • Compound data types: 
    • Vector (Vec [])
      • A vector consists of a list of elements of a single type, either a basic data type or a bundle data type, indexed by an integer.
    • Bundle (Bnd {})
      •  A bundle consists of a grouping of any number of elements of any type. 

Type Prefix

  • “n” – Integer number (typically a count of items)
  • “m” – Integer number (typically a bit mask)
  • “w” – Integer number of bits
  • “sz” – Integer number of bytes
  • “id” – Integer identifier or integer index into a vector
  • “is” – Boolean that describes a property
  • “can” – Boolean that describes a capability
  • “use” – Boolean that enables functionality
  • “fn” – Enumerated that selects functionality
  • “str” – String 

Product Prefix

  • Ncore (NCR)
  • Concerto (CON)
  • CodaCache (CCA)
  • Symphony (SYM)
  • Library (LIB)
  • CommonCachePipe (CCP)
  • ......

It will be great if we can grandfather older Ncore parameters and follow the new convention for new parameters 

 

The other option proposed here is to use hierarchy to identify parameters. 

Notes: 

  1. A parameter shown in plain text represents a compound data type.  “ParameterName[]” indicates a vector type of basic types, “ParameterName{}” indicates a bundle type, and “ParameterName[]{}” indicates a vector of bundles.

  2. parameter name starts with all caps product name prefix followed by lowercase type prefix or name 

Examples: 

  • NCRnSnoopFilters (integer type parameter)
  • NCRagentAiuInfo[]{} (Vector of bundles)
