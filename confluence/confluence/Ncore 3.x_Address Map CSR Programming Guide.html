The Ncore3 Address Map in all AIUs and DCEs must be programmed correctly, so as to ensure all transactions that flow through AIUs and DCEs, including DCE-initiated Recall transaction, for the same address, are seeing the same address region, for data consistency. If programmed incorrectly, then data could be written to the wrong location, or read from the wrong location.

All AIUs and DCEs to be configured identically for the Ncore3 Address Map CSR registers.

Registers common in AIUs and DCEs:

GPRAR<i>, GPRBAR<i> registers where i = 0 . .(nGPRA-1)

BRAR register

AMIGR register

AMIFR register

Registers that only exist in AIUs:

NRSBAR register

The value of nGPRA should be the same for all AIUs and DCEs.

The value of GPRAR<i>, GPRBAR<i>, BRAR, AMIGR, AMIFR should be the same for all AIUs and DCEs for the following bit fields:

GPRAR<i>.Valid

GPRAR<i>.HUT

GPRAR<i>.Size

GPRAR<i>.HomeUnitId

GPRAR<i>.NC

GPRAR<i>,NSX

GPRBAR<i>.AddrLo - must be GPRAR<i>.Size aligned (e.g. if Size is 1MB, {AddrLo,12’b0} must be 1MB aligned)

GPRBAR<i>.AddrHi - must be GPRAR<i>.Size aligned (e.g. if Size is 1MB, {AddrHi,12’b0} must be 1MB aligned)

BRAR.Valid

BRAR.Size

BRAR.HUT

BRAR.HomeUnitId

BRAR.AddrLo - must be BRAR.Size aligned (e.g. if Size is 1MB, {AddrLo,12’b0} must be 1MB aligned)

BRAR.AddrHi - must be BRAR.Size aligned (e.g. if Size is 1MB, {AddrHi,12’b0} must be 1MB aligned)

AMIGR.AMIGS

AMIGR.Valid

AMIFR.A2WIFId

AMIFR.A3WIFId

AMIFR.A4WIFId

AMIFR.A8WIFId

AMIFR.A16WIFId

The value of NRSBAR should be the same for all AIUs.

NRSBAR.BaseAddr

The GPR regions specified in each {GPRAR<i>.AddrLo, GPRAR<i>.AddrHi} && (GPRAR<i>.Valid==1) must be each unique, i.e. each GPR region cannot overlap with any other GPR regions.

NOTES:

(1)

The Ncore3 Address Map module computes the DMI NUnitId (Node Unit ID) by using this formula:

DMI NUnitId = dmiIGS[ HomeUnitId ][ dmi_id_offset ]

where

HomeUnitId is from one of the address-region-hit GPRAR<i>/BRAR<i> HomeUnitId field

dmiIGS = AMIGR.Valid ? dmiIGSV[0] : dmiIGSV [ AMIGR_AMIGS ];

dmiIGSV is the JSON parameter DMI Interleave Group Set Vector

The dmiIGSV is a vector (i.e. array) of Interleave Group Set (IGS).
Each Interleave Group Set contains one or more Interleave Groups (IG).
Each Interleave Group is a vector of DMI Node Unit IDs.
Example:

[ {IGV : [ {DMIIDV:[0]}, {DMIIDV:[1,2]}, {DMIIDV:[3,4,5]}, {DMIIDV:[6,7,8,9]} ] },
{IGV : [ {DMIIDV:[0,1,2]}, {DMIIDV:[3,4,5,6]}, {DMIIDV:[7]}, {DMIIDV:[8,9]} ] }
]

dmi_id_offset is selected from one of the following values:

dmi_id_offset_2WIF = hashing(address, dmi2WIFV [ AMIFR_A2WIFId ] );

dmi_id_offset_3WIF = hashing(address, dmi3WIFV [ AMIFR_A3WIFId ] );

dmi_id_offset_4WIF = hashing(address, dmi4WIFV [ AMIFR_A4WIFId ] );

dmi_id_offset_8WIF = hashing(address, dmi8WIFV [ AMIFR_A8WIFId ] );

dmi_id_offset_16WIF = hashing(address, dmi16WIFV [ AMIFR_A16WIFId ] );

The select for the dmi_id_offset is from one of the address-region-hit GPRAR<i>/BRAR<i> SizeOfIG

The dmi2WIFV is a vector (i.e. array) of 2-Way Interleave Functions.
Each Interleave Function comprises an array of integer for primary bits, and an array of string for secondary bits.
Example:
[ {PrimaryBits : [7], SecondaryBits: ["'h10000000"]},
{PrimaryBits : [9], SecondaryBits: ["'h40000000"]}
]

Given

dmi_id_offset_2WIF = hashing(address, dmi2WIFV [ AMIFR_A2WIFId ] );

The hashing function is given an address, and an element {PrimaryBits, SecondaryBits} selected by the AMIFR_A2WIFId.

For example:

if AMIFR_A2WIFId = 0, then dmi2WIFV[0] = {PrimaryBits : [7], SecondaryBits: ["'h10000000"]}, therefore

dmi_id_offset = hashing(address, dmi2WIFV[0])

=> dmi_id_offset = hashing(address, {PrimaryBits : [7], SecondaryBits: ["'h10000000"]} )

=> dmi_id_offset = address[7] XOR address[28]


The dmi3WIFV is a vector (i.e. array) of 3-Way Interleave Functions.
Each Interleave Function comprises an array of integer for primary bits, and an array of string for secondary bits.
Example:

[ {PrimaryBits : [7, 8], SecondaryBits: ["'h10000000", "'h20000000"]},
{PrimaryBits : [9, 10], SecondaryBits: ["'h40000000", "'h80000000"]}
]


The dmi4WIFV is a vector (i.e. array) of 4-Way Interleave Functions.
Each Interleave Function comprises an array of integer for primary bits, and an array of string for secondary bits.
Example:
[ {PrimaryBits : [7, 8], SecondaryBits: ["'h10000000", "'h20000000"]},
{PrimaryBits : [9, 10], SecondaryBits: ["'h40000000", "'h80000000"]}
]

Given

dmi_id_offset_4WIF = hashing(address, dmi4WIFV [ AMIFR_A4WIFId ] );

The hashing function is given an address, and an element {PrimaryBits, SecondaryBits} selected by the AMIFR_A4WIFId.

For example:

if AMIFR_A4WIFId = 0,

then dmi4WIFV[0] = {PrimaryBits : [7, 8], SecondaryBits: ["'h10000000", "'h20000000"]}, therefore

dmi_id_offset = hashing(address, dmi4WIFV[0])

=> dmi_id_offset = hashing(address, {PrimaryBits : [7, 8], SecondaryBits: ["'h10000000", "'h20000000"]} )

=> dmi_id_offset = { (address[7] XOR address[28]), (address[8] XOR address[29]) };


The dmi8WIFV is a vector (i.e. array) of 8-Way Interleave Functions.
Each Interleave Function comprises an array of integer for primary bits, and an array of string for secondary bits.
Example:
[ {PrimaryBits : [7, 8, 9], SecondaryBits: ["'h10000000", "'h20000000", "'h40000000"]},
{PrimaryBits : [9, 10, 11], SecondaryBits: ["'h40000000", "'h80000000", "'h20000000"]}
]


The dmi16WIFV is a vector (i.e. array) of 16-Way Interleave Functions.
Each Interleave Function comprises an array of integer for primary bits, and an array of string for secondary bits.
Example:

[ {PrimaryBits : [6, 7, 8, 9], SecondaryBits: ["'h10000000", "'h20000000", "'h40000000", "'h04000000"]},
{PrimaryBits : [9, 10, 11, 12], SecondaryBits: ["'h40000000", "'h80000000", "'h20000000", "'h08000000"]}
]
