Usage:

IrunCoverCheck.js a helper script to run formal cover check tool to generate exclusions using the code coverage output of runsim. This tool needs a user to first execute runsim with code coverage enabled.


  Usage: runCovercheck.js [options]                        // Option -rd (Input Regression Directory) is required for the tool to find the ucdb files

  Options:

    -rd <string>     : Input Regression Directory  - Specify the regression directory from runsim's code coverage run upto the timestamped folder

   -od <string>     : Output Dump Directory        - Specify the output directory to dump the generated files

    -c <string>      : Config Name                 - Specify the config name to run in a single mode (Default runs all the configs present in the regression folder)

    -t <string>      : Timeout (in minutes)        - Specify the timeout in minutes to run the covercheck tool for (Default 30 mins)

    -h               : Help                        - Display this help page

Example command:

node hw-scripts/runCovercheck.js -rd /scratch/kavish/22OctCovercheck/hw-sym/dv/regression/2020_10_22_1031 -od /scratch/kavish/coverCheckOut -t 30


  • Mandatory - The script needs an input regression directory (-rd) pointed upto the timestamped folder so it knows which runsim regression runs code coverage needs to be run cover check on. This directory is generated after you execute runsim with enabling code coverage using -exe option and enabling 10th bit in it (generally -exe 5f7 is the code you want to run)
    • Example : -rd /scratch/kavish/22OctCovercheck/hw-sym/dv/regression/2020_10_22_1031
  • Mandatory - The script needs an output directory (-od) to dump the results in.
    • Example:  1031 -od /scratch/kavish/coverCheckOut
  • Optional – User can provide the timeout of covercheck run with -t option in minutes. Default is 30 mins
  • Optional – User can provide a specific config name in a regression on whom they want to run the covercheck tool on, this is a single config run mode, by default the script runs the tool on all configs present in input regression directory

Output Directory:

  • In the output directory provided to the script -od, there will be 2 depths of folders created as env_name/config_name
    • Example:
