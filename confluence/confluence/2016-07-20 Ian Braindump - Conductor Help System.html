Discussion items

  • UserParameters.xls (in schema) is used to dive the help system.  The 'prefix' column is used to determine the path into the help system.
  • A script (populate_descriptions.js) in the schema repo is used to convert the excel file into parameterDescription.json,   The script is called automatically as part of npm pre-publish.  This script also provides a .h file that is given to techPubs, where Robohelp uses it to identify the various sections.  This needs to be manually regenerated each time.
  • In the 'help' repo, There is a Multiscreen_HTML5 directory that Tech pubs provides, which contains all of the html files for the help system.  The new directory from techpubs just replaces the existing one each time. 
  • Adding PDF files
  • In the help repo, there's a 'PDFs' directory which contains all the pdf documents.
  • Remember to bump the version of help with each change.
