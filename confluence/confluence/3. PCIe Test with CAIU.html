Experimental Environment:

  • Only coherent traffic
  • Address increased by cache line, no overlap between NCAIUs
  • CAIU is enabled. Read 512 transaction using RU before activating other initiators to make snoop traffic from NCAIUs.
    • Snoop is targeting 25%. Selected addresses 25% at cache warm up phase.
  • Disabled starvation by assigning 0 at the starvation register
  • Injected x2 traffics from each initiator
  • Address increased by cache line, no overlap between NCAIUs
  • All traffic is targeting DMI
  • SRAM model integrated with DMI
    • 100 cycle latency at destination (77ns, 1.3GHz target frequency)
    • Read outstanding: 64, write outstanding: 32

1) Observation

  • BW is similar with 0ns SRAM model case.
  • Resource status: DCE MRD credit is not bottleneck anymore
  • CAIU to DCE credits are highly consumed, but cannot say it is the bottleneck of the system.
  • For the higher QOS value shows better latency numbers, however, 
    • The traffic with higher qos value shows 8% of the latency improvement, which is less than 0ns case.
    • The proposition is that the latency improvement by QoS becomes low because system itself is not limited by local credit.
  • Why MRD credit would not be the system bottleneck any more?
    • with 0 ns delay model, MRD turn-around time becomes excessive around 20 ns 
    • Therefore, MRD turn-around latency is extremely long in case of 0ns SRAM model, and it degrades the system performance by making MRD credit utilization so high.
