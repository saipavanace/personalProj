engVerId is 32bit integer value to track information about configuration and customer.

Code is implemented here
http://gitlab.arteris.com/maestro/maestro/-/blob/Ncore3.4/develop/contrib/libaccess/src/utilities/ConfigurationEncryption.cc

Current 32 bits allocation

19 bits are reserved for MPF hash (every time gen_collateral command is issued, copy mpf is saved. After that 128 bits MD5 hash is used to get hash and last 19 bits used for engVerId)

13 bits are reserved for CHIP_ID from a license file.

Example if CHIP_ID is 1001, engVerId looks like

xxxxxxxxxxxxxxxxxxx0001111101001, where x is a mpf hash.

implVerId is 12 bits integer to store Ncore version.

4 bits is used to store every digit.
