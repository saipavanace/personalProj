Discussion items

ItemNotes
System Constraints
  • Need to have a plan for system constraints, and working examples of simple constraints (e.g. max # of AIUs)
Top-level generation
  • Will do incremental calls to server, bottom-up generation of modules
  • Question: create intermediate and top level modules by writing TACHL to pass to the server, or JSON to drive something like John's generic wrapper?
  • Possible problem ... our networks are interlaced. For example, the control network can reach in and connect to elements in the request / reply network. How do we create modules that work with that and still honor things like user chosen module boundaries?
Stateful or stateless server?
  • A stateful server requires session management. We would either need to hold a connection open until all generation is done or make sure that subsequent calls go to the same server instance (or at least one with access to the same disk space). It could simplify the assembly of the generated modules if they are all stored on the server and only retrieved when the design is done.
  • A stateless server requires passing state information between the server and the client so that subsequent calls can do the right thing (e.g. the hashmap so that any given paramerization of a unit is only generated once). Doing incremental changes (e.g. 'ECO' mode) could be easier with a stateless server.
