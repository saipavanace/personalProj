Symphony has two Power Management Blocks, the PMA Slave (PMA_S) and PMA Master (PMA_M). The diagram below shows an example Symphony controlled by a Q Channel Interface.

PMA Slave (PMA_S)

The PMA Slave controls the power state of a single Symphony block. The PMA Slave takes in a single Q or P Channel Interface and interfaces with the block to set it to a sleep state, or wake it up out of a sleep state.

The PMA Slave can be configured in the following ways:

    1. Q or P Channel Interface
    2. A configurable timeout value where the PMA will deny a request after a certain amount of clock cycles.
    3. A configurable abort where the PMA will immediately deny any power-down request based on a specific Symphony block state.

PMA Master (PMA_M)

The PMA Master takes in multiple Q and P Channel interfaces from PMA Slaves and aggregates them into a single Q or P Channel Interface. The PMA Master will only accept a power-down request if all slaves accept the power-down request. If one slave denys the power down request the PMA Master will cancel the power-down request to each of the other slaves

The PMA Master can be configured in the following ways:

    1. Q or P Channel Interface on the Master side

PMU

The power management unit which sits outside the context of Symphony. This block controls the power state of each domain through a P or Q interface.
