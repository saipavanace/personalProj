What?

We need nCore model to do performance analysis for architecture exploration task to choose best configuration.

Need for Speed

System with cache coherency has significant warm-up time.

Behavior of system will depend on state of caches, directories, tables.

User may want to run seconds of real time before start collecting performance data.

nCore may require x100 more cycles compare to FlexNoC

FPGA emulation

In-premises FPGA emulation is not new. Expensive, hard to scale. 

Cloud FPGA emulation like: https://aws.amazon.com/ec2/instance-types/f1/ may be interesting option:

  • no FPGA ownership cost
  • unified HW architecture
  • cloud based FPGA synthesis.
  • scalable – user may start multiple instances in parallel for shorter time
  • neutral environment. Customer can run experiments in isolated environment without sharing project details with Arteris. Arteris can do the same.

Development/Maintenance cost

Fast nCore TLM model can potentially be x100 faster then Verilated model and 80% cycle accurate.

Development cost of fast nCore TLM model has to be compared with development of FlexNoC memory scheduler that proven to be time consuming.

Verilated / FPGA uses Verilog as the source for model.
