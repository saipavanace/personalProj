Development
  • Create a UVM component ('concerto_sys_checker')  similar to that of concerto_ref_wrapper.
  • This component subscribes to all agent & bridge AIU's native interface AXI Rd/Wr channels to monitor requests and responses
  • Matches requests to responses and calculates RD/WR latencies and bandwidth.
  • Latency is the number of cycles taken for first beat of Rd or Wr response observed on native interface start from first beat of request accepted by the agent.
  • Traffic is controlled by using knobs and check-in a fixed performance testlist. More details below
  • Add knob '+sys_perf_test' so that this component is not active by-default. Also this knob ensures that there are BFM no delays applied
 Tests 

We will have following set of performance tests. Are these sufficient??

  1. Read Latency Test
  2. Write Latency Test
  3. Write Back Latency Test
  4. Read Bandwidth Test
  5. Write Bandwidth Test

Rd Latency Test

  • Test drives single read Request per agent in sequential manner i.e meaning drive one request wait for response and then initiate new request on another agent. Test iterates over all the agents
  • For ACE Agents, Rd sequence initiates with either ReadClean or ReadNotSharedDirty or ReadShared transaction and wait for response data. Latency is the difference time divided by clock period
  • For ACE-Lite Agents & NCB agents,  Rd sequence initiates with ReadOnce transaction and wait for response data.
  • Test will make sure that there are no Address Collisions. All requests will end up fetching from memory

Wr Latency Test

  • Similar to that of Rd latency test expect that Wr sequence is initiated with WriteUnique transaction.

WrBack Test

  • For all Ace Agents, initiate a Read request and then issue a Write-Back transaction on Wr sequence. Latency is calculated from the point Write Back sequence is initiated on Wr channel and response received on Bresp channel.
  • Do we need to run WriteBack sequences on NCB agents??

Band-Width Test

  • For Rd/Wr bandwidth tests we drive specific number requests (determined by a knob) per agent and wait until all responses are received. Calculate the time taken  and translate it to number of bits per second.
  • We will re-use the above latency tests with knobs values changed.
  • No address collisions
