1 HourErrors and Interrupt verification priorities and stratergiesSantosh Ramu

Errors and interrupts.

All blocks below will be verified at native layer 4x4 for proof of functionality, full system verification will not be performed because combination of formal verification for connectivity and verification of proof of functionality will be sufficient.


Firewall: Poision bit in packet, target sends the data back to initiator and initiator will flag error, Firewall can optionally send interrupt.

Verification: Verify proof of concept with 1 Firewall at top level to see interrupt was generated.


Sym_multicast: Generates in only 1 case as described in the confluence page.  

Verification:  1. Verify if interrupt is generated as per description.

                        2. Verify EarlyResponseErrMultiple is working correctly.

                        3. EarlyResponseMsgSeqNum cannot be verified at Native layer.

                        4. Don’t verify optional registers.

Interrupt accumulator:

Verification: 1. Clearing interrupt registers inside interrupt generating blocks like PMON stats should clear registers inside interrupt accumulator block. But other way round will set the interrupts in interrupt block in next cycle.

                       2. Verify the interrupt masking.

                       3. Verify connectivity of internal interrupts to interrupt accumulator block formally.

VC_firewall_adapter:

Verification:   1. Formally verify VC firewall adapter to do connectivity checks.

                         2. Verify this block’s interrupt generation functionality as per description .



VC_Multicast:

Verification: 1. Formally verify VC multicast adapter to do connectivity checks.

                         2. Verify this block’s interrupt generation functionality as per description .


PMON_probe: Interrupt is customer defined, any event can create interrupts. Use packet count and count_event to fire interrupt.
PMON_stats: Same with stats, 1 interrupt based on count, reserve bit 0 for that.

Verification:

                    Generic design: Verify if interrupt is fired for packet count and set mask bit once interrupt is fired to not fire any more interrupts.

                    Stand alone testbench: Verify various flavors of PMON probe and stats interrupt generation.


ATUI_AXI:
Verification: 1. Verify internally generated errors,

                       2. For buffered writes interrupt generation due to errors needs to be verified.


ATUI_APB:

Verification: 1. Internal errors need to be verified.

                       2. No way to verify buffered write interrupt in APB so no need to verify.


ATUI_CTL:

Verificaiton: No need to verify as these errors are verified at ATUI_AXI.


ATUT_AXI, APB and CTL needs to be defined before we verify it.

Config network: Config network error will be marked as special slave error and it will be sent to the ATUI to raise the interrupt, this needs to be verified as configuration errors, 2 types of errors in config network NS and decode error and both trigger same kind of interrupt.

Highest priority: ATUI interrupts.
