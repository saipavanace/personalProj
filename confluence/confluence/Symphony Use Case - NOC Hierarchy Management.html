Name: NOC Hierarchy Management

User Roles: Angie_Architect

State: New

Phase: unspecified

Goals:

The design of an SOC often includes several NOCs.  NOCs may be reused from previous projects.  NOCs may be worked on by different teams.  NOCs may be imported from a separate FlexNoC run or a third party.  In order to enable this, Symphony will support a hierarchy of NOC collections.  Each collection will consist of one or more NOCs and zero or more other NOC collections.  The user will want to manipulate the hierarchy of NOCs to create logical collections for the purpose of editing separately, running P&R modularly, modular testbenches, etc.

Question:  How does this work along with virtual NOCs?  Can we completely ignore virtual NOCs wrt the NOC hierarchy management, or do we need (for example) a way to display the hierarchy restricted to a set of virtual NOCs?

One or more NOC definitions
