This feature helps test writes control random variables in sequence item of a VIP (ex :axi4,atp,smi) via +args.

It also supports controlling the variable constraint on per instance basis.

A new feature has been added to constraint a variable within a range using min and max +args.

Here are few examples being used in the current Symphony DV testlists.


Atp pkt_sid controlled on per instance basis

+atp_txn_i0_pkt_sid=0,+atp_txn_i1_pkt_sid=1,+atp_txn_i2_pkt_sid=2,+atp_txn_i3_pkt_sid=3,+atp_txn_i4_pkt_sid=4,+atp_txn_i5_pkt_sid=5


axi awaddr and araddr min and max range being set via +args in hex reprensentation.

+awaddr_min=h80000000,+araddr_min=h80000000,+awaddr_max=h8fffffff,+araddr_max=h8fffffff


To implement it in respective blocks here are the steps.

1.Extend the transaction classes from sym_txn_seq_item.sv (I added a variable called m_plusargs).

2.Wherever a variable is defined pass the variable as a string to an js object(this has list of all variables defined).

Eg

<% var h_attr =[];%>

Class atp_txn extends sym_txn_seq_item

int pkt_tid;

<% h_attr.push(“pkt_tid”)%>

int pkt_sid;

<% h_attr.push(“pkt_sid”)%>

3.Call the below js function after all the variable declaration.

<%=utilFunctionsDV.setConstraint(h_attr,"  ")%>

4.Call the below js function in pre_randomize function

Eg:

Function void pre_randomize();

<%=utilFunctionsDV.setPlusargs(h_attr,"  ")%>

Endfunction:


Now you can set a random variable to a given value or set of values via +args.

Eg

+pkt_tid=2  // sets pkt_tid=2 for all the transaction which has the field pkt_id

Or

+atp_txn_pkt_tid=2, +atp_txn_pkt_tid=3 //picks pkt_tid between 2,3 for type atp_txn

Or

+atp_txn_i0_pkt_tid=2, +atp_txn_i0_pkt_tid=3, +atp_txn_i1_pkt_tid=4 +atp_txn_i1_pkt_tid=5 //picks pkt_tid between 2,3 for instance atp_txn_i0( based on how the name of transaction is defined, here i0 stands for instance 0), picks pkt_tid between 4,5 for instance atp_txn_i1(inst 1),
