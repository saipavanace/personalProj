module_name

aw_fifo

JavaScript parameters

depth: Depth of pipe. Must be a positive power of 2.

N: Number of entries in the FIFO to expose starting from the earliest one. Must be a positive number. Non-powers of 2 also supported.

inWidth: Width of write data or wdata. Must be 1 or greater. This also matches the width of each entry of the internal symmetric width FIFO.

pipeBackWard: A boolean. When false, the FIFO will accept a write even when full if there is a read attempted at the same clock. This will however create a timing path between write and read control signals.

Derived parameters

outWidth: Width of the rdata bus. This is computed as outWidth = N x inWidth;

dMinWidth: Width of the dMin bus. This is computed as log2ceil ( depth + 1 );

Ports

input wr;                                                          // To be asserted when wdata is to be written into FIFO
input rd;                                                           // To be asserted when rdata is to be read from the FIFO
input [ dMinWidth-1 : 0 ] rd_n;                        // Specifies the number of entries to pop from the FIFO
input [ inWidth-1 : 0 ]          wdata;                 // Write-data

output full;                                                     // Status signal to indicate when the FIFO is full
output empty;                                                // Status signal to indicate when the FIFO is empty
output [ dMinWidth-1 : 0 ] dMin;                   // Status signal that tells how many valid entries exist in the FIFO, when empty = 0.
output [ outWidth-1 : 0 ]     rdata;                 // Read-data. Must be of width outWidth = N x inWidth.

Description

The aw_fifo or Asymmetric width FIFO is a synchronous FIFO that has different widths for write and read data. Only the case where inWidth < outWidth (Narrow-to-wide) is described here. The other cases are deprecated.

The ratio between the two widths is called N = outWidth / inWidth is always a positive integer. Only one entry can be written into the FIFO in a single clock, but multiple entries can be read in a single clock. This means the write-pointer always advances by at most one each clock, while the read-pointer can increment by an amount up to depth each clock.

The FIFO also permits variable read. This means that the number of entries that can be popped off each clock from the top of the FIFO is variable and is specified using the input rd_n and by asserting input rd high.
