JavaScript parameters

reqWidth: Width of data-bus connecting the gen_async_adapter_in and gen_async_adapter_out in the request path. Needs to be 2 or greater.

respWidth: Width of data-bus connecting the gen_async_adapter_in and gen_async_adapter_out in the response path. Needs to be 2 or greater.

reqDepth: Depth of FIFOs in async_adapters in the request path. Needs to be 2 or greater.

respDepth: Depth of FIFOs in async_adapters in the response path. Needs to be 2 or greater.

async: A boolean - true or false. To be set to true if the async adapters run on asynchronous clocks and false otherwise.

I/O

req_async_ sub-interface:

input [reqDepth-1:0] read_sel; 
output [reqWidth-1:0] data; 
output [(params.async ? Math.ceil(Math.log(params.depth)/Math.LN2)+1 : params.depth+1)-1:0] write_ptr, 
input [(params.async ? Math.ceil(Math.log(params.depth)/Math.LN2)+1 : params.depth+1)-1:0] read_ptr;

resp_async_ sub-interface:

output [respDepth-1:0] read_sel; 
input [respWidth-1:0] data; 
input [(params.async ? Math.ceil(Math.log(params.depth)/Math.LN2)+1 : params.depth+1)-1:0] write_ptr, 
output [(params.async ? Math.ceil(Math.log(params.depth)/Math.LN2)+1 : params.depth+1)-1:0] read_ptr;

pma_ sub-interface

output asleep;
input busy;

Description

The InterfaceCPA is used for communication and data-transfer between OBSOLETE-DELETE HW-CTF clk_pwr_adapter_in and HW-CTF clk_pwr_adapter_out. It has three sub-interfaces - req_async, resp_async_, and pma_.

The req_async_ and resp_async_ sub-interfaces correspond to InterfaceAsyncFIFO connecting the async-adapters (gen_async_adapter and gen_vc_async_adapter) on the request and response paths respectively.

The pma_ sub-interface corresponds to signals used to drive the pmaControlInterface of OBSOLETE-DELETE HW-CTF clk_pwr_adapter_in and HW-CTF clk_pwr_adapter_out).
