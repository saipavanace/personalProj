v1.0Added additional detailsRobert Podnar

Every memory instance, including memory instance for a bank

-has an entry in the memory generation object.

-has its module name and instance name specified by Maestro.

-has a corresponding generic interface for memory sideband signals if the memory instance is not of flop-based memory.

-has the parameters for the new memory wrapper (See NCore 4.0 Memory Wrapper) 

NOTE: Ncore 3.2 only uses single-port memory, either SRAM-based or flop-based.

NOTE: Ncore 3.2 does not use the new memory wrapper.

New memory interface object.

Every non-flop based memory has a corresponding memory interface instance for memory sideband signals. 

The memory interface is the generic interface as specified by hw-lib/cpr/interface/InterfaceGeneric.interface.cpr

The memory interface object has a field "name" that has a matching "rtlPrefixString" entry in the memory generation object, where the field "name" is the corresponding field "rtlPrefixString" appended with "_".
