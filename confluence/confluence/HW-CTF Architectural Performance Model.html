Introduction

The Architectural Performance Model (APM) is a bring up version of the eventual Performance model that the customers will use for Symphony. This model was developed with the intent of equipping the architects with an "approximately timed" software model that complements their understanding and highlights the impact of their architectural decisions on the system level performance. The model has several embedded parameters that can be tuned to achieve a cycle accurate representation of the design. The model is highly scalable and can currently handle any non-broadcast network topology. The scope of development of this model will be refined and reviewed as specified in the Scope section.

Model Attributes

Language

APM is developed using C++, in combination with TLM2.0 libraries. TLM2.0 libraries provide a skeletal frame work to implement event driven sockets on the initiators (drivers) and targets (memories). This has greatly helped in speeding up the bring up time of this model.

Components

The main components of the model, as it is currently defined, are listed below.

  • Driver (Initiator)
  • Switch
  • Buffer
  • Target (Memory/Endpoint)

Detailed descriptions on each of these components will be added subsequently.

Performance Metrics

The performance metrics reported by the model are defined by the architects. Currently, each component of the model generates a summarized report over each run. Each of the targets report the following metrics.

  • Individual FLIT Latency
  • Min FLIT latency per target
  • Max FLIT latency per target
  • Average FLIT latency over active time, per target
  • Throughput per target
  • Cycles per FLIT (CPF) per target (An estimate of network utilization)

Each of the drivers currently report the following metrics

  • Transaction Count (Number of transactions issued)
  • Max Queue Size attained
