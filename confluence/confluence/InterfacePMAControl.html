Signals:

output asleep; // Signal indicating the controlled block is in a powerdown mode.
output goto_sleep; // Signal indicating the controlled block should go into powerdown mode.
input  busy; // Signal indicating the controlled block is busy and cannot powerdown.
input  abort; // Signal indicating the controlled block is aborting a powerdown request.

Description:

This interface defines the connection between a pma Slave and the block (or blocks using a pma_mux) it controls. See sym_pma_slave description in the pma page for more info. Search "Slave Block Interface".
