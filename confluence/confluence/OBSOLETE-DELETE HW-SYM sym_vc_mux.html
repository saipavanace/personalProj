Module Name

sym_vc_mux

Parameters:

{
  "interfaces" : {
    "clkInterface" : {InterfaceCLK}, 
    "cfgInterface" : {InterfaceAPB}, // optional, expected if weightsProg is yes.
    "outInterface" : {InterfaceATP},
    "inInterfaces" : [{InterfaceATP},,,], // array of out interfaces
    "protectionInterface" : {InterfacePROT} // optional, expected if protectionStyle is not null
  }, 
  "numPri" : positive integer,
  "mstrArbLck" : true or false,
  "mstLckStyleVld" : true or false,
  "arbType" : {
    "master" : "a_string",

  },
  "arbRdyAware"  : "yes" or "no",
  "weights" : {"master" : [positive integer,,], // array depth = numVc in outInterface
                "vcs" : [{"vc":[positive integer,,]},,] // array depth = numVc in outInterface

  "vcMap" : [{    // array of input VC to output port and and output port VC. length equals number of inInterfaces.
     "inVc" : [array of integers],           // 0 to number of inInterfaces VCs minus 1. Length is number of inInterface VCs.
     "outVc" : [array of integers]},         // 0 to number of output VCs on that outInterface minus 1. Length matches inVc.
     {in1VcMap}
     ...
     {in\=inInterfaces.length-1=\VcMap}],
  "protectionStyle" : protectionStyle}

Description

Takes in multiple VC links and  and collapses it to a single link with a mapping of VCs. The most common use case is from multiple non-VC links to a single VC link.

There are no restrictions on the map of input VCs to output VCs. This is because there is a sym_switch embedded in this block which can arbitrate multiple request for the same same VC.
