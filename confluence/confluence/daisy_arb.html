I/O

u.port('input','clk',1);
u.port('input','reset_n',1);
u.port('input','in_req',width);
u.port('input','next',1);  // When asserted update the state of the arbiters.
for (var i = 0; i < depth; i++) {
  u.port('output','out_gnt_'+i,width);

}

Description

The block is meant to aid in the situation where you have multiple requests that can be allocated to multiple targets in the same cycle. Any request can be allocated to any target. The block does this by daisy chaining arbiters.

Suppose you have a situation where you have multiple FIFOs trying to allocate into multiple accumulators. For example, 8 FIFOs allocating into 3 accumulators. In this case, you would set width = 8 and depth = 3. The first arbiter is used to grant for the first accumulator which is output on the output out_gnt_0. This grant vector is then inverted and ANDed with in_req and applied to the second arbiter. Finally, for the third arbiter, the outputs of the first two arbiters (out_gnt_0 and out_gnt_1) are ORed together, then inverted and ANDed with in_req and applied as the input.

All the arbiters have internal state, but they are all forced to have the state of the first arbiter.
