gen_cfg.pl

The gen_cfg.pl script can be used to generate all supported maestro configurations that are used by the models. To understand how to run maestro itself to generate these, see Maestro Note

Maestro Examples

  • input files are not needed for the default configurations
  • run command: gen_cfg.pl --queue
  • output json: perf-vc/tests/json/maestro_example/hw_config_*.top.level.rtl.csr.json
  • Note that output json is not expected to be checked in, but always generated so that it is always current with latest maestro

hw-test-projects

  • input tcl files are from hw-test-projects/ncore_v3.0_configs/fsys_config/
  • run command: gen_cfg.pl --proj --queue
  • output json: perf-vc/tests/json/hw-test-projects/hw_cfg_*.top.level.rtl.csr.json
  • Note that these json are currently checked in

Gen Cfg Details

Gen Cfg Detail
Name:
       gen_cfg.pl - generate standard cfg jsons

Usage:
       gen_cfg.pl [--help] [--queue] [--proj]

Description:
       Run maestro to generate standard jsons in perf-vc/tests/json
       -h , --help
              Print this message and exit
       -p , --proj
              Use tcl files in hw-test-projects from workspace and put
              json file in perf-vc/tests/json/hw-test_projects
              Default is to use maestro configs an dput in maestro_examples
       -q , --queue
              Run on the queue in parallel instead of in order locally
       -k , --keep
              Keep all of the files in the run directory.
              Default is to keep only if there are fails.
