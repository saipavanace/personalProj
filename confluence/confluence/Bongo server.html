Introduction

The following document describes architecture and structure of the server-side (codename Bongo) of the ArterisIP product line.

The goal of the Bongo application is to provide overall platform and underlying infrastructure for multiple services required by the ArterisIP client applications.

Bongo instance has ability to deploy, run, update and manage multiple services.

Bongo runs multiple instances of needed services and exposes and API gateway what is accessible by the client application.

Bongo is responsible on client authentication and protection of the communication channel between client and the service.

Bongo instance can support multiple open client sections.

Instance provides service discovery mechanism.

Installation / Configuration

Bongo server instance can be installed on the ArterisIP server infrastructure, on premises or in the cloud.

Installation procedure includes checking of the execution environment and resources:

  • Linux, libraries and tool versions
  • memory
  • disk space
  • CPUs

Administration

Bongo instance provides Web based administration tools that helps with management of:

  • update channel
  • peer instances
  • services
  • users
  • versions
  • compute / storage resources
  • licenses
  • 3rd party tools
  • life-cycle status and analytics

Continuous deployment

Once Bongo instance is up and running, it connects to the update channel and install all needed services.

Bongo platform provides unified automatic way for each service to perform:

  • service unit testing
  • platform testing
  • delivery to staging
  • application acceptance testing
  • deployment via update channels
  • post deployment QA and feedback

Each service can be deployed or updated independently and without server or services downtime.

Bongo instance need to be able to:

  • isolate service instances
  • constrain the resources consumed by a service instance
  • monitor each instance
  • scale number of needed services

Bongo instance can provide feedback via update channel.

Bongo services

Bongo instance run and manage multiple loosely coupled services.

Services running under Bongo developed, deployed and updated independently.

Each Bongo service handles HTTP requests, executes business logic, access database and returns JSON/XML/etc. results.

Services are the logical components responsible on different functional aspects of the final application.

Bongo allows each of development teams focus on specific functionality, providing infrastructure, integration and testing capabilities.

Service component must be easy to create, deploy and update.
