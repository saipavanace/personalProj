The Maestro tool uses Tool Command Language(Tcl) as the scripting language integrated in it.


Tcl capabilities

You can enter individual commands in Maestro Tcl Shell from the command line. All the individual commands can be found in Maestro Tcl Commands

You can also run individual commands from the the Tcl console at the bottom of the Maestro GUI.

You can use the Tcl Shell mode to interact directly by asking for information from the design, particularly around parameter settings and object dependencies.

Example: query information regarding the gating of the clock domain attached to a particular object.


To know all the commands that Maestro supports, you can use the help command

maestro -help


You can also create an entire design using Tcl and run this script from the command line. 

You can also source a tcl script from the Tcl console in the Maestro GUI.


Sourcing a Tcl Script

A Tcl script can be sourced from either one of the command-line options or from the GUI.

A Tcl script can be sourced from the Tcl command-line option in the following way:

source <file-name.tcl>

The same Tcl script can also be sourced from the Maestro GUI using the same command in the Tcl console.


When a TCl script is invoked, it prints out all the operations taking place and all other operations are blocked until the script completes.

If the script execution is interrupted before completion, you lose all the work done since your last save.


Maestro log files

When the Maestro executable is invoked, it writes the maestro.log file in the same directory where the design is executed. The log file contains a record of all the operations performed at every stage of the design. It categorizes the messages as Info, Warning, Deprecated or Error.

You will not be able to generate the RTL incase of any Errors present in the maestro.log.
