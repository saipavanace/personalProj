3.Test Bench Support

Release R1

  1. R1 Testbench is supported with default config-mode (to irritate mode-in)=0

       2. User Settable Boot time setting of off-chip source (config_mode to irritate mode_in) supported with -so "\+config_mode=<num>"

       3. PLS NOTE: In case, you dont use -so "\+config_mode=<num>", the default value would hold until the first value in pamMode lookup is 0 for the config (recommend to use for the mode TB wants to bootup with)


Release R2/R3

  1. R2/R3 Testbench is supported with simulation cycle config-mode change

           For eg to support: reset --> +seq=mode0_seq (set the offchip source to 0)--> traffic stimulus --> prepare to change mode --> idle the traffic --> +seq=mode2_seq (set the off-chip source to 2) --> traffic_stimulus
