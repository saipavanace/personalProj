Legato Regression Setup

As a part of the demonstration, a script has been written that runs all the library blocks that have stand alone library benches associated with them. The script is called:

run_legato.sh

And is located at:

hw-lib/library_dut

The script is not very automated. If blocks are added to the library, you will need to manually modify a file called blockList.json. The hope is this will eventually be integrated in runsim.

To run the script create a symphony area and run the legato regression

/engr/dev/tools/script/setup_sym
source SOURCEME
runsim -l legato -g

Now, go to a directory any where and create a place to store the data:

mkdir legato_lib_cover
cd legato_lib_cover

In this directory you will need a file called:

blockList.json

This file contains a json array of the library blocks you want extracted from the legato configs to run coverage on:

[
"sym_switch",
"rdy_vld_pipe",
"smi_async_adapter",
"bits2cnt",
"gen_async_adapter",
"apb_async_adapter",
"sym_nRate_adapter"
]

This is where you add and remove library elements from the run.

When you run a legato regression in symphony, there will be a results directory stored at:

$WORK_TOP/regression

In this example, that directory for me was 2022_11_28_0821. You could have multiple directories under regression; you need to pick one that holds the results of the legato regression.

Run the command:

$WORK_TOP/../../hw-lib/library_dut/run_legato.sh /scratch/jcodd/regress/hw-sym/dv/regression/2022_11_28_0821

run_legato.sh is not very automated. It doesn’t automatically find all the configs that are in legato, there is a line per bench. If you need to add a config, you will need to duplicate a line and update the information to point at a new config. If you need to remove a config, you will need to delete the line that runs the config from the script.

The directories are the run areas of the regressions, one per block type run. The directory only contains the data from the last run done for a block type. If you want to rerun a fail, just copy and paste the command in verif.log onto the command line while sitting at the level you ran run_legato.sh at.
