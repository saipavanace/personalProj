numPri: A positive integer that indicates the number of priorities,

mstrArbLck: A boolean. Refer HW-CTF sym_sw_arb for description.

arbType: A string that specifies the master arbiter type. Legal values: "arb_rr1, arb_rr2", "arb_wrr_rr1", "arb_wrr_rr2", "arb_pri_rr1", "arb_pri_rr2",

arbRdyAware: Boolean value, When true, the master arbiter uses the ready and valid to make a request. When false, just valid.

weights: array of positive integers and of length = number of VCs, that specify the weights for the master arbiter,

weightsProg: Boolean value to be set to true if weights are to required to be programmable,

pipeLevelLut: Depth of pipe in LUT,

pipeLevelATP: Depth of ATP pipes inside SMI,

pipeLevelSmiHdr: Depth of SMI HDR pipe inside SMI,

pipeLevelSmiDP: Depth of SMI DP pipe inside SMI,

assertOn: 1 - Enable Verilog assertions. 0 - Disable Verilog assertion,

protectionStyle: an Object with the following keys"forceErr" : "yes" or "no", "protection" : "ecc" or "parity", "scrub" : "yes" or "no", "useCorrected" : "yes" or "no", "protWidth" : an integer,

enPathLookup: 1 - Enable Path lookup. 0 - Disable Path Lookup,

routeOnSteer: If this is set to true, steer bits are used for determine route path lookup. If false steer bits are ignored. This has an effect only when enPathLookup is 1 (as expected for a packetizer in a Target ATU)

routeOnLabel: If this is set to true, multicast 'label' bits are used for determine route path lookup. If false steer bits are ignored. Expected to be used when a network has multicast blocks. This has an effect only when enPathLookup is 1 (as expected for a packetizer in a Target ATU).

pathLut: Array of objects. Lookup Table to calculate route on the response path. Each object inside the array must be of the following type(format) - { "targ_id": target ID string, "route": route string }.
