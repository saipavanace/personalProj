Testbench:

  1. The following checks will be performed on every test defined below..
  2. Scoreboard/Assertion/TACHL will check for the following::
  • (1a) -  Make sure SRAM_sym_fifo_adapter is used by the RTL,  NOT the REGISTER type.   

          (Implement through tb_wrapper.tachl (instance array format) –  DV)

  • (2a) -  Make sure SRAM_sym_fifo(s) is/are empty, at the end of simulation. 


  • (3a) -  Make sure that  FIFO depth is not violated. 

   (This check will be made through Assertions – RTL)

Things to know:

  1. The sym_fifo_adapter resides in the network_layer. It is possible to have such adapter in both: Request and Response paths. 
  2. The configuration listed in the table below will include all three packet styles randomized:  "serial" or "parallel" .
  3. sram_sym_fifo_adaptor →  SSFA

Answers  for  (1a) and (2a)::

  • How to make sure the RTL is using SRAM_fifo  instead of  REGISTER_fifo ?   

          Answer: (Need to be implemented through JSON file)

  • How to determine that the SRAM_fifo  is empty?

          Answer::  Use the following signals: in_valid and out_valid  signals from sym_fifo_mem_1p module — not just the empty signal


Some questions from Junie:

  • What is the reset value of the SRAM after reset?

           Answer: The SRAM doesn’t get reset. The FIFO uses pointers, so the contents of unused entries don’t matter.

                          Entries won’t be read until they are written.

  • Do we do BIST for the SRAM?

   Answer:  No, BIST is a customer supplied function.

  • If the clock is gated, what happen to the SRAM module?

           Answer:  Its clock gets turned off.

  • About clock, we are targeting 2-GHZ, but not sure the SRAM can use it…

           Answer: We’re adding the ability to step down the frequency of the SRAM by Customer. The customer will then be able

           to choose the frequency of the SRAM in the FIFO based on the what speeds they have available. 

  • There are no interrupts generated by the Sram Fifos for R1 – that implies no interrupt testing are required for this release.

          Please refer to SYM-5782.
