Merging Coverage:

You can merge multiple VDB directories by specifying them after the -dir option on the URG command line. The input VDBs can be generated at compile time, at simulation time, or by another merge operation. For example:

urg -dir regress2_design.vdb regress2_run1.vdb mymerge.vdb -dbname simv2.vdb

This command reads all the specified VDB directories and merges them to create a new VDB directory named simv2.vdb. The -noreport option does not generate a report at the same time. It is better to generate reports separately.

If you have a long list of VDB directories to be merged, you can use the -f option to URG. This option enables you to specify all the databases for merging in a file. For example, the following urg -f command takes the list of databases for merging from the vdblist file:

urg -f vdblist -dbname final_merge.vdb -noreport

Merging Code Coverage Across Instances:

 mkdir <dir>

cd <dir>

cp <coverage path>/hier.cfg .

urg -hier hier.cfg -metric line+cond+branch+fsm -metric group+assert -lca -parallel -dir <#1>.vdb <#2>.vdb -group merge_across_scopes -flex_merge union -show tests -dbname <output_dbname>.vdb -format both -report urgReport -map <instance name> 

Best Known Method:

To handle functional coverage, it is advised that you create a verification plan (.hvp) per configuration and have it stored per block so it can be reusable. If this is your first iteration, you can generate a new plan and add features and measures as you find fit for your configuration
