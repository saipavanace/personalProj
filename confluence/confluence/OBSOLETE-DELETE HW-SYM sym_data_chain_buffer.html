module_name:

sym_data_chain_buffer

Features

  • This module is used to store chains of data beats.
  • These chains can be constructed from multiple interleaved channels.
  • The data beat chains can be retrieved in any order. (Thus why a regular FIFO cannot be used)


I/O


input                                                in_valid;

input [log2ceil(numChannel)-1:0]   in_channel

input [wData-1:0]                            in_data

output                                             in_ready;

output [log2ceil(depth) - 1:0]          free_ptr;


output                                             out_valid;

output [wData-1:0]                          out_data

output [log2ceil(depth) - 1:0]           next_ptr;

input                                                out_ready;

input   [log2ceil(depth) - 1:0]           read_ptr;

Parameters

{

     wData: 1 - inf, // Width of the data to be stored,

     depth: 1 - inf // The total amount of possible stored beats.

     numChannel: 1 - inf // The total amount of input channels

}

Description

This module is used to store chains of data beats. The user will store the free_ptr from the first data beat going in and use that to look up its chain of data beats at a later time.

Data coming in can be interleaved between the channels. In turn this means that numChannel amount of chains can be in construction at any given time. 

When retrieving data the read_ptr will be driven to the entry that needs to be read.
