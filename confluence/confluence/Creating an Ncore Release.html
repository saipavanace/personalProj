1. Go to or create a git workspace containing 'client' and 'bundler' for the branch that you want to release.
    Example:

    git clone -b Ncore-1.6 ssh://git@stash.arteris.com:7999/con/client.git
git clone -b Ncore-1.6 ssh://git@stash.arteris.com:7999/con/bundler.git

  2. Update the client/cpanel/package.json with the version numbers of any changed dependency modules, E.g.:

    "@arteris/codegen": "1.6.45",

  3. Update the client/package.json with the release version string for the release (or release candidate),  E.g.

    "version": "1.6.0-rc.17",

     

  4. Commit and push the client changes:

    git commit -a
git push
  5. Wait for the client Bamboo build for that branch to finish.
  6. If the Bamboo build was successful, it should now allow you to 'Deploy' the results of that build.  If you have adequate permissions to deploy the build, the right side of the build page will look like:

    If the 'Create release' button isn't showing, check to make sure that you are logged in to Bamboo.

  7. Click on the Create release button.  You will be redirected to a Create new release page:


  8. Change the This version field to a name in the form <releaseName>-<buildNumber>, e.g.

    1.6.0-rc.18-70

  9. Click on Create release.  That will bring you to a Release page, e.g.:

  10.  Click on the Deploy button.  There will be a dropdown that allows you to select one of the deployment types that you have permissions for, e.g.:

  11. Click on Debug . Then click on Start Deployment .  Once the deployment is successful, it will create a file in /engr/dev/releases/conductor (also visible in http://dev.arteris.com/releases/conductor/), e.g.:

    $ ls -l conductor-1.6.0-rc.17-69+debug.tgz 
-rw-r--r-- 1 bamboo 501 199848728 Sep 30 13:44 conductor-1.6.0-rc.17-69+debug.tgz

  12. Use the browser back button to go back two pages, and repeat the previous two steps for the Release build.

  13. Repeat again for the NoRTL build.

  14. Once the deployment of the Release, Debug, and NoRTL builds complete, you can the create the bundles which install those builds.  This is done by modifying the bundler package so that Bamboo will create the bundles.  Change the bundler/env file to have the release and build information for the current release/build, e.g.:

    CONDUCTOR_VERSION=1.6.0-rc.17
CONDUCTOR_BUILD_NUM=-69
  15. Commit and push the changes to bundler.

  16. Wait for Bamboo to finish running bundler.  Once it completes, there will be installers in /engr/dev/releases/Ncore for the Debug, Release and NoRTL builds (also visible in http://dev.arteris.com/releases/Ncore/), e.g.:

    $ ls -l Ncore-1.6.0-rc.17-69*
-rwxr-xr-x 1 bamboo 501 159152675 Sep 30 13:51 Ncore-1.6.0-rc.17-69+debug.run*
-rwxr-xr-x 1 bamboo 501 159152675 Sep 30 13:51 Ncore-1.6.0-rc.17-69+noRTL.run*
-rwxr-xr-x 1 bamboo 501 134921152 Sep 30 13:49 Ncore-1.6.0-rc.17-69.run*

  20. Update the symbolic links to point to the new versions.  For an 'ir' or 'rc' the symbolic links are named 'internal', 'internalDebug', and 'internalNoRTL'.  For a customer release version the symbolic links are named 'release', 'debug', and 'noRTL'.

    $ ln -fsn Ncore-1.6.1-rc.1-91 internal
$ ln -fsn Ncore-1.6.1-rc.1-91+debug internalDebug
$ ln -fsn Ncore-1.6.1-rc.1-91+noRTL internalNoRTL
