NOTE: Please run maestro on one of the Centos 7 machines.


There is a new version of the environment module for testing maestro:

  • % module load dev/maestro_test/latest
  • % module load dev/maestro_test/v1.2.1

 

It will set up your PATH and other environment variables with the tools necessary for the Design and DV teams to test the initial internal release of Maestro.This prepends maestro, verilator, and run_maestro to PATH.

 

We have also setup another module which keep up with the most recent changes to dependencies:

  • % module load dev/maestro_test/bleeding_edge

Once you have loaded the dev/maestro_test/<version> module, to generate rtl/dv jsons you can create a new output directory and run:

  • % run_maestro

To generate rtl/dv jsons + rtl, pass the -g option:

  •  % run_maestro --gen_rtl 

 

To generate the rtl/dv jsons + unit jsons + rtl:

  •  % run_maestro --gen_rtl --ncr_debug_json --sym_debug_json 

 

To get a list of versions and past versions with dependencies:

  •  % run_maestro --show_versions

 

Some useful commands: 

  • % run_maestro --help (help command / view options)
  • % run_maestro --repo_path <path> (override path to hw/sw dependencies; must specify a directory that contains hw-ncr, hw-sym, hw-lib, hw-ccp, and utils)
  • % run_maestro -t <path> (specify top tcl file path)
  • % run_maestro -c (cleanup generated collateral)
  • % run_maestro --ncr_debug_json --sym_debug_json (generate ncore / symphony unit jsons to output/debug/ncr_debug_jsons & output/debug/sym_debug_jsons)
  • % run_maestro --use_tachl_debug (use tachl debugger)
  • % module purge (unload all loaded modulefiles)
  • % module list (list all currently loaded modulefiles)
  • % module avail dev/maestro_test (check all available dev/maestro_test modules)

Instead of specifying --repo_path you can explicitly override each hw/sw dependency with these run_maestro flags:

 

  • --hw_ncr_path <path>
  • --hw_sym_path <path>
  • --hw_lib_path <path>
  • --hw_ccp_path <path>
  • --sw_utl_path <path>

 

The output directory structure had been updated:

  • rtl & dv jsons are in output/debug
  • rtl & flists are in output/rtl/design
  • ipxact is in output/IPXACT
  • ncore unit jsons are in output/debug/ncr_debug_jsons
  • symphony unit jsons are in output/debug/sym_debug_jsons

 

To get a copy of Tcl files for modification run this command in the current output directory: 

  • % cp -r $MAESTRO_EXAMPLES/sa_dryrun/* .

The default from the run_maestro command will use these tcl files from this directory.

 

To run verilator on the RTL files in output/rtl/design run from the current directory: 

  • % verilator --lint-only --error-limit 0 -Wall -Wno-UNUSED -Werror-UNDRIVEN -Ioutput/rtl/design output/rtl/design/gen_wrapper.v

Note that there are currently several verilator errors in the RTL.
