Experimental Environment:

  • Address increased by cache line, no overlap between NCAIUs
  • Back-to-back traffic injected
  • All traffic is targeting DMI
  • In QoS enabled configuration, assigned 7 only to NCAIU0.
  • SRAM model update
    • 100 cycle latency at destination (77ns, 1.3GHz target frequency)
    • 64 outstanding capability, which matches with DMI RTT entry

1) Observation

  • QoS effect shows when system is congested.
    • When all the NCAIUs are concurrently working, observing 47% of the latency improvement on QoS enabled initiator, with 5% of BW improvement
  • When system is idle, do not see much improvement 
  • Sequence 4 shows meaningful disadvantage when QoS is enabled..
    • Observing DCE credit is not balanced well also, in case of sequence 4. 
    • The balancing became worse only when QoS is enabled.
    • Observing that the DCE cmd buffer occupancy is not balanced in that case.
  • Maximum achievable BW is 92% when all the NCAIUs are concurrently working, and limited by MRD credit of the DCE.
