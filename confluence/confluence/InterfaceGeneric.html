javascript parameters:

{ 
  "name": "a_string", 
  "params": 
     {
         wIn : integer, // Width of all input signals concatenated 
         wOut : integer // Width of all output signals concatenated
     },
  "direction" : "master" or "slave", 
  "interface": "InterfaceGeneric",
  "synonymsOn" : true,
  "synonyms" : {
              "in":  [
                      {
                        "name" : string, // Name of the signal,
                        "width": integer, // Width of the signal, 
                        "upf": integer, // Value to tie signal to in UPF (In Decimal),
                        "type": string // Behavior of signal for use in pipe lining and synchronization  ("pulse" or "level")
                       },
                      ...
                      ]
              "out": [ <Same as Above> ]

                       }

}

Signals:

output [wOut-1:0] out; // Signal carrying the concatenation of all generically defined output signals.
input  [wIn-1:0] in; //Signal carrying the concatenation of all generically defined input signals.

Description:

This interface can be used in multiple situations:

  1. Carry a user defined protection interface to the placeholder block in each NCore Unit.
  2. ... 

Functions:

In the bundleFunction.js library there will be additional functions which can take in the synonym and perform useful functions:

Function NameDescriptionArgumentsImplemented?
wiresFromSynCreates wire definitions for the signals contained in the In and Out arrays.(prefix,synonym,bundleFunction)Yep
regsFromSynCreates reg definitions for the signals contained in the In and Out arrays.(prefix,synonym,bundleFunction)Yep
explodeInFromSynAssigns all the user defined wires from the In signal.(prefix,synonym)Yep
assignOutFromSynBundles all the user defined wires into the Out signal.(prefix,synonym)Yep
explodeSyn

Converts the In Out signals into the user generated signals.

-Performs both explodeInFromCatDef,assignOutFromCatDef, and possibly {wires/reg}FromCatDef

(prefix,synonym, {wire or reg or none},bundleFunction)Yep
SynToBundle

Returns a bundle with the In signals. Out Signals, or both.

(synonym, {in,out,or both})Yep
