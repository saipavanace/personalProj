This function returns the width of a field that exists in a link packet description.

It is called as follows:

var return_data =  getPktFieldWidth(link_width,link_packet_def,field,type,beat)   (type and beat are optional)

link_width : width of the link in bits

link_packet_def : packetDesc

field: text string used to match against field names.

type: Optional argument. Default = "X" (Unknown). 'type' has to be specified as one of the following values "H" (Header), "F"(Fixed), "C" (Copy/Clone) or "N" (NULL)

beat: Optional argument. Default = -1. Specifies the beat number in the packet at which the required field is to be searched for.


The function relies on another function HW-CTF extractFieldBitIndexes which returns start and end indexes of the field.. And then width is calculated as start-end+1. if the field isn't found, the width returned is zero.


For typical usage, the optional fields can be skipped, and the first two arguments link_width and link_packet_def can be partially applied to create a bound function.
Example:

var fwidth = symLib.getPktFieldWidth.bind(null, wLink, pktDef);  // Create a bound function. This step is not necessary but using this avoids function calls with several arguments.

fwidth("H_sec")  // Returns the width of H_sec (first instance) in pktDef. And guaranteed to not match "H_sec_fail"

fwidth("F_data") // Returns the width of the first instance of F_data in pktDef

fwidth("NULL_D") // Returns the width of the first instance of NULL_D for Header-Serial and width of NULL_D if present in first-beat for Header-parallel

fwidth("NULL","N",0) // Looks for any NULL field in the first beat.

fwidth("NULL","N",2) // Looks for NULL field in beat=2 (Could be a NULL_H field in a multi-beat header)
