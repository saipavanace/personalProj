Blocks

Legato blocks used by Ncore:

HW-CTF sym_switch

HW-CTF sym_pipe_adapter

HW-CTF sym_buf_switch

HW-CTF smi_async_adapter

HW-CTF dw_adapter

OBSOLETE-DELETE HW-CTF smi_pkt

HW-CTF smi_depkt

In addition Ncore uses the following ATUs to create the configuration network:

HW-CTF atui_axi

OBSOLETE-DELETE PAGE HW-CTF atut_apb

Where the atui_axi connects to the sys DII and there is a dedicated atut_apb for any Ncore block that has an apb configuration port on it. The above two blocks can be configured to have a configuration port on them, but in the Ncore use case, this is never done.

Interfaces

The following interfaces are used by Ncore associated with Legato:

HW-CTF InterfaceATP

HW-CTF InterfaceCTL

HW-CTF InterfaceSMI

The following interfaces are shared with Ncore and are from the shared library:

InterfaceCLK

InterfaceAPB

InterfaceAsyncCrFIFO

InterfaceAsyncFIFO

InterfacePMAControl

Packets

ATP interfaces carry packets and those packets are defined with a packet definitions that follow the format defined at:

HW-CTF packetDesc

This format is very flexible so some restrictions and structure is applied to it for the present hardware. In Legato a set of elements in a Fabric share a common Base Packet Definition. From the Base Packet Definition, link packet packet definitions are derived and the following specifies that algorithm at the bottom of the page:

HW-CTF Base Packet Parameters & Relationships

The above link also describes how Base Packet Definitions are constructed based on the parameters of the Interfaces and blocks connected to the networks. The fields in a packet are not fixed and change width and number based on how the blocks surrounding the network are configured. The spreadsheet that is attached to the above page describes these rules and version 134 is what applies to Ncore 3.2.
