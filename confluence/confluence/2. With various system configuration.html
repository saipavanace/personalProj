We selected three test cases from the test 1 as below:

  • Read only concurrent test – Test 1-8
  • Write only concurrent test – Test 1-7
  • PCIe scenario, CAIU enabled - Test 2-3


In this test, SRAM model integrated with DMI is configured as below

  • 100 cycle turn-around time (77 ns, 13.GHz)
  • Read outstanding: 64
  • Write outstanding: 32


TestObservation
1. Read only concurrent test
  • QoS effect is being observed when system is congested.
  • When system is idle, do not see much improvement 
  • Sequence 4 shows meaningful disadvantage when QoS is enabled. 
    • Observing DCE credit is not balanced well, because, 
    • DCE skid buffer occupation balancing is broken. → Need more investigation
  • Maximum achievable BW is 92% when all the NCAIUs are concurrently working, and limited by MRD credit of the DCE. 
2. Write only concurrent test
  • QoS effect is being observed when system is congested.
    • When all the NCAIUs are concurrently working, observing 40% of the latency improvement on QoS enabled initiator, with 7% of BW improvement
  • When system is idle, do not see much improvement – shows slightly worse data than qos off case.
  • For the sequence 1 and 2, system is congested due to WTT entry of the DMI, and in case of 3, 4, and 5, system is congested because of RB entry of the DCE.
3. PCIe Test with CAIU
  • BW is similar with 0ns SRAM model case.
  • Resource status: DCE MRD credit is not bottleneck anymore
    • with 0ns latency, DCE MRD credit was the bottleneck, mainly because of the MRD turn-around time. 
  • For the higher QOS value shows better latency numbers, however, 
    • The traffic with higher qos value shows 8% of the latency improvement, which is smaller than 0ns case.
    • The smaller latency improvement becomes low, because system itself is not limited by local credit.
