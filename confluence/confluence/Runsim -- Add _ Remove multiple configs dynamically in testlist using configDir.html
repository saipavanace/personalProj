runsim provides the ability to add & remove multiple configs from the repository dynamically into the testlist. Following are the rules & assumptions

Define dynamicConfigDir in top-level testlist → $WORK_TOP/scripts/runsim_testlist:

"dynamicConfigDir": {
    "gen_cfg_top": "$WORK_TOP/gen_cfg",
    "tcl_cfg_top": "$WORK_TOP/tcl_cfg"
}

Parse through $WORK_TOP/gen_cfg/ and look for all hierarchies for native layer environment folders. Example:
  1. axi_axi → all configs inside will be grouped in top_axi_axi environment
  2. axi_apb → all configs inside will be grouped in top_axi_apb environment and so on
  3. apb_apb
  4. apb_axi
  5. nl → top_nl

Excluding configurations:

  1. Each $WORK_TOP/gen_cfg/axi_axi/exclude – exclusion directory.

Labelling 

  1. runsim adds default config & test label - 'dynamicConfig' on each configuration for easier filtration of dynamically added configs and it's tests.

Test cases:

  1. Each environment type (axi_xxx, apb_xxx, etc) should have default set of test cases to attach with each configuration.
  2. These test cases are selected using labels.
    1. Format of label - 'default_<env>'
    2. Example - 'default_axi_axi', default_axi_apb', 'default_nl' and so on.

Same structure we will follow for TCL based configurations.
