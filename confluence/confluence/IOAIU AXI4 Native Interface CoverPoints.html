Refer Section: A7.2.4 Exclusive access restrictions:

  • The address of an exclusive access must be aligned to the total number of bytes in the transaction, that is, the product of the burst size and burst length.

  • The number of bytes to be transferred in an exclusive access burst must be a power of 2, that is, 1, 2, 4, 8, 16, 32, 64, or 128 bytes.
  • The maximum number of bytes that can be transferred in an exclusive burst is 128.
  • In AXI4, the burst length for an exclusive access must not exceed 16 transfers.

  • The value of the AxCACHE signals must guarantee that the slave that is monitoring the exclusive access sees the transaction. For example, an exclusive access must not have an AxCACHE value that indicates that the transaction is Cacheable, i.e AxCACHE[3:2] = 'b00

  • If there is a buffer or cache which might respond to an exclusive access before it reaches the monitor, then the exclusive access must be Non-bufferable  i.e AxCACHE[0]=0 or Non-cacheable i.e AxCACHE[3:2] = 'b00

  • The transaction type must be ReadNoSnoop or WriteNoSnoop, i.e non-coherent.
  • The domain must be Non-shareable or System-shareable i.e non-coherent (This applies to ACE interface)
