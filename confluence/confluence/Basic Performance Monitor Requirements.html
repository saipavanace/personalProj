• Ability to filter packets for performance monitoring
    • Same filter capabilities as tracing
  • Ability to measure end-to-end transaction latency of given point (can be at Initiator ATU only)
    • Results can be reported as average and max latency measured in the specified period
    • Histogram type of output - not required for basic mode
  • Ability to measure Bandwidth at a given point
    • Need to distinguish packet consumed bandwidth vs data payload bandwidth (e.g. if packet header consumes BW of the links, it will be different from payload bandwidth)
    • measuring other attributes, like backpressure cycles, etc. - not required for basic mode
  • Output
    • Through registers (registers to program when to start/stop measurement)
    • Through ATB interface - not required for basic mode - this is for measuring periodically and dumping results on ATB
