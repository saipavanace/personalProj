parameters

	"data_width" : any positive integer, 
        "addr_width" : and positive integer equal to or less than data_width
}

I/O

input syndrome[ecc_width-1:0];     // syndrome from ecc_dec

output addr_err;           // The address portion has a single bit error

Description

Takes in the syndrome and determines that the bit in error is associated with the address portion of ECC protected data of an SRAM.


The code uses a Modified Hamming Code for the SECED encoding.
The assumption is when data is written into an SRAM, that the upper bits of data fed to the ecc encoder are the address of the SRAM the data is being written to:
assign ecc_data = {addr,data};
