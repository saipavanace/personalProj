The Bucketizer is a tool to help simplify timing reports and identify the failures.

The tool reads the report and creates “buckets” of related failing paths. This help identify the potential fixes that will help the most.

Usage: bucketizer [options] >

<block name> Required field

-drop Required field, i.e., 8.0, 9.0

-timing_report Optional field, timing report file to read in. Default is makeit output

-report_dir Optional field, sub-dir relevant to block path. Default is makeit output

-date Optional field, Format is MM-DD-YYYY. if not provided, current date will be used

-prev_drop Optional field, previous drop number to compare to

-drop_note Optional, could be anything

-corner Optional, set to max, min or test. Default is max

-type Optional, set to functional or test. Default is functional

-slack_lesser_than Optional, default is 4.0ps. Paths with slack < this value are reported as failing paths

-max_paths Set the maximum number of paths that are stored in a bucket.
This is useful for limiting the amount of disk space a drop uses.
Setting this to anything larger than 0 will limit the paths saved.
Setting this to 0 or less will dump all paths. Default is 0.

-max_buckets Set the maximum number of buckets that are reported in a group (Internal,External,StartEnd,Net) This is useful for limiting the amount of run time Default is 25.

-help | -? Print this usage message.

Usage Example:
bucketizer -timing_report dc_timing_max.rpt -report_dir dc_syn/reports -local -drop 11 -prev_drop 10 -drop_note "anything" -stage dc dmi_a
bucketizer -timing_report timing.max -report_dir timing/reports -drop 11 -prev_drop 10 -date 03-04-2005 -drop_note "anything" -stage pt -slack_lesser_than 0.1 design_top
bucketizer -remove_drop 1 -drop 11 -stage pt -corner max design_top
