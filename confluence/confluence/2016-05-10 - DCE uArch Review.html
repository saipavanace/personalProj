Goals

  • Review the DCE uArchitecture, looking for optimization opportunities.

Discussion items

Notes:

  • The credit check prevents all SNPs (and HNTs) from being sent if any agent is out of credits.  See CMPS v0.22 p82.

Potential optimizations

  • We can add an output bypass that sends SNP/MRD/STR directory without pushing into a SNP/MRD/STR fifo, when we know that the transaction is guaranteed to win access to the SFI master interface, saving a cycle of latency when the system is idle.
  • In the case of a DCE with only Null filters, we can probably eliminate a pipeline stage (P1 in the DM).
  • We can make the SNPreq fifo shorter by preventing coh (not updates) requests through the DM.   If we do this, it becomes economical to store everything we need in the SNP FIFO, eliminating the SNP mux from the output of the ATT.
  • HNTreqs are not sent if there are no SNP credits.  There is no reason to prevent this.
  • Alternatively:  Break the SNP FIFO into two parts:  The portion close to the head stores all of the required ATT information.
  • Area savings:  We could reuse the bits in the ATT that store the request vector to keep track of the number of responses pending.
