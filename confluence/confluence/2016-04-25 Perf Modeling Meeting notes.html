There's a perf/ subdir after generation.

perf/

build.sh

 

Before you can run bui;d.sh, need to set up some environment variables:

  • CONDUCTOR_HOME = path/to/install
  • VERILATOR_ROOT = path/to/includes  (Note: Verilator is installed as part of Conductor, so this probably doens't ned to be touched if CONDUCTOR_HOME is set.)
  • SYSTEMC  = path/to/install
  • Platform Architect
    • COWAREHOME = path/to/install
    • SNPSLMD_LICENSE_FILE=

 

Users must call $COWAREHOME/setup.sh -pa to set up platform architect.  (Independatn of us.)

 

From RTL Generation:

  • build.sh
  • pct_import.tcl
  • top.h - header file that creates the top level.
  • tb.h - example testbench that instantiates top.h, drives ocp and clock and reset, and measures execution time.  Is meant as a starting point for users. Also include example makefile?

Then, in perf/:

  • sh build.sh
    • Takes 2~20 minutes
    • For each module
      • Creates folder
      • runs verilator to create C++ file
      • creates makefile (from verilator)
      • runs make, which does the C++ compile for each unit, into an object file.  (.so)

  • platformcreator pct_import.tcl  
    • Note: pct_import.tcl was generated into the perf directory at generation time.)
    • This creates a new PA project,
    • sets up the paths
    • loads all the modules
    • Saves the project.
    • Opens the project in Platform architect

  • In platform architect, you will see the subsystem, including FT-ACE and FT-AXI bridges connected to the AXI and ACE interfaces of the Ncore system.

  • From here, the user starts to connect blocks to drive traffic.
    • User needs to create clock & reset, and connect TLM drivers.
