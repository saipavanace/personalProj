This is guide how to install and use remote-ssh extension.

  1. Install Visual Studio Code (Note: our Centos7 machines are old and latest Visual Studio Code and extensions might not work properly. Latest stable version is 1.89.*)

  2. Open VS Code and open extension tab (Ctrl + Shift + x shortcut)

  3. Search for “Remote - SSH” extension and click install.

    1. Follow Getting started instructions in extension page

    2. Or after installation need to open the command palette (F1 or Ctrl + Shift + P shortcut) and enter and select “Remote-SSH: Connect to Host” (here will be list of already configured server or select “+ Add new SSH Host…”

      1. Enter server information username@calistoga.arteris.com

      2. It will open VS code with password prompt

      3. After entering it might say that remote server uses unsupported OS, which we should “Allow”

      4. Done. After that either open folder in “Explorer” tab (Ctrl + Shift + E) or use integrated terminal (CTRL + ~) and open project with terminal command “code path_to_folder"

Useful tricks:

  1. When you do “code path_to_folder” it will open a new VS Code” but if you type “code path_to_file” it will open this file in the current VS Code”

  2. After connecting to remote host, VS code will install ~/.vscode-server, sometimes in future it could be overloaded or stop working properly constantly disconnecting from remote-server it is better to run vscode command “Remote-SSH: Kill VS Code server on host”

  3. When connection or reconnection issue is happening, it might be VS Code is auto updated and not compatible with remote server anymore

  4. VS Code debugger (Using Visual Studio Code to Edit and Debug Maestro )
