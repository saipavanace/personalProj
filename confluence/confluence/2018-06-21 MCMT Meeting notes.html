50 minSymphony Technology OverviewSyed
  • Components are containers with various functionality
  • ATU needs to understand the nature of all targets and format the packets on the transport such that the target can understand.
  • ATU routing question - we have source routing (source creates the whole path via the packet); second routing algorthim is incremental routing; also have obvlivious routing and dynamic routing (need to read the network and maintain ordering).
  • HW currently supports Source ID routing or Dest ID routing. First release needs to be source oblivious routing. SW mainly cares if it is static or dynamic.
  • Maestro has to create the route table through a routing module. Not just egress ports but also what VC. SW decisions can be made on heuristics to start.
  • Links are containers with attributes like performance, QoS, etc. We need to determine with HW what exactly can be in that container. It abstracts the link.
  • Between fabrics we have the ability to change the rules between fabrics at the adapters.
  • AIUs are the Native + Common Translation layer connect through SMI.
  • ATUs have the full 3 layers - Native, CTL and PL.
  • HW inference - mapping high level characteristic into the parameters for the units to produce the solution promised ito the User.
  • Operationally - what is needed by the software will be in BtB about the components (restrictions, rules, parameters)
  • Centralized Parameter Repository is functional name for BtB.
