DVE can be used for examining customer waveforms that have been sent to us for debug. 

If a customer has provided us a waveform in "*.fsdb" format, please ask them to provide it in another format, VCD/VPD, etc. 


NOTE: We also have the option to convert an fsdb file into a vcd format directly. This works after loading in the snps/ae module in step 1 below.

%fsdb2vcd <fsdb_filename> -o <vcd_filename.vcd>


Execute the following in a new terminal. 

1) The following command loads in the proper tool versions required. Currently we need the following version of DVE: /engr/dev/tools/synopsys/vcs_vP-2019.06-SP1-1/bin/dve

module load snps/ae

2) The next step is to remove the AE licenses that are pointed by loading the ae module & set the required ones HW engineering uses. (tcsh) Execute the following commands or just include those licenses in your .csh file. 

setenv SNPSLMD_LICENSE_FILE
setenv SNPSLMD_LICENSE_FILE 5285@lic-node0.arteris.com:5285@lic03.arteris.com

3) Invoke the DVE tool. You may get an error/warning message that you can just click OK on to continue. 

dve&
Click on "File"->"Open database" and locate your <vcd_filename.vcd> file to load.
