General Setup


Door Code/Alarm


Your badge will work on the building front door and the Arteris employee entrance.
Your alarm system password is given to you on your first day. This password is associated with the badge/key to enter the office, so make sure to carry your badge/key at all times.
To disarm the system, enter your alarm system password followed by 1
To arm the alarm system, enter alarm system password followed by 2
If you are the first person to enter the office, you will disarm the system. Your have 30 seconds to disarm the system.
If you are the last person to leave the office, you will arm the system. You have 30 seconds to arm the system and leave the office.
If the alarm system is set off, immediately call 1-800-341-8282 and give them your alarm system password

VNC Server/Viewer


If you need to run program with GUI, then you will need to start a VNC server. Type the following to see a list of VNC server command options.
vncserver --help
To start VNC server, simply type the following:
vncserver -geometry [width]x[height] :[number]
[width] and [height] indicates the screen size of the VNC window. We recommend 1920 for [width] and 1080 for [height] to match the default monitor screen resolution.[number] can be any number. In the example shown in Figure 4, we pick "13" as the number. Note that this number will be unavailable if another user has already started a VNC session with it, so please try to pick a number that's unique.



Figure 4 – Starting VNC server
If you are starting the VNC server for the first time, you will be prompted to enter a password for the VNC session. This password is used for your VNC session only and it is not related to your Linux password.
Next, we need to start a VNC viewer, which can be found in the following location:
C:\Program Files\RealVNC\VNC Viewer\vncviewer.exe

Figure 5 – VNC viewer login screen
You will see a login screen like that shown in Figure 5. Simply enter the name of the server in which you started the VNC session (in this example, it is "ridge.arteris.com", followed by semicolon and the unique VNC number that you picked (in this example, it is 13). You will then be prompted to enter the VNC password that you picked when you started the VNC server for the first time. After you have successfully logged in, you will see the following screen as shown in Figure 6.


Figure 6 – VNC viewer screen after logging in
You can open a Linux terminal by right clicking anywhere on the desktop and select "Open in Terminal".
If you are done with your work, it is recommended that you close the VNC viewer, and then go back to the PuTTY terminal to terminate the VNC sessions (otherwise other people cannot use it). To kill a VNC session, type the following:
vncserver –kill :[number]
Again, [number] is the number you picked for your VNC session. In our example, it is 13, as shown in Figure 7.

Figure 7 – Terminating VNC session

Repository Access


GIT/Stash


Projects, and source codes are stored remotely in GIT at "stash.arteris.com". You can access it either through a web browser in Windows or through Linux. The web interface is called "Stash", and it is essentially the web version of the GIT repository. It is located here:
https://stash.arteris.com/
There are some public repositories that are available to everyone, but most engineering documentations and source codes are private. You will need to login to access them. Click on the "Log In" button on the top right of your browser will lead you to the following page, shown in Figure 10.



Figure 10 – Stash login screen
Your username and password are the same as your Linux password. Note that you may have to login into Linux first and change your initial password before Stash is activated. Once you are logged in, you should be able to see all the repositories, as shown in Figure 11. Note that you may get certificate error warnings. You can safely accept the certificate into your browser. If you use Firefox, you can save the certificate so that you don't get nagged every time. If you use Google Chrome, follow the steps found in the first answer here:
http://stackoverflow.com/questions/7580508/getting-chrome-to-accept-self-signed-localhost-certificate
To clone a specific repository in GIT, we first go to that repository in Stash, look up the clone address in Stash by clicking the "Clone" button on the left. Under the "HTTP" field, it shows the clone address that we will use in the Linux command prompt.


Figure 11 – Using Stash to navigate a repository
Now we are ready to go back to the Linux command prompt to start cloning GIT repository. First, we need to type the following command to turn off SSL verification
git config --global http.sslVerify false
Next, we will type the GIT cloning command
git clone [clone address]
Clone address is the address in the "HTTP" field shown in Figure 11. In this case, it is "https://michaelw@stash.arteris.com/scm/ed/engdocs.git". The "Clone" button in Figure 11 will automatically figure out the correct clone address, so you do not have to worry about it. Figure 12 shows the full commands for cloning a GIT repository.
If you are not familiar with GIT, you can find many tutorials online. Please see Chapter 3.0 for more details. Table 2 lists some common GIT commands.

SVN/Tortoise SVN


Documentation has been migrated to SVN, which allows for better version controls. To check out SVN database initially, simply create a directory, and type the following command:
svn co svn+ssh://svn.arteris.com/engr/dev/appdata/svnrepos/engdocs/trunk
This is illustrated in Figure 13.

Figure 13 – SVN command to check out a database
You can also access the documents by web access:
http://svn.arteris.com
If you are not familiar with SVN, you can find many tutorials online. Please see Chapter 3.0 for more details. Table 3 lists some common SVN commands.

Commands

Description

Syntax

Check Out

Check out the remote repository to a local area

svn co svn+[path]

Add

Place a file in a buffer so that it is scheduled to be checked in to the remote repository during the next commit

svn add [file]

Delete

Place a file in a buffer so that it is scheduled to be deleted from the remote repository during the next commit

svn del [file]

Lock

Lock a file so that no one can modify it in the remote repository

svn lock [file]

Unlock

Unlock a file

svn unlock [file]

Commit

Push all the changes in the buffer to the remote repository

svn ci -m "[message]"

Update

Pull all updates in the remote repository to update the local area (does not restore locally deleted/modified files)

svn update

Revert

Revert a file to its state in the remote repository

svn revert [file]

Table 3 – List of common SVN commands
