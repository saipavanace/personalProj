Signals:

output bist_next;      // Signal indicating that the unit being tested should move to the next state
input bist_next_ack;   // Signal from tested unit indicating it has changed state
input domain_is_on;    // Signal from tested unit indicating its clock/power domain is on. If it is off it will be skipped in the BIST check.

Description:

This interface defines the connection between a block and the Functional Safety Controller in relation to BIST. The domin_is_on signal should be driven from the "asleep" signal from its respective pma on its InterfacePMAControl
