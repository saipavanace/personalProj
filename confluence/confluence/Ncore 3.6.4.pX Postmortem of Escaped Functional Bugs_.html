Recommendations to prevent these bugs moving forward:

  1. Some bugs can be prevented with more through and effective architecture, micro-architecture & test-plan reviews among Design, DV & Arch teams.

  2. Around 5 clean release regressions before the actual release (instead of only one regression with only RTL bug resolved).

    1. “Clean regressions” means we raise our bar to 100% pass-rate regardless of design or DV issues.

    2. We need reduce our regression wall-time from 3 days to 1 day by adding 3x license during release time frame.

  3. Add parameters coverage closure in addition to functional and code coverages closure as part of release checklist.

Reasons we found new bugs starting Q4, 2024:

  1. Started using latest version of VIPs (2024.09) with enhancements in constraints and checks.

  2. We also start analyzing more carefully if Ncore 3.7 bugs can effect Ncore 3.6 version as well.

  3. More customers started using Ncore 3.6 release and found issues in ther own custom configurations.
