2 Testbench description

DVE testbench focuses on verifying functionality of the DVE unit by generating stimulus on input interfaces and monitoring DVE behavior on output interfaces. These stimulus and resulting outputs are then compared with predicted behavior generated by a runtime scoreboard. Predicted behaviors are implemented based on Ncore System Arch spec, DVE micro architecture spec and other related Ncore architectural documents.

2.2 List and description of TB components

DVE testbench consists of below components:

SMI BFM:

An UVM agent that communicates with DVE's Concerto interfaces. DVE tests will start executing DVE sequence on this agent. The purpose of DVE sequence is to act like the rest of the NCore modules and drive DVE's RX ports in response to TX port messages. This sequence has DVM CmdReq generation support. It will drive DVE's CmdReq RX interface to inject DVMs in constrained random manner.
DVE testbench reuses the existing SMI BFM. Test bench wrote a new DVE sequence which generates DVM commands and responds to DVE's smi messages.

DVE Scoreboard:

Scoreboard is the testbench model of DVE RTL. The scoreboard connects to the smi agent in the testbench and observes same stimulus as RTL. Scoreboard uses this observed stimulus to predict RTL's expected behavior(according to the architectural specifications) and compares it against actual observed behavior from DUT.
DVE Scoreboard is a new component.
