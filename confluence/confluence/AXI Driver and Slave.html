AXI Driver

  • This model is to read a STL file and generate AXI Read/Write requests. STL reader already exists on osci-lib/verif directory. We need create a wrapper that instantiates the STL reader, reads the provided file (constructor argument) and generates AXI traffic. The transactions generated must be of type “axi-tlm/axi/axi_tlm.h”. AXI TLM packet already exists in osci-lib/axi-tlm directory. Transaction forwarded to DUT must be aligned to value specified in STL file.

AXI Slave

  • This Model is able to accept AXI read and write requests (ArReq and AwReq) and generate respective responses (RDAT and BRSP). The incoming transactions will be of type “axi_tlm.h” and outgoing responses will also be of same type.
