Overview:

Arteris has promised NXP a second version of the RTL for their design on 01/31/2020.  In order to have adequate testing on the code producing that RTL, the DV group has requested that no unnecessary changes be made after 01/15/2020.  In order to keep changes to a minimum and allow work to proceed on the master branch, we are making a new branch ( release-ncore-3.0-NXP-drop2 ) of each of the repositories and only changes needed for the NXP drop 2 RTL will be allowed on that branch.  A 'Change Control Board' will review all changes.

Change Procedure

Making a change on a release branch is more involved than a regular change to the development ( master ) branch.  The release branch can not be modified directly.  Changes will instead be made on a new feature/fix branch and then submitted with a Merge Request.

The examples below use the following placeholders that should be replaced with settings appropriate for your change:

  • repository: hardware/foofoo
  • JIRA: FOO-42
  • release branch name: release-foo
  • feature/fix branch name: spotter-foo-42

Create a JIRA for the Change

If a JIRA doesn't already exist, create one describing the change needed and justifying why that change needs to be included in the release.

Get Approval for the Change

Before working on a change that is intended to go into the release branch, you should get an agreement from the Change Control Board that the change is needed for that release.

Clone the Repositories Involved

Create a clone of the repository or repositories involved in the usual way.

 

 % git clone git@gitlab.arteris.com:hardware/foofoo


Create a Feature/Fix Branch

Create a new branch for your feature/fix.  The new branch should be based on the release branch.

% git checkout -b myname-foo-42 release-foo
% git push -u origin myname-foo-42

Make and Test Changes

Make your changes and manually run tests to verify that your changes work.

Add, Commit, and Push Your Changes to Your Feature/Fix Branch

% git add <changed files>
% git commit -m <commit message>
% git push

Create a Merge Request

  1. Go to the merge request page of your gitlab repository (http://gitlab.arteris.com/hardware/package/foo/merge-requests)
  2. Click on 'Create New Merge Request'
  3. Choose the 'Source Branch' (myname-foo-42)
  4. Choose the 'Target Branch' (release-foo)
  5. Click on 'Compare Branches and Continue'
  6. Set the 'Title' for your merge request.  Start the title with 'WIP:' if the code is to be reviewed but is not yet ready for merging.
  7. Fill in the 'Description' (optional)
  8. Assign the Merge Request to the person designated as the reviewer for release changes for that repository.

Inform the Change Control Board

If the Change Control Board approves the change after the Merge Request review, the changes will be merged into the release branch.
