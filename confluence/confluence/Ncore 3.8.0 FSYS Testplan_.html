Testname

Description

Stimulus

Check

1

point to point data transfer

Basic Write read sanity test. Master A on Die 0 writes to a memory location, Master B on Die 1 reads it back

Master A → Write(Addr = 0x1000, Data = 0xABCD) Master B → Read(0x1000) → Receives 0xABCD

Data integrity between the write data and read data on 2 chiplets

2

Read-Write with Cache invalidation

Basic coherency check test. Master A on Die 0 reads a line, Master B on Die 1 writes to it.

Step 1: Master A → Read(0x2000) → gets line in Shared

Step 2: Master B → Write(0x2000) → NoC triggers invalidate to A

A's cache line is invalidated before B completes write

3

Conflicting writes

Both Master A (Die 0) and Master B (Die 1) attempt to write the same cache line simultaneously

Step 1: A → Write(0x3000, 0xAAAA)

Step 2: B → Write(0x3000, 0xBBBB)

Final memory value is one of the two writes, and coherence protocol preserved serialization

4

Soft reset test

Die 1 is soft-reset during traffic

  1. Step 1: Generate traffic between Die 0 and Die 1.

  2. Step 2: Reset Die 1 and reinitialize the inter-die NoC.

All links recover, traffic resumes correctly, and no stale data is accepted after reinitialization

5

Incomplete snoop

Master A on Die 0 writes a modified line. Die 1’s cache has a stale copy

Drop or delay snoop response from Die 1

The NoC times out or retries. Eventually, modified data from A is committed to memory or forwarded

6

Out of Order responses

Master A sends 2 reads: R1 and R2

Delay the R1’s response and send R2’s response early

Check for out of order read response behavior {Reordering Needed} ??

7

Deadlock due to cyclic dependency

Create 3-way or 4-way transaction dependency

  1. Die 0 waiting on Die 1 snoop

  2. Die 1 waiting on Die 2

  3. Die 2 waiting on Die 0

System should avoid deadlock
