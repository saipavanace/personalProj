This function returns an array with a pair of integers that are field_start and field_end indexes for the input field that exists in a link packet description.

It is called as follows:

var return_data =  extractFieldBitIndexes(link_width,link_packet_def,field,offset, type,beat);

link_width : width of the link in bits

link_packet_def : packetDesc

field: text string used to match against field names.

offset: Optional argument. Default = 0. An integer that is subtracted from the field_start before it is returned. This is useful for example to access field indexes in the header-portion only i.e. assuming the last bit of the header aligns with index 0.

type: Optional argument. Default = "X" (Unknown). 'type' has to be specified as one of the following values "H" (Header), "F"(Fixed), "C" (Copy/Clone) or "N" (NULL)

beat: Optional argument. Default = -1. Specifies the beat number in the packet at which the required field is to be searched for.


This function relies on the function HW-SYM getFieldPosAndBeatInLinkPacket, and the match_style is computed based on the type of the field.

If type is not specified, the function sees the default value 'X', in which case the type of the field is re-computed inside the function based on the input field name (argument 'field') as follows:

type = "N" for names that match NULL*,
type = "H" for names that match H_*,
type = "C" for names that match C_*,
Throws an error when names don't match any of the above.

if type is specified, then it is used to compute match_style for the HW-SYM getFieldPosAndBeatInLinkPacket as follows:

For type = "N", match_style = "*"
For type ="H", match_style = "exact"
For type = "F","C" match_style = "num"

if beat is not specified, the function sees the default value of -1. In this case, the beat is assumed to be the beat that has the first instance of the field. The exception is when the type="N" and field is "NULL_D" in which case, the beat is always assumed to be the first beat i.e. beat=0.

if beat is specified, the specified beat is used to search for the input field (field name) in the specified beat of the packet.

if the field specified isn't found in the specified beat of the packet, the returned value is [-1, 0].
