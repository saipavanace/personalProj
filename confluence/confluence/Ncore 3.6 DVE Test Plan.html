2 Introduction

DVE is the central block in Concerto System for Distributed Virtual Memory (DVM) transaction and is the point of serialization for all the DVM requests sent to it. DVE generates appropriate protocol snoops to be broadcast to the required snooping agents. DVE also collects all the snoop responses from various snooping agents and forwards the summary result to the requesting agent.

DVE supports following features:

  • The DVE is the Point of Serialization for DVM requests.

  • The DVE contains a System Coherency manager engine (SysCo) for obtaining status from initiator agents.

  • The DVE contains Even Broadcaster, to broadcast events generated within the Ncore or received to Ncore to all the active agents.

  • The DVE contains a trace and debug accumulator receive and store trace debug packets.

  • DVE TB has most of the checks within dve_sb.svh and dve_sb_txn.svh.

  • The dve_sb has three queues that are of dve_sb_txn data type.

    • m_ott_q : It is a queue of all pending cmd_req/snp_req that are needed to complete.

    • m_sys_q : This queue keeps track of system messages.

    • trace_pkt_q : This queue keeps track of trace information.

  • dve_sb_txn is a calss object that keeps trac of all transactions and construct expected transaction packets and send to scoreboard for checks.

4.2 System Events

4.2.1 Stimulus Description

  • SysEvent requests and responses uses system messages. These system messages are used to convey operating state information or state transitions for the agents or asynchronous events within the endpoints of the Ncore domain.

  • DVE has Event Broadcaster, that receives SysReq.Events from iAIUs (Initiator AIUs) and broadcasts to all the active AIUs (Coherency AIUs).

  • DVE TB generates and drives the SysReqsEvents to DVE as iAIUs.

  • Now DVE also sends SysReq (Event Broadcaster).
