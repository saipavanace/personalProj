Here is the way to achieve it via macros. Below is an example transaction base item and extended classes and the methods to be called.

The mbp_packet is the base class which has the members p_args,p_args_min,p_args_max (required for the randomization method) and the macro defined. Also there is a set_attr function which populates this queues based on +args which needs to be called in the pre_randomize calls.

The extended classes ext_mbp_packet and ext1_mbp_packet only have to call the pre_randomize method with new member attributes and need to add the `SET_CONST for them.

With the +arg methods you can control per type or per instance basis , you can add ranges with min and max. Eg:

"+m_packet0_m_addr=2000" "+m_packet1_m_addr=2200" "+ext_mbp_packet_m_addr=2210" "+ext1_mbp_packet_m_addr=2220" "+err=1" "+m_packet0_m_data=0" "+m_packet1_m_data=10" "+m_packet2_m_data=20" "+m_packet3_m_data=30"  "+m_addr_min=0" "+m_addr_max=3000"

 

To incorporate this in respective projects use the p_args* attributes,macro `SET_CONST and the method set_attr in the base class and add the calls for each random attribute.
