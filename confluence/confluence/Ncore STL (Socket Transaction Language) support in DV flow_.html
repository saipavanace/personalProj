How to check-out and run simulations:

Steps to take checkout :

            Step 1: mkdir <your_work_dir>; cd <your_work_dir>;

            Step 2:  /engr/dev/tools/script/setup_concerto

            Step 3: source SOURCEME.csh ncore3.4 (or 'source SOURCEME ncore3.4' if your $SHELL=bash)

            Step 4: cd $WORK_TOP

 

To run simulation :

Runcmd : runsim -e fsys -c hw_cfg_43 -t ioaiu_random -co "'+define+USE_STL_TRACE +define+STL_FILE_FULL_PATH=/scratch/pzare/stl_files_path/'" -w

Defines :

USE_STL_TRACE  : Enables STL flow in the TB

STL_FILE_FULL_PATH=/scratch/pzare/stl_files_path/  : This define will give user control for directory of stl files. Required to provide directory path for reg.stl and accX.stl { axi0_if -> acc0.stl, axi1_if -> acc1.stl……}

Plusargs : These plusargs added to give support for enable/disable register or traffic by stl

-so "disable_stl_csr"

Provide in cmd line to disable register configurations through the stl

-so "+normal_axi_txn"

Provide in cmd line to disable stl traffic/stimulus on the axi interface.

DV Testbench Environment Details:

Register configurations :

DV has support of register configurations through the reg.stl file

Task for stl paraser - concerto_fullsys_test::stl_csr_write();

It will read complete reg.stl file and will write the csr model on the ioaiu0_axi_if.

It will take reg.stl file provided by +define+STL_FILE_FULL_PATH from the cmdline.

Reg config file : $WORK_TOP/dv/full_sys/tests/concerto_fullsys_test.svh 

Stimulus Drive : 

After completion of the register configurations, DV has support of STL parser to read the  stimulus from provided stl file in the form of burst read, burst write, idle, wait and comments.

There will be one parser associate to the single channel.

It will read all the information from stl file and will store in the variables.

There is support in the axi_master_pipelined_seq to collect the data from these variables and to drive it on the respective interface, channel with no input stl file will not be driving any traffic.

User has to provide stl files from cmd lines +define+STL_FILE_FULL_PATH

Naming conventions in the parser are as suggested – acc0.stl,acc1.stl,acc2.stl,acc3.stl……

Parser Files : $WORK_TOP/dv/common/lib_tb/axi_stl_traffic.sv

Stimulus driving sequence :$WORK_TOP/dv/common/lib_tb/axi_seq.sv (axi_master_pipelined_seq)
