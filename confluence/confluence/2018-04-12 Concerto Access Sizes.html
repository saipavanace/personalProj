 ASize, ALength 
  • Any transaction entering the network must be translated by the AIU to something Concerto supports. Currently Concerto only supports transactions with size a power of 2, so AIU will translate to the smallest power of 2 aligned access that can encompass the original transaction.
  • A read to memory that was larger than need would not be harmful (other than possible extra bandwidth used) but a read to IO device could have detrimental side effects. To avoid this, the target agent must translate the request back into it's original transaction size (using ASize,ALength in mpf1 field) when send on the downstream AXI bus. That way, only the exact bytes needed are requested from the IO device.
 Responsibilities 
  • DII must translate read requests into native bus sizes, read from the IO device, and then repackage back into concerto message wSize transaction, aligned and padding as necessary.
  • DMI can translate, but is not needed since only memory is attached, the only loss being unnecessary bandwidth on the downstream bus.
  • Someone needs to handle the read masking, telling AIU what are the active bytes in the read. Since the AIU initiated the transaction, it is assumed it has the necessary info to figure it out itself.
