  • Eg: compile command

  • Eg: run command
    node $WORK_TOP/../scripts/qrsim.js -e chi_aiu -q -n hw_cfg_20_xtrap -t chi_aiu_bringup_test -p +random_txn_test,+k_num_snoop=500,+en_excl_txn,+k_targ_src_id_chk_en,+UVM_VERBOSITY=UVM_NONE,+UVM_MAX_QUIT_COUNT=1 -R 990526160 -X xprop_init_disable_time=10ns,xprop_error_limit=10000,xprop_assert_limit=1,xprop_exclude_file=$WORK_TOP/dv/chi_aiu/xprop_exclude_file/hw_cfg_7_exclude_file

 

Compile time options:

xprop_mode=<type> - valid values are xtrap, resolve and pass. This option specifies the mode in which the xprop simulation is run.

 

Run time options:

xprop_init_disable_time=<time_duration> - by default this number is 0. This option specifies the duration at the beginning of the simulation for which the xprop is disabled (usually till reset is released xprop should be disabled. You can set this based on when the reset is released)

 

xprop_error_limit=<n> - by default this number is 5. The simulation will terminate after reaching 5 xprop errors. In order to let the simulation run to see all the error in a single run you can set this to big number.

 

xprop_assert_limit=<n> - by default this number is 5. If a same signal is repeatedly becoming 'x' you can control how many times it is displayed in the log file by setting appropriate value. You can set it to 1 for debugging.


xprop_exclude_file=<exclude_file> - This option is used to specify the exclusion file for this xprop simulation. Provide the path and filename for this option (see example above).


Note:

1) Use xprop_error_limit and xprop_assert_limit options only for debugging purpose. In the regression do not set these options.

2) xprop_exclude_file optoin: You will have to create separate exclusion file for each configuration. The exclusion should be done only after checking with the designer

XPROP Exclusion:

1. You can add the following in xpropdofile:

  • assertion enable -off -filter XPROP_FF_31  /top/u
  • assertion enable -off -filter XPROP_FF_* -r /top/u   [-r for recursive]


2. You can also exclude specific module/instance/file:

XPROP EXCLUSION 1: xprop exclusion by design hierarchy

  • -xprop,mode=resolve,report=error <This applies to entire design>
  • -xprop,mode=none,object=<module or hier. Instance that you want to remove from xprop>
  • e.g. -xprop,mode=none,object=/tb/u1” at vopt, no xprop assertion will fire for /tb/u1

XPROP EXCLUSION 2: xprop exclusion by design file lists

  • -xprop,mode=pass,report=error <This applies to entire design>
  • -xprop,mode=none,filelist=<list of file to exclude from xpropx
  • e.g. -xprop,mode=none,filelist=ex_list.f no xprop assertion will fire for files listed in ex_list.f

XPROP EXCLUSION 3: xprop exclusion by HDL inline attributes

For Verilog

                             (*xprop_off = "TRUE"*)

                             module module_name

                             ……

                             endmodule

For VHDL

                            architecture arch1 of entity1 is

                            SIGNAL g_valid : STD_LOGIC;

                            attribute xprop_off : string;

                            attribute xprop_off of arch1 : architecture is “TRUE” ;

                            begin

Please refer to User Guide and Command Reference Mannual for more detail.

Example exclusion file:

assertion enable -off -filter XPROP_COMB_916 /tb_top/dut/unit/ncore3_addr_map
assertion enable -off -filter XPROP_COMB_951 /tb_top/dut/unit/ncore3_addr_map

Example xpropdofile (This is automatically taken care by the script now):

xprop assertlimit 1;
assertion enable -off -filter XPROP_COMB_756 /tb_top/dut/unit/ncore3_addr_map
xprop disable;
run 10ns;
xprop enable;
