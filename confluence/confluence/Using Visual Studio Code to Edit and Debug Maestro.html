The following instructions assume a clean clone of Maestro.


Visual Studio Setup.zip

  1. Install VS code, for example according to the instructions at https://code.visualstudio.com/docs/setup/linux :

sudo rpm --import https://packages.microsoft.com/keys/microsoft.asc
sudo sh -c 'echo -e "[code]\nname=Visual Studio Code\nbaseurl=https://packages.microsoft.com/yumrepos/vscode\nenabled=1\ngpgcheck=1\ngpgkey=https://packages.microsoft.com/keys/microsoft.asc" > /etc/yum.repos.d/vscode.repo'
yum check-update
sudo yum install code 

2. Launch it. These extensions may be useful, you are encouraged to install them:

  • Better C++ Syntax v1.17.2

  • C/C++ v1.19.1

  • C/C++ Extension Pack v1.3.0 (includes some of the others in this list)

  • C/C++ Themes v2.0.0

  • CMake v0.0.17

  • CMake Tools v1.16.32

  • Git History v0.6.20

  • GitLens – Git supercharged v14.2.1

  • Remote – SSH v0.102.0

  • Remote Explorer v0.4.0

  • Tcl v0.4.3

3. Place the attached files in a new .vscode directory under maestro/:

  • c_cpp_properties.json

  • settings.json

  • tasks.json

  • launch.json

4. Change to: cd /path/to/maestro

5. Launch: code . &

Useful keyboard shortcuts:

  • Shift-Ctrl-B to launch a build

  • Ctrl-P open file by typing the filename

  • F5 to run debugger

  • F10 to step over

  • F11 to step into

  • Ctrl-G to line number (type line number at top)

  • Ctrl-Shift-F or magnifying glass in left tool bar to global search with filters


6. Upon first invocation of the build, you will be asked to pick a compiler (Ncore 3.8 needs gcc 8.3). At each invocation, you will also be asked for a build target; the pre-packaged ones are:


build is equivalent on the command line to cmake build_debug

build_nocmake is equivalent on the command line to cmake build_debug_nocmake

These shortcuts can be edited, or new ones hand-defined, in .vscode/tasks.json.
