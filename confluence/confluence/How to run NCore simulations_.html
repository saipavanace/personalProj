Making NCore 3.2.0 Space (Release made on May 17th):

STEP 1: mkdir <your_work_dir>; cd <your_work_dir>;

STEP 2:  /engr/dev/tools/script/setup_concerto

STEP 3: source SOURCEME.csh ncore3.2.0 (or 'source SOURCEME ncore3.2.0' if your $SHELL=bash)

NOTE: Nothing can be checked in

Making NCore 3.2.1 Space:

STEP 1: mkdir <your_work_dir>; cd <your_work_dir>;

STEP 2:  /engr/dev/tools/script/setup_concerto

STEP 3: source SOURCEME.csh ncore3.2.1 (or 'source SOURCEME ncore3.2.1' if your $SHELL=bash)

STEP 4: cd $WORK_TOP

STEP 5: $WORK_TOP/dv/scripts/refresh_rtl

STEP 6: source SOURCEME_image.tcsh (or 'source SOURCEME_image' if your $SHELL=bash)

To check in any changes to hw-ncr repo (Ncore3.2.0/develop) for ncore3.2.1 project do normal git add, git commit and git push (Please qualify your changes locally atleast by running $WORK_TOP/dv/scripts/checkin_test -n -l -o -p).

NOTE: we have to do refresh_rtl (STEP 5 and STEP 6) because the maestro image is not built yet with all the bug fixes that are being made.

Making NCore 3.4 Space:

STEP 1: mkdir <your_work_dir>; cd <your_work_dir>;

STEP 2:  /engr/dev/tools/script/setup_concerto

STEP 3: source SOURCEME.csh ncore3.4 (or 'source SOURCEME ncore3.4' if your $SHELL=bash)

STEP 4: cd $WORK_TOP

STEP 5: Run sanity simulation using this command "runsim -e fsys -c hw_cfg_3 -t directed_wr_rd -i 1 -w"

Following Steps 5 & 6 are not mandatory - they are only required when local rtl, CPR, or Customer TB changes are made

STEP 6: $WORK_TOP/dv/scripts/refresh_rtl

STEP 7: source SOURCEME_image.tcsh (or 'source SOURCEME_image' if your $SHELL=bash)

Running full regression suite

STEP 8: cd $WORK_TOP/dv

STEP 9: $WORK_TOP/dv/scripts/checkin_test -n -o -l -p

Pushing New Files (3.3 and 3.4)

STEP 10: ./push_ncr (ensure you have setup your Jenkins account as specified in Using the Jenkins REST API)

Making NCore 3.6 Space:

STEP 1: mkdir <your_work_dir>; cd <your_work_dir>;

STEP 2:  /engr/dev/tools/script/setup_concerto Ncore3.6/develop

STEP 3: source hw-ncr/dv/scripts/setup_ncore36.csh (or 'source hw-ncr/dv/scripts/setup_ncore36' if your $SHELL=bash)

STEP 4: cd $WORK_TOP

STEP 5: Run sanity simulation using this command "runsim -e fsys -c hw_cfg_3 -t directed_wr_rd -i 1 -w"

Following Steps 5 & 6 are not mandatory - they are only required when local rtl, CPR, or Customer TB changes are made

STEP 6: $WORK_TOP/dv/scripts/refresh_rtl

STEP 7: source SOURCEME_image.tcsh (or 'source SOURCEME_image' if your $SHELL=bash)

Running full regression suite

STEP 8: cd $WORK_TOP/dv

STEP 9: $WORK_TOP/dv/scripts/checkin_test -n -o -l -p

Pushing New Files

STEP 10: ./push_ncr (ensure you have setup your Jenkins account as specified in Using the Jenkins REST API)

Setting up image of a particular Maestro build_no:

dv/scripts/refresh_rtl -b <build_no>

for environments already pointing to existing Maestro images, you can use dv/scripts/refresh_rtl -u option to upgrade to the latest stable version

One time setup required before using the ./push_ncr script:

Using the Jenkins REST API#Settingupyour.netrc

STEP 3: ./push_ncr

To run a single test, open the testlist for the given env - hw-ncr/dv/<env>/tb/testlist. The file is organized into sections, one for each config.

STEP 1: Run the compile command, a node command at the top of the section, to generate the config.

STEP 2:  Run the required test / simulation by running the node command with the -t option in the same section. Optionally add a -r '-qwavedb=+signal+class=foo+transaction' at the end of the command to generate waves.
