Discussion items

 

Requirements

  • Support people using Platform Architect (All current customers)
  • Support plain old OSCI SystemC
  • Interoperate with FlexNoC models.

 

Original plan:

  • Only build pin-to-FT-XXX adapters, since a generic FT-* interface can be connected to a generic TLM interface, the functionality is ratcheted down to basic TLM.
  • This would allow FT-level interoperability inside PA, and TLM interoperability outside of PA, with only a single implementation on our part.
  • We had intended to work with SNPS with this.  The benefit to SNPS was that it would give customers some impetus to using PA instead of OSCI SystemC.

Ideally:

  • Generate one set of adapters that corresponds to FT-semantics.
  • If we can have a package that allows our FT-adapters to talk to generic FT interfaces, that's excellent.

 

Desired functionality

  • Outside of PA, in OSCI SystemC, Ncore/FlexNoC interoperability would be provided by Ncore FT interfaces connecting to the FT-to-FlexNoC extensions adapter to FlexNoC.
