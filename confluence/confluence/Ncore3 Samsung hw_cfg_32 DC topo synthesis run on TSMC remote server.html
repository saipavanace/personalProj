Step 2: Copy the required target memories

cp -r /scratch/rsalamat/memories/hw_cfg_32/memories .


Step 3: Run Aria to generate the synthesis script that will be put in output/synthesis/synopsys/scripts

node /home/boon/run_aria_ncore3_dc_topo.js


This is the contents of the /home/boon/run_aria_ncore3_dc_topo.js :

var {genDcSynScript} = require('/engr/dev/releases/utils/aria/latest/aria-linux-x64/resources/app/index');
genDcSynScript({
topDesign:'gen_wrapper',
cprInterfaceDirs: `${process.env.REPO_PATH}/assets/interface`,
techNode: 'tsmc7',
topoMode: true,
incrementalOpt: true,
clockUncertainty: 0.15,

rtlWrapperDir: 'memories',

hardMacroDbs:  'memories/libraries_ssgn/*_lib.db',

ulvtPercentage: 5,
compileCommand: 'compile_ultra -spg -no_boundary_optimization -gate_clock',
bottomUpSynthesis: true
});



NOTE: To do a quick check synthesis using zero-wire-load synthesis, i.e. read in the design and perform checks without doing synthesis on our server, copy over the Aria script and add this line to it:

topoMode: false,

checkOnly: true

Then re-run Aria with the copied-over-and-modified script.


Step 4: Create a tar file

cd ..

tar cvf hw_cfg_32.tar hw_cfg_32


Step 4: Submit the tar file to Jenkins TSMC Synthesis to run synthesis; see the link below to check the progress and to get back the synthesis results.

http://jenkins.arteris.com:8080/job/TSMC_Synthesis/


The synthesis build on Jenkins runs the synthesis by calling the synthesis Makefile using this command line:

cd output/synthesis/synopsys/top; make -j4 -f ../scripts/Makefile


**NOTE**: Maestro/Aria synthesis flow assumes:
output/synthesis/synopsys/scripts are the synthesis scripts directory
output/synthesis/synopsys/top are the synthesis run directory

According to the Maestro/Aria flow, user is supposed to
  cd output/synthesis/synopsys/top
  make -f ../scripts/Makefile
