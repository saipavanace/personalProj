How To Use:

$WORK_TOP/tcl_gen.v3/randomizer.tcl -h

Usage :  randomizer.tcl -n <number_of_configs> -b <base version> -s <user_suffix>

-n : number of configurations that user want to create (optional, default value is 5).
-b : base version (optional, default value is picked randomly from available bases).
-s : user suffix added at the end of configuration name (optional).
-h : print this message.

example : tclsh randomizer.tcl -n 10 -b base2 -s striping
          tclsh randomizer.tcl -b base3
          tclsh randomizer.tcl -h


References:

  1. More bases details are at $WORK_TOP/parameters/features_base_x.xlsx
  2. User settable parameters are at $WORK_TOP/parameters/user_params.csv



BASE 0:

Features:

  1. 4x4 Ring topology with 4 switches.
  2. Randomize memory map 
  3. With the exception of Striping, the followings are supported: Sram Fifo, Fifo Rate, Reordering Buffer and Firewall are all supported.
  4. CSR support has also been added.

Randomization scope:

       Randomization support will soon be added for the following

  1. ATU block Sram fifo parameters.
  2. Support of reordering buffer


BASE 1:

Features:

  1. 4x7 xbar topology with 2 switches
  2. Memory map :fixed
  3. No special features (no striping, no pmon...)

Randomization scope:

  1. Socket parameters
  2. ATU block pipe parameters.

BASE 2:

Features:

  1. 4x7 mesh topology with randomized nodes position
  2. Memory map :fixed
  3. No striping
  4. Support of reordering buffer

Randomization scope:

  1. Socket parameters
  2. ATU block perf parameters.
  3. ATU block pipe parameters.

BASE 3:

Features:

  1. 4x7 Xbar topology with 4 switches
  2. Randomize memory map with base/offset and memory constraints integration.
  3. Stripping, interrupt, and CSR enabled.
  4. Support of reordering buffer

Randomization scope:

  1. Socket parameters
  2. ATU block perf parameters.
  3. ATU block pipe parameters.
  4. Buffered switch parameters.
  5. Switch type randomized (buffered , non-buffered)
  6. timeout errors and exclusive enabled.

BASE 4:

Features:

  1. 4x9 mesh topology.
  2. Randomize memory map with base/offset and memory constraints integration.
  3. Support multiple clocks (6clocks).
  4. Stripping, interrupt, and CSR enabled.
  5. Support of reordering buffer

Randomization scope:

  1. Socket parameters
  2. ATU block perf parameters.
  3. ATU block pipe parameters.
  4. Buffered switch parameters.
  5. Switch type randomized (only buffered switch )
  6. timeout errors and exclusive enabled.
  7. ATP parameters (only in ATUs)
  8. Clock Async adapter param
  9. Dw adapter params
  10. Header network (only parallel and serial)

BASE 5:

Features:

  1. 5x25 mesh topology. 
  2. Enable Stripping, interrupt, and CSR features. 
  3. Memory Map randomization with overlapping region
  4. Striping group configurations are implemented (2, 4 , 8) 
  5. 8 memory modes supporting 
  6. Many interleaving group created 

Randomization scope:

  1. Socket parameters
  2. Memory Map params with large ATUS (Base Addr and memory size) taking into account all the constraints (see Symphony R1 Parameter and Specification 4.5.9 Memory map constraints )
  3. “memoryBaseOffset” randomization to address larger target address region than initiator. (This parameter will be applied to initiator)

BASE 5p1:

Features:

  1. 5x5 mesh topology. 
  2. Enable Stripping, interrupt, and CSR features. 
  3. Support 2 memory modes
  4. Memory Map concatenation feature (if user would want to have larger address map size then the base address, it must be partitioned into small parts and then should be concatenated.)

Randomization scope:

  1. Socket parameters (with Psel > 1)

  2. ATU block perf parameters.
  3. ATU block pipe parameters.
  4. Buffered switch parameters.
