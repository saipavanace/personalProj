Modules for the PA Tiger Team testers

module load pa-tiger

Usage:

module load pa-tiger

What does pa-tiger load?

The pa-tiger module will set all of the environment variables needed to run

  • The latest version of Synopsys Platform Architect (pct), updated with the latest Synopsys drop of the ARTERIS_Ncore_BL bus library, and patched with the SBLGCCI_BL path for write interleaving.
  • The latest RC version of Ncore, updated with the latest build of Angelo and Alfredo.
  • FlexNoC 3.5.5

The module load of pa-tiger should set up everything such that you don't need a special cwr.lib file.  Everything that was previously added by the cwr.lib file is now present in the installed version of Platform Architect.  If you do have a cwr.lib then any settings for ARTERIS_Ncore_BL or SBLGCCI_BL will override the settings provided by the pa-tiger module.

NOTE: You may have to remove your ~/.platform_creator_preferences file and/or your ~/.synopsys directory to get PA to stop trying to restore the settings that were previously used for those libraries.

How does it work?

Requesting a module load of pa-tiger will load the default version of pa-tiger as specified in the <MODULEPATH>/pa-tiger/.version file.  That module will first print the version that is actually selected, e.g.:

Using module pa-tiger/drop1_models-155

The default version of pa-tiger is updated whenever there is a new drop of the Synopsys ARTERIS_Ncore_BL bus library or when there is a new build of Angelo, so if you might want to recreate the same environment later save the module name reported and do a 'module load' with that name instead of using the default.

module load pa-tiger/ultra

This module is not yet available.  When it is available, it will set everything up for the usage of a PA-Ultra updated with the latest Synopsys drop of the ARTERIS_Ncore_BL bus library.

Modules for the PA Tiger Team developers

When modifying and building new versions of Angelo, you need to test your own Angelo coderather than the code that resulted from the last Bamboo build.  The module system can still be used to set everything up for you.  This is done by loading PA or PA-Ultra and loading ncore/by_path to set environment variables to point to your Ncore/Angelo build.

module load pa-tiger/pa

Usage:

% module load pa-tiger/pa

What does pa-tiger/pa load?

  • The latest version of Synopsys Platform Architect (pct) that contains the latest Synopsys drop of the ARTERIS_Ncore_BL bus library, and is patched with the SBLGCCI_BL path for write interleaving installed.

module load ncore/by_path

Usage:

% setenv CONDUCTOR_HOME <your ncore installation which contains your exported 'pa' files>
% module load ncore/by_path

Usually you will be using ncore/by_path along with PA or PA-Ultra.  You can load both at the same time, e.g.:

% setenv CONDUCTOR_HOME <your ncore installation, e.g. $HOME/workspace/my_ncore>
% module load pa-tiger/pa ncore/by_path

 

What does ncore/by_path load?

  • Your Ncore installation, which is expected to have
    • a bin directory with a copy of alberto
    • a pa directory containing the exported pa files from your build of angelo
  • FlexNoC 3.5.5
