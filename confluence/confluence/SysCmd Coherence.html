Overview

The purpose of these SYSreq messages is to propagate system related information between agents or functional units within the system. The SysCo protocol is a 4 state handshake protocol implemented by ARM processor DSU. The purpose of the protocol is to attach and detach the agent to the coherent domain. Sysco test bench is part of the bigger IOAIU test bench. Scoreboard and stimulus are part of the IOAIU environment. Sysco protocol check looks at the Concerto messages (from AIUs) and response queue in the scoreboard and checks it against the predicted Sysco state.

Stimulus Descriptions 

The random stimulus involved in sysco testing is those of the sequence:axi_master_pipeline_seq. As a part of sysco testing, the AMBA traffic (coherent and non-coherent) are generated using 4 instance of axi_master_pipeline_seq, each served to run during a specific phase of sysco:-

  • before attach : constrained to generate only non-coherent traffic

  • after attach : constraints to traffic follows the randomization from the env as there is no restriction

  • after detach: constrained to generate only non-coherent traffic

  • after reattach: constraints to traffic follows the randomization from the env as there is no restriction

Also, there is a snoop seq running in the background that stresses the snoop transaction generated by the BFM to the target AIUs. Since the randomization parameters of these sequences shall already be covered in the appropriate test plans, please refer to those for details.
