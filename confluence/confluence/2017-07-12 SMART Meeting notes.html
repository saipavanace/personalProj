30minSecurityJohn Coddington (Deactivated)
  • Security is not just firewalls
  • 4k boundaries for MMU access enforcement (configuration time)
  • Hypervisor, debug and virtual network regions; the r/w accesses will change between the 3 regions
  • virtual networks and QoS - prevent interference between VNs
  • Trustzone support - what to do when there is mix of protocols? (e.g. OCP initiator). Current plan is to use user bits and map them to the ARM domain
  • if someone is trying to attack the system, plan is to use perfmon, and trigger an interrupt to inform system SW
  • Need to protect the debug system as well
  • Hypervisor is the most trusted SW
  • We have some partnership with Rambus / CRI on working with certificates from HW all the way to DRMs, application level SW
  • Configuration SW - on a system level view, set up behaviors for different regions, and have SW to set them up
  • will have pre-set firewalls for common customer use case
  • memory maps restrict r/w permissions? current plan is to use firewalls to do so
  • firewall - header cycles: if there are too many header cycles, need many pipelines in firewall - SW needs to be able to flag that - either restrict them from doing it or give a warning
  • DoS: firewall sits right next to / inside ATU and drop the access
  • NoC to NoC - how do you mark the original source of an error? There will be global source ID for composition of NoCs
15minConfigJohn Coddington (Deactivated)
  • APB used to r/w registers
  • All Symphony units that can have register accesses will have APB interface
  • Ncore - allow OCP interface for register accesses
  • SW needs to hook up the config network automatically
  • IDs for targets - may be collapsed for better QoR. If everything has unique ID, then will need a separate ATU per target
  • architecturally, 256B minimum address space per unit; implementation: 4k address space minimum per unit. Need to consider the case where there are 400 ATUs, for example.
    • But if IDs are collapsed, then architecturally multiple units can share the 4k space
  • TACHL flow for CSR generation
  • IPXACT register description should be able for customers to publish

20minErrors and interruptsJohn Coddington (Deactivated)
  • Single event, single interrupt source
  • First receiver/generator(?) of the error is the block that generates the interrupt - still under consideration
    • but lose the info of where the exact error come from
  • how about decoupling error detection from generation/reporting interrupt? Advantage: the error detection module can be shared in the topology
  • ATU, fabrics will have accumulation register and masking register that produces 1 interrupt
  • Domains will have accumulation, masking, domain masking register that produces one interrupt
  • Added hier will have accumulation register / masking register that produces 1 interrupt
30minA Day in the Life of Transaction RequestDavid Brown (Deactivated), Shailendra Aulakh (Deactivated)
  • Based on prototype done by Tom & Dave
  • when there are bubbles in read data, separate packets will be formed. The purpose is to not hold up the network when there are bubbles in read data - but how does it work when the header doesn't contain packet size information?
  • When there are bubbles in write data, the behavior is different - we do not form separate packets, as there may be deadlock scenarios on the write side.
