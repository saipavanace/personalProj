I’m in the process of updating the micro-architecture document in the meantime here is a quick answer to your questions:


> Also, if you have checked-in JSON files for ACEAiu, AceLiteAiu and AceLite-e Aiu’s, please point me to those files?

Files at rtl/ioaiu/src/ioaiu_config*.json


  • eSkid: What is this parameter for? Is this a user Visible parameter? If so, what are its limits?

It might be useful when we do timing and performance analysis to disable some of the hardware features. For now assume it is always 1.

  • nOttDataBanks: Is this a user Visible parameter? Meaning is it always tied to 1.

This might be useful to improve performance. We should verify at least 1 and 2 setting. Default value is 1.

  • PriSubDiagAddrBits: For Ncore-v3.0 or Ncore-v3.1, do we support this parameter to hold values? If so what is the length?
  • SecSubRows: For Ncore-v3.0 or Ncore-v3.1, do we support this parameter to hold values? If so what is the length?

Yes, these are used to select the DCE. The length and decoding is up to the user.

  • eAge: What is his parameter for? Is this User Visible parameter? If so, what are its limits? Is this common parameter for all Ncore units?

This enables the “Age” feature. It can be either 0 or 1. The Age feature adds area and affects timing for little return (in my opinion) so I provided the ability to remove the feature. We should verify both 0 and 1 values. The other unit that might have this is chi-aiu but I’m not sure if it exists there (or needs to exist).

  • eStrave: This enables Starvation detection. This can be enabled even without QoS, i.e. interested in knowing whether a request hasn’t been serviced in a while and to give it preferential treatment while arbitrating. We should test both 0 and 1 setting.
  • QosMap: This defines how QoS values are mapped into buckets. If QoS is not enabled, then this parameter is [].

The following map indicates that QoS values 0 and 1 map into the highest priority bucket 0; and all other QoS values are in bucket 1.

    "QosMap": ["16'h3", "16'hfffc"],

We should test no QoS, 1, 2, 3, and 4 buckets. Mapping of QoS values onto buckets can be random.


Please continue to point out missing information in the document so I can add it.


Nabil
