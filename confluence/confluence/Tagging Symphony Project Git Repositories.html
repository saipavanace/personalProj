Hello All,

 

I have added the create_tag script in hw-sym to create tags in all the Symphony (hw-sym,hw-dv,hw-lib & hw-scripts) repos.

A. Tag your repository:

After you commit & push your changes in any Symphony repos, run the following command:

% $WORK_TOP/scripts/create_tag.sh -a <tag_name> -m “your tag message”

The script will add tags for all the repos, please make sure you don’t have any uncommitted changes in either repos.

B. Checkout Tagged repository:

To checkout a particular tag simply use the setup_sym script with an additional “-c” option

% /engr/dev/tools/script/setup_sym -c <tag_name>

 

To reproduce the same scenario you simply need to add a <tag_name> in the create_tag script and use setup_sym to checkout a new area with option -c using same <tag_name>

I’d like to point out few things to be careful of for successfully tagging both our git repos:

  1. Make sure you don’t have uncommitted changes in either repos (hw-sym & hw-dv are the most moving ones to take care of). The script will fail to push the tags if so.
  2. Add tag message along with tag to track it down in case of some mess up:
    1. create_tag.sh -a <TAG_NAME> -m “your tag message”.
  3. Check for any error message while running the create tag script.
  4. You can see the tags in a repo by running following command:
    1. git tag
    2. git show <tag_name>

Please let me know if you have any questions.
