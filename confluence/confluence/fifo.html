Descriptions

This FIFO is used as an elastic store for data.

This FIFO can be configured as an empty, pass through FIFO when the parameter zero_depth is set to 1.

This FIFO can be configured as an bypass-able FIFO when the parameter bypass_mode is set to 1.

When the number of entries in the FIFO is greater or equal to 4, then "head of queue" optimization logic is generated such that the FIFO output comes directly from flops without any combinatorial logic in between.
