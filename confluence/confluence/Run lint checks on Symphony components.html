This flow describes the steps for running the LINT flow in Symphony, which was ported from CodaCache. The flow generates RTL based on an input configuration  (specified by a JSON file), and then runs checks using Spyglass lint tool. This is useful for design engineers, to ensure their design is free of lint errors (width mismatch, undriven nets etc.). It also aids verification engineers to identify such bugs before simulation.

Step-by-step guide

In short, the flow uses the qrsim.js script with the switch -L. Below are the detailed steps for running the flow.

  1. mkdir ie1_top
  2. cd ie1_top
  3. Run: /engr/dev/tools/script/setup_sym -d master
  4. source SOURCEME.csh
  5. Edit $WORK_TOP/scripts/pkgMap.json:
    1. Add the component_name you wish to run link checks on. 
    2. Specify the rtl,lib, and jsLib paths.
  6. Now you can run lint by using -L switch with the script qrsim.js by running the following command:
    1. node $WORK_TOP/../../scripts/qrsim.js -e component_name -a -n config_name -s JSON_FILE_PATH -L
    2. The component_name in the above command should match that specified in pkgMap.json. Some other options you could add and their meanings:
      1. -a is for RTL compile only.
      2. -c is for RTL+TB compile.
      3. -q is for Questa simulation.
      4. -h is for help and to list all other options.
  7. As an example, for running lint checks on DW Adapter, you could run the following command:
    1. node $WORK_TOP/../../scripts/qrsim.js -e dw_adapter -a -n hp2hpw2n -s $WORK_TOP/../hw/blocks/dw_adapter/rtl/params/hp2hpw2n.json -L
  8. Reports: Go through the reports log: WORK_TOP/../debug/component_name/config_name/lint.log and review all errors and warnings. This is the spyglass moresimple.rpt report. By default all the other reports are deleted after the run. If you need to view the other reports, you need to edit the $WORK_TOP/scripts/lint_script and comment out the following line: rm -rf $dest_lint_log/lint.$config_name in the script.
  9. Specifying Waivers: To waive off any lint warnings or errors, add the waivers in the $WORK_TOP/scripts/waivers.swl and re-run the qrsim.js script to re-generate the report.
