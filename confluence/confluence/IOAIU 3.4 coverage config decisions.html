I created a list of all the possible configs that might be used in IOAIU 3.4 coverage. The first seven are the ones being used today - the “original_7”. Below that are additional configs with my preliminary recommendation that needs input from RTL.

Notably, I suggest that we need to add the 2-core Mobileye config, since during RC, it did not exist. Then there are questions about an 8 core config and multiported IOAIU configs that are not AXI proxy. Obviously, any additional configs present schedule risk.

At the bottom of the list, I mentioned the 1-core Mobileye configs. During some preliminary parameter coverage investigation, I did not notice any “system level” parameters where our “original_7” had a missing value on the extremes, smallest or largest legal values, that the Mobileye 1-core configs hit.
