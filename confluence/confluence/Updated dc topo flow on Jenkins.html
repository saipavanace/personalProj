1. Create a new workspace
  • mkdir <My_workspace>
  • cd <My_workspace>
  • /engr/dev/tools/script/setup_concerto
  • source SOURCEME ncore3.0.1

2. Make rtl changes and build the local copy of Maestro

  • Make rtl changes if needed
  • $WORK_TOP/dv/scripts/refresh_rtl

3. Generate RTL

  • mkdir <My synthesis folder>
  • cd <My synthesis folder>
  • maestro -c -s <configuration_file> (for hw_config_10 : maestro -c -s /home/bmadon/hw_config_10_synthesis/hw_config_10.tcl )

4. Link the external memories

  • The list of external memories is available in output/rtl/models/memories.
  • If wrappers are already available for this configuration copy the memories folder in <My synthesis folder>
  • run Aria from <My synthesis folder> : node /home/smusini/Aria/run_aria_ncore3_dc_topo.js (the parameter for the location of the wrappers and for the corresponding db file is set in the aria arguments, make sure this is correct)

5. Copy the palceholders and the external memories in the main design folder. This will avoid unresolved references for the palceholder and for the memories for which you don't have wrappers yet. it will allow to use flop instead and avoid unresolved references

  • (from the <My synthesis folder>) cp output/rtl/placeholders/*.v output/rtl/design
  • (from the <My synthesis folder>) cp output/rtl/models/memories/*.v output/rtl/design

6. Copy the original Makefile and update it to synthesize the block you want.

  • (from the <My synthesis folder>) cp output/synthesis/synopsys/scripts/Makefile output/synthesis/synopsys/scripts/MakefileOrg
  • The Makefile should look like this :


scriptDir = $(dir $(realpath $(firstword $(MAKEFILE_LIST))))
subDesignDdcs = ../ioaiu_top_a/outputs/ioaiu_top_a.gate_level.ddc ../dce_a/outputs/dce_a.gate_level.ddc ../dii_top_a/outputs/dii_top_a.gate_level.ddc ../dmi_a/outputs/dmi_a.gate_level.ddc ../dve_a/outputs/dve_a.gate_level.ddc ../fsc_a/outputs/fsc_a.gate_level.ddc ../ioaiu_top_b/outputs/ioaiu_top_b.gate_level.ddc ../config_dii_a/outputs/config_dii_a.gate_level.ddc ../grb_a/outputs/grb_a.gate_level.ddc

.PHONY: run
run: runtop

runtop: $(subDesignDdcs)
mkdir -p gen_wrapper;\
cd gen_wrapper;\
mkdir -p outputs;\
cp ../ioaiu_top_a/outputs/ioaiu_top_a.gate_level.ddc outputs ;\
cp ../dce_a/outputs/dce_a.gate_level.ddc outputs ;\
cp ../dii_top_a/outputs/dii_top_a.gate_level.ddc outputs ;\
cp ../dmi_a/outputs/dmi_a.gate_level.ddc outputs ;\
cp ../dve_a/outputs/dve_a.gate_level.ddc outputs ;\
cp ../fsc_a/outputs/fsc_a.gate_level.ddc outputs ;\
cp ../ioaiu_top_b/outputs/ioaiu_top_b.gate_level.ddc outputs ;\
cp ../config_dii_a/outputs/config_dii_a.gate_level.ddc outputs ;\
cp ../grb_a/outputs/grb_a.gate_level.ddc outputs ;\
dc_shell-t -64bit -f $(scriptDir)/gen_wrapper/dc.tcl -output_log_file gen_wrapper.dc.log

  • To synthesize only some block remove the strikthrough lines and keep only the modules you want in subDesignDdcs list of modules
  • For a full bottom up synthesis don't touch the file


7.Create a tarball to send to jenkins 

  • cd $WORK_TOP
  • tar cvf <MY synthesis archive>.tar <My synthesis folder>


8.Start a run on Jenkins 

  • Go to : Jenkins (arteris.com)
  • paste the path to the tar you just created (inclusing the name of the tar)
  • Select Ncore3 in the drop box called PORJ_TYPE
  • click build
  • The result of the synthesis will be in /scratch/jenkins/synthesis/out
