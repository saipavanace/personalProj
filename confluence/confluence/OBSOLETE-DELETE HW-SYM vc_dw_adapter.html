module_name

vc_dw_adapter


JavaScript parameters

dfDepth: Depth of pipe in data-payload data path. Must be a non-negative integral power of 2. For Narrow-to-wide conversion (ingress-link is narrower than the egress link), dfDepth must be at least equal to N = width of F_data of wide link / width of F_data on narrow link. For Wide-to-narrow conversion (ingress link is wider than egress link), dfDepth can also be set to zero if hfDepth is also set to 0.  

hfDepth: Depth of pipe in header data path. Must be a non-negative integral power of 2. Can be 0 only if dfDepth is also set to zero.

lckStyleVld: A boolean. Setting to true makes arbiter lock on valid, and setting to false locks on ready. Default false. When hfDepth == dfDepth == 0, set this to true to avoid valid from retracting.

suppressBlankBeats: Setting to 1 will suppress or swallow blank beats, which is the expected behavior for width adapters on the request path. 0 disables this (disabled for response path width adapters)

protectionStyle: protectionStyle

numPri: Number of priorities. Positive number 1 or greater.

mstrArbLck: A boolean. Master arbiter lock. Default false. True means arbiter locks. False implies arbiter doesn't lock.

arbType: A string that specifies the master arbiter type. Legal values: "arb_rr2", "arb_wrr_rr1", "arb_wrr_rr2", "arb_pri_rr1", "arb_pri_rr2".

arbRdyAware: When true, the master arbiter uses the ready and valid to make a request. When false, it relies just on valid.

weights: An array of positive integers and of length = number of VCs. The elements specify the weights for the master arbiter.

weightsProg: A Boolean value to be set to true if weights are required to be programmable.

Interfaces

clkInterface: A slave interface of type InterfaceCLK.

inInterface: A slave interface of type InterfaceATP for the ingress ATP link.

outInterface: A master interface of type InterfaceATP for the egress ATP link.

protInterface: (Optional interface). A master interface of type InterfacePROT.
