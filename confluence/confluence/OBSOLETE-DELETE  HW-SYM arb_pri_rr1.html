Description

Round Robin Priority arbiter. Each priority arbitrates like RR1 and each priority stores its own state. in_state and out_state are organized {priority numPri-1 state, priority numPri-2  state . . . priority 0 state}.

The highest priority is 0, followed by 1 and so on. If a single request at a higher priority exists, that priority will win.

Starvation mode, if implemented, will be implemented outside of this module.

Conceptually, a request will come in with a priority. The input of this block assumes a set of requests per priority level. So if you had a 3 priorities and 3 ports all requesting  with the following priorities:

portpriority
01
12
20

this would be converted to:

portpri = 0pri = 1pri = 2
0in_req_0[0] = 0in_req_1[0] = 1in_req_2[0] = 0
1in_req_0[1] = 0in_req_1[1] = 0in_req_2[1] = 1
2in_req_0[2] = 1in_req_1[2] = 0in_req_2[2] = 0

In this case, port 2 would win because it's the only port requesting at the highest priority.
