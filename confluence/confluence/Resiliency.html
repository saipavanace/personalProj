Injecting Uncorrectable error in SMI request (header/message/data)

Fault occurs when the SMI request (header/message/data) is received by a unit with uncorrectable error injected

  • SMI errors can be injected at HDR, NDP, or DP. Errors will be injected into randomly selected bits to the full length, including payloads and their protection bits.

  •  Error injection is controlled by below pulsarg
    1) +inj_cntl  = 2 will inject ECC UnCorrectable Error.
    2)+inj_cntl  = 4 will inject Parity UnCorrectable Error.

fault_checker.bist_next

fault_checker.bist_next_ack

In real full-system, FSC asserts bist_next to indicate that the output checkers FSM need to execute the next step of the BIST sequence. In unit-level TB, we do the following --

  1. force bist_next to 1

  2. wait for bist_next_ack to be 1

  3. wait for 2 cycles

  4. force bist_next to 0

  5. wait for bist_next_ack to be 0

moves the BIST FSM to the next step.

Remarks

Checks are implemented per unit:

  1. After the first BIST step, Corr/Mission/Latent faults should all be 0

  2. After the second BIST step, Mission and Latent both faults go to 1

  3. After the third BIST step, Mission fault should be 0 and Latent fault should be 1

  4. After the fourth BIST step, Mission fault should be 1 and Latent fault should be 0

  5. After the fifth BIST step, Mission fault should be 1 and Latent fault should be 0

  6. After final BIST step, Mission/Latent faults should all be 0

    1. After final BIST step finishes, wait for N (random) cycles then, check all faults should be 0
