2ATUI_AXI

  • Timeout Error

External error:

Error showing up on packet


  • Internal errors:
    RW error: PAM entry indicates target can only receive writes or reads but incoming transaction is the opposite.
    Timeout Error - A transaction hasn't seen a completed response within a timeout to 2*timeout amount of cycles. In this case, the sequence number of the timed-out transaction will be poisoned.
    Buffered Write - Error seen inside the response of a buffered write. Since the response has already been made an interrupt will be set and the context will be cleared.
    Decode Error - Error occurs when there is not a match to the PAM table and decode errors are enabled. (If not enabled the packet will match the default entry).

  • Externally Generated Errors - Errors show up inside the packet.

5ATUT_AXI

Internally generated:

  • Timeout Error
  • Buffered write
  • Exclusive Monitor Timeout Error

Internally Generated Errors

  • Timeout Error - This means a transaction hasn't seen a completed response within a timeout to 2*timeout amount of cycles. In this case, the sequence number of the timed-out transaction will be poisoned.
  • Buffered Write - Error seen inside the response of a buffered write. Since the response has already been made an interrupt will be set and the context will be cleared.
  • Exclusive Monitor Timeout Error - If the exclusive monitor does not receive a corresponding exclusive write before a timeout to 2*timeout amount of cycles & the exclusive monitor is enabled, then the monitor will be cleared.
