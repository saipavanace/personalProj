  1. Only allow 4 outstanding DVMs at any given time.

The IOAIU and CHI AIU changes is what allows the DVE to increase the number of STT entries it has as previously the number of outstanding entries was used to enforce the following:

  • CHI DSU - Only accept one sync at a time.

  • AR DSU - Only accept 4 outstanding DVM at a time

The increased number of entries in the DVE needs to ensure that Syncs can make progress for each ACE agent. Any non-sync ahead of a sync should make progress, so just need a landing spot for each agent. Emulation will give a look into this, and fsys should be able to test this.
