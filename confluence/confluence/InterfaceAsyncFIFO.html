javascript parameters:

{ 
 "name": "a_string", 
 "params": {
   "depth": 2 or greater,
   "width": 2 or greater,
   "async": true or false}, 
 "direction": "master" or "slave", 
 "interface": "InterfaceAsyncFIFO"
}

I/O:

input [depth-1:0] read_sel; 
output [width-1:0] data; 
output in_off; 
input out_off;
output [(params.async ? Math.ceil(Math.log(params.depth)/Math.LN2)+1 : params.depth+1)-1:0] write_ptr, 
input [(params.async ? Math.ceil(Math.log(params.depth)/Math.LN2)+1 : params.depth+1)-1:0] read_ptr;

Description:

This is the interface between the two sides of and async adapter. The async adapter has two modes (async = yes or async = no). When async equals yes, the two clocks have no phase relationship to each other. When async equals no, the two clocks have different frequencies, but are derived from the same root clock and have a known phase relationship to each other. The mode affects the signals write_ptr and read_ptr.

The signals in_off and out_off are used to communicate the status of the other side when the power or clock is turned off. The async adapter uses these signals to reset the state of side that is still on.
