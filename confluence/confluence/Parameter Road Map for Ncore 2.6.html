1. Requirement
    Need to expand parameter support in  Synopsys PA. The current parameter support does not cover all the parameters that affect performance. Adding the relevant parameters enables the user to modify them in Synopsys PA and analyze performance metrics. It also increases the usability of Alberto for parameter validation checking.

  2. Problem
    Based on the requirement, the relevant parameters affect FlexNoC. At present, Ncore generates SystemC Fast Models for all Ncore components and also invokes FlexNoC to generate the AV Model for the PDD (ccti only) derived from Ncore parameters. PA does not allow modification of these parameters as it is not possible to modify FlexNoC AV model in PA based on parameter changes. In Ncore 2.5, if the user wants to modify parameters that affect FlecNoC, the changes must be done in Ncore and a new SystemC model must be exported.

  3. Solution
    The solution is to enable a partial set of parameters that have a higher probability of modification for performance analysis. 
    1. Due to complex calculations, suggestions may not contain values i.e. only provide the user with a list of parameters that should be modified to resolve the issue.
    2. Generate a FlexNoC AV model with the max TransID widths and nPending transactions (master and slave).
