Below are the steps to convert existing switch and switch_hier configurations in Symphony regression list to JS randomizer flow.

1.To convert any topology to JS flow, there is a list of files which needs to be created.

  • param.json - this is the constraint file which include any constraints on parameters related to the topologies, also can include any constraint which otherwise causes and rtl or bench issues
  • top.json.tachl - which lists the sequence of files to be resolved and processed by the tool.
  • top_tb_wrapper.json.tachl - this is the top level tb wrapper json file which needs to be built based on the topology.This generally include calls to output various parameters based on the randomization
  • top_wrapper.json.tachl - this is the top level rtl wrapper file which build the rtl underneath.

Examples for the above files can be found at $WORK_TOP/js_randomizer/switch_dw_rand ( params_hier.json,top.json.tachl,top_tb_wrapper.json.tachl,top_wrapper.json.tachl)
