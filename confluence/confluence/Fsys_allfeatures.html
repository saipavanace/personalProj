Custom Config: Based on Ventana with lots of changes made for DV (Refer: CONC-16823 more info)

  • DII: Enable CMDOverflowBufInSRAM by programming the nCMDSkidBufArb to 16 and nCMDSkidBufSize to 64

  • DCE: Enable CMDOverflowBufInSRAM by programming the nCMDSkidBufArb to 16 and nCMDSkidBufSize to 64

    • nDmiMrdCredits parameter is not present in the configuration

  • DMI: All 4 DMIs have the same parameters programed and are identical. These are my observations based on referencing the parameter spec for 3.7 rev version 0.82 and the current test-plan:

  • Need to instantiate exclusive monitors "nExclusiveEntries": 0,

  • Address translation registers is not enabled "useAddrTranslation": 0 & "nAddrTransRegisters": 0

  • Way partitioning isn’t enabled "useWayPartitioning":0 "nWayPartitioningRegisters": 0

  • Tag and Data banks could be programmed to different values across multiple DMIs "nTagBanks": 1 "nDataBanks": 1

  • SMC should be enabled “useCmc”:0

  • Scratchpad isn’t used in SMC “useScratchpad”:0

  • Atomics should be enabled when SMC is used in an instance with scratchpad and without scratchpad "useAtomic": 0

  • PLRU could be instantiated as a replacement policy choice in one instance with SMC "RepPolicy": "RANDOM"

  • QOS could be enabled "fnEnableQos": 0

  • Native interface protection could be enabled "enableNativeIntfProtection": false

  • AWID address bits could be set to pick from a user/customer specified value

    • set_attribute -object $dmi_unit -name addressBits -value_list [list 31 32]

  • Address hashing can be enabled using this example [Secsubrows in ccp params]

    • set randproc_path [file dirname $base_config_dir]

    • source [file join $randproc_path global_procs/randomize_secondary_bits.tcl]

    • set rand_sec_bits [generateSecondaryBits 3 1 35 43 [list 8 9 10] $printDebug]

    • if {$printDebug == 1} { puts "The overall generated secondary bits are: { $rand_sec_bits }"}

    • Dmi::set_SecSubRows [Dmi::get_units {0}] $rand_sec_bits

  • Non-zero values for UESR in AXI “wAwUser": 0 “wArUser": 0

  • Enable MRDOverflowBufInSRAM by programming the nMRDSkidBufArb to 16 and nMRDSkidBufSize to 64

  • Enable CMDOverflowBufInSRAM by programming the nCMDSkidBufArb to 16 and nCMDSkidBufSize to 64
