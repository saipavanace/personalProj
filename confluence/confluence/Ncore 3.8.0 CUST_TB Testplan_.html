1. Introduction: ￼

This document goes through the testcases and infrastructure of customer TB including the customer deliverables and internal sanity regressions

  • The customer testbench is a mostly static example testbench generated by Maestro along with the RTL

  • After running a simulation, the customer Testbench is present in $WORK_TOP/dv/debug/<env_name>/<config_name>/exe/output/tb/<vcs|xsim>

4. Verification Scope

  • We don’t intend to verify the functionality of Ncore with customer TB (they are done in our regular testbenches)

  • We make sure that the Testbench we are giving to the customer compiles clean and the tests are able to finish without errors (or without hangs)

  • It is not possible to check the behavior of the customer TB with every possible config and so we select a few configurations which best represents the variations in configs and run them instead

  • Since we don’t have a scoreboard in customer TB, it is hard to verify the performance metrics of Ncore as well (we also don’t have a benchmark to compare the metrics against)

  • We verify the functionality of Makefile itself to make sure we are able to run testcases using makefile the way the customer uses it.

  • Protocol checks are enabled in the Synopsys and Cadence VIPs which catch any protocol violations
