Base Class

class VlogExpr

Composition

None

Description

This class is used to represent a Verilog signal of a certain width or a bit-slice of a Verilog signal. Every signal has start, end and name properties associated with it. The end is assumed to be 0 by default and can be optionally specified. The start value is determined internally using the width specified when calling the constructor.

Interface (Methods)

The class provides the following additional interface in addition to the methods it inherits from parent classes:

constructor ( width, name, options )

Creates an instance of VlogSignal. end index is set to zero by default. name is a string parameter that specifies the name of the signal and width is the width of the signal. The optional parameter options is an object that can have the following properties: end : A positive integer or zero to specify the end offset of the signal.

slice ( start, end )

Return type: VlogSignal. Returns a bit-vector resulting from the slice operation on the VlogSignal instance on which this method is called. If start and end exceed the boundaries of the vector, an error will be thrown. start and end are indices of the slice to be extracted.

lower ( width, end )

Return type: VlogSignal. Returns lower or least-significant width bits starting from index end. end is optional and defaults to zero if not specified.

upper (width, end )

Return type: VlogSignal. Returns upper or most-significant width bits upto bit with index start. start is optional and defaults to the most-significant bit of the bit-vector on which this is applied. 

trunc ( n )

Return type: VlogSignal. Returns a truncated ( to n bits ) vector of the original vector or net on which this method is called.

bit ( i )

Return type: VlogSignal. Returns a single bit from the original vector or net on which this method is called.

msb ( )

Return type: VlogSignal. Returns the most significant bit.

lsb ( )

Return type: VlogSignal. Returns the least significant bit.

sliceX ( { start, end } )  (Relative slice)

Return type: VlogSignal. Similar to slice method with two important differences:

  1. Call signature: Now expects a single parameter object with keys: start and end ( Same as start, end indices for slice ). This is to simplify method calls with results from other libraries that return a slice object ( { start, end } ).
  2. Offset addition: Automatically adds the end index of the bit-vector / net, on which this method is called, to the start and end parameters before slicing. This allows us to treat the bit-vector as though it's end index was zero, and hence allowing passing a slice object without modification or shifting. This is why it is called a relative slice.

declare ( type )

Return type: string. Returns the Verilog declaration of the signal. type parameter is optional and a string that can take only two legal values: "wire", "reg".
