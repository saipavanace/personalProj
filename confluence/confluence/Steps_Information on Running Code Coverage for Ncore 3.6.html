#1. Populate and create your own workarea.

#2. Kickoff regression suite for code coverage from the runsim command line:

runsim -e <block_name> -c <config_name> -l regr_cov -v UVM_NONE -i <num> -g -c_cov

This will:

  1. Run all tests with the label ‘regr_cov’ for <num> iterations &

  2. Merges individual tests' VDB databases together.

Example DVE command: (Will run default iterations of each test without -i <num> )

runsim -e dve -c hw_config_resiliency_duplication_pma -l regr_cov -v UVM_NONE -g -c_cov

#3 Use DVE to analyze and view coverage DB file, write exclusions, etc.

dve -cov -covdir $WORK_TOP/regression/<timestamp>/debug/dve/hw_config_resiliency_duplication_pma/coverage/merged_dve_hw_config_resiliency_duplication_pma.vdb/

Can also use web browser to view HTML/text reports of collected code coverage if desired. Cannot write exclusions. 

firefox $WORK_TOP/regression/<timestamp>/debug/dve/hw_config_resiliency_duplication_pma/coverage/urgReport/dashboard.html &
vi $WORK_TOP/regression/<timestamp>/debug/dve/hw_config_resiliency_duplication_pma/coverage/urgReport/dashboard.txt &
