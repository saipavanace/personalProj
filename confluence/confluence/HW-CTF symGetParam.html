Usage:

function symGetParam(param,getParam)

param is the named element in an object you are getting the value of

getParam is a pointer to the standard getParam in the library.

Description:

This function enables the use of hierarchical JSON files. It works in the following way: When ever it comes across an object that is not an array, it looks for a element named "file_path." If the element exists, it loads the JSON file pointed at by the value of file_path into a variable that is merges it into the object where it found the element named "file_path." The exception to this the object is not merged if the object pointed to by "file_path" is an array. In this case the array replaces the object that "file_path" was in.

The string specifying the file path can contain environmental variables.

The purpose of this is to reduce that amount of typing needed to created JSON files that have a lot of repeated data and make the files easier to read.

Examples:

You have a JSON file with the following in it:

 "moduleParams" : {"masterInterface" : {"name": "b_","signals": {"valid" : 1, "ready" : -1, "bus" : 32}},

 "localInterfaces" : ["a_","g_"],
 "moduleInterfaces" : ["a_","b_"]
 },
 {"name": "i1",
 "module" : "gen_test",
 "moduleParams" : {"masterInterface" : {"name": "b_","signals": {"valid" : 1, "ready" : -1, "bus" : 32}},
 "slaveInterface" : {"name": "a_","signals": {"valid" : 1, "ready" : -1, "bus" : 32}}},
 "localInterfaces" : ["b_","h_"],
 "moduleInterfaces" : ["a_","b_"]
 },
 {"name": "i2",
 "module" : "gen_test",
 "moduleParams" : {"masterInterface" : {"name": "b_","signals": {"valid" : 1, "ready" : -1, "bus" : 32}},
 "slaveInterface" : {"name": "a_","signals": {"valid" : 1, "ready" : -1, "bus" : 32}}}, 
 "localInterfaces" : ["c_","i_"],
 "moduleInterfaces" : ["a_","b_"]
 }
 ]
