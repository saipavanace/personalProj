This function returns either an empty array or a single-element array with a string that is a bit slice corresponding to the specified field on a bus/wire assuming the bus/wire aligns with the entire beat of a packet.

It is called as follows:

var return_data =  extractField(link_width,link_packet_def,wire, field,offset);  // Offset is optional

link_width : width of the link in bits

link_packet_def : packetDesc

wire: A string that is the verilog bus or wire name from which the bit-slice corresponding to the field are to be extracted. It is assumed that the start and end of the beat aligns with the entire 'wire' or bus. Use offset if not aligned or displaced.

field: text string used to match against field names.

offset: Optional argument. Default = 0. An integer that is subtracted from the field_start before it is returned. This is useful for example to access field indexes in the header-portion only i.e. assuming the last bit of the header aligns with index 0.


This function relies on the HW-SYM extractFieldBitIndexes function. If the field does not exist, the function returns an empty array.
Example:

extractField(wLink, pktDef, "atp_bus", "H_tid")   // Could return something like ["atp_bus[42:40]"] assuming bits 42-40 on "atp_bus" corresponds to H_tid according to the pktDef.
