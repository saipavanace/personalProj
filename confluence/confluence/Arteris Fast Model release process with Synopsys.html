Ncore 2.5 is released with the ability to generate SystemC Fast Models. This feature is enabled with the support provided by Arteris IP and Synopsys.

ArterisIP Delivery

  • Delivery to Synopsys
    • The Angelo team sends a tgz file containing the entire Angelo repository and the libncorepa.so file generated using Cmake. 
  • Delivery to Customer
    • The SystemC Fast models are packaged as part of the conductor installation.
    • Based on request from Synopsys, the directory structure in the conductor installation directory is as follows:
    • Alberto and pa containing all the files are packaged with conductor.
    • When the customer brings up Synopsys PA, he needs to set CONDUCTOR_HOME = <Conductor installation path>.

Synopsys Delivery

  • Delivery to Customer
    • Synopsys delivers a package called ARTERIS_Ncore_BL i.e. the Arteris IP Ncore Bus Library. 
    • This library is needed to bring up Ncore SystemC Fast Models in Synopsys Platform Architect.  
    • The ARTERIS_Ncore_BL contains .so files for each of the Ncore units (TLM wrappers), example Ncore configuration and tcl scripts.
    • The .so files for all the units are built using C++ header files provided by the Arteris IP Angelo team in their delivery to Synopsys.
    • The Header files present in the snps directory (in the above diagram) are the common header files used by both Synopsys and Angelo team and hence should be consistent.
