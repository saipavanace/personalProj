This feature is useful for those who has broken long regressions due to some unfortunate script issue or due to some hang issue, in that case runsim is not able to generate end report/triage report for the same. But with this feature now user has power to generate report for any regression which has regression directory available.

Syntax:

Case-1:
runsim -regr_rpt <regression_path>

In regression path User has to provide particular path for which he/she wants to generate the report.
By hitting the above command it will generate .txt, .html report in User’s regression path directory as well as it will send e-mail the report at User’s mailing address which User had provided under -m option while he/she launched the regression.

Caes-2:
runsim -regr_rpt <regression_path> -m <mail-address>

If In-case User forgot to use -m option at the regression launch time and if User wants the report at User’s mailing address the User has to use the Above mentioned command instead of Case-1

Limitation:

Right now, there is only one limitation for this feature that is If any User is running any regression sequentially and provided “rand“ value under -s option on command line then User might see wrong seed value under “seed“ column of signature triage information table in the report. in this case it is advisable to open the respective log file and get the seed value from there for re-run. Also Code-coverage calculations are based on seed value so code coverage numbers could be wrong in this case.

For now, Work is in-progress for this corner case, will try to resolve this asap.

Examples:

Local regression:

  1. cd $WORK_TOP/regression

  2. cd <regression directory name for which you want to generate the report>

  3. pwd

  4. Copy the path shown on terminal after pwd

  5. cd $WORK_TOP

  6. runsim -regr_rpt <Copied path from regression directory>

Jenkins regression:

  • Go to your jenkins regression dashboard

  • Go to your regression

  • Look at the regression date for which you didn’t get the report or you want to generate the report

  • Go to http://dev.arteris.com/dv_reg_ncore/reports/ and click the latest regression for your block with the same data and time.

  • Copy $REGR_TOP from that regression

  • Go to the terminal

  • cd $WORK_TOP

  • runsim -regr_rpt <copied path from $REGR_TOP>

  • Check your regression report at $WORK_TOP/regression/<Regression_name>
