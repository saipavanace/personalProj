"Magic Happens Here" can be used to describe some of the information that the hardware design code is expecting to be provided by the Maestro software.  Many but not all of those expectations are known the software team.  This page is an attempt to gather together all of the "Magic Happens Here" expectations.

 CategoryDescriptionNotes
1 Design

The ability to clone parts of the design and stamp out multiple instances.

  • requires hardware support presently missing.
 
2DesignSoftware can freeze a design and go into a special ECO mode to limit the changes that occur in the outputted assets  (RTL, IPXACT . . . ) when the customer makes design changes. 
3NetlistSoftware generates the top level netlist and any arbitrary hierarchy required by the customer. 
4Netlist

Software guarantees that any of the topologies are dead lock free:

  • If there are outside connections that can cause deadlock provided a means for customers to input that information
  • Also composing multiple NOCs (Like above but it’s the tool that’s doing it and it can autogenerate the info needed).
 
5NetlistSoftware generates the connectivity and hierarchy required when the customer defines power domains and clock domains. 
6NetlistSoftware generates the connectivity and the hierarchy required, based on customer inputs, on how interrupts are hooked up and accumulated. 
7Netlist

Software generates the connectivity and the hierarchy needed to implement resilient features.

  • Instancing and connecting up the Safety Controller.
 
8Netlist

Software provides the means by which a customer can drop in “customer code” into the ATUs and Firewalls.

  • requires HW support presently missing.
 
9Netlist

Software provides a means by which customers can add custom inputs and custom outputs that can tie into their customer logic (8), interrupts, debug events and debug control.

 
10NetlistSoftware provides a means for the customer to custom configure debug elements. 
11Address Map

Software generates the connectivity and address maps of where config/status/error/interrupt registers exists.

  • Software provides the customer with means to space the registers as directed by the customer for future expansion and ECO needs.
  • Software provides an ECO mode where “new stuff” gets stuck starting at a designated place if the gaps available are too small to handle the changes.
 
12Resiliency

Software provides a means for the Customer to turn on and off resilient features in a modular manner.

 
13TestbenchSoftware generates a validation bench that allows the customer to run a reduced set of tests to validate connectivity and address maps based on their design. 
14Export

Software generates all the assets needed by a customer to integrate the design into their design:

  • IPXACT descriptions.
  • Performance models assets that can be used in a customers performance model environment.
  • Verification assets that can be used In a customers verification environment.
  • RTL
  • Synthesis scripts.
 
15TestbenchSoftware generates System-Verilog interfaces (equivalent to TACHL generated RTL interfaces) and connect them with RTL interfaces (I/O signals). 
16TestbenchSoftware generates random configurations (of interfaces & units) with smart constraints on interdependence parameters at topology level. 
17NetlistSoftware determines the contents of a packet based on customers input and the contents of a fabric.
