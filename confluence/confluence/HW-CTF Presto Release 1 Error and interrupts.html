HW-CTF sym_multicast

This block does not exist in Symphony R1. Multicast block fires an interrupt when buffering is enabled and a target returns an error. Verify that this causes an interrupt to fire.

Error Status Register @ offset=0
[0] EarlyResponseErrVld. If this bit is set, then an error for early response has occurred. Writing a one to this bit clears this bit.
[1] EarlyResponseErrMultiple. If this bit is set, then multiple errors for early response have occurred. Writing a one to this bit clears this bit. In case of multiple errors for early response, only the first error for early response is logged.
Error Location Register 0 @ offset=4
[wSrcId-1:0] EarlyResponseSrcId. This field indicates the multicast request packet's Source ID in for the early response error.
Error Location Register 1 @ offset=8
[wMsgSeqnum-1:0] EarlyResponseMsgSeqnum. This field indicates the multicast request packet's Message Sequence Number for the early response error.
Error Location Register 2 @ offset=12
[wMultiLabel-1:0] EarlyResponseMultiLabel. This field indicates the multicast request packet's Multicast Label for the early response error.
Error Location Register 3 @ offset=16
[wMultiMask-1:0] EarlyResponseMultiMask. This field indicates the multicast request packet's Multicast Mask for the early response error.
Error Location Register 4 @ offset=20
[wTargPortVec-1:0] EarlyResponseTargPortVec. This field indicates the Target Port Vector for the early response error. Bit 0 corresponds to the Multicast Egress Port 0, bit 1 corresponds to the Multicast Egress Port 1, etc.


Note: The diagram in the section “Presto 1.0” shows the various functional blocks and their connections.

The "Per Label Context Block" generates error interrupt for any bufferable transaction that has error indicated in the its late response in HW-CTF sym_multicast

The number of targets per Multicast Block is limited to maximum 32 if the LABEL mask register is implemented and the number of mask bits is the same as the number of targets. Note that the number of mask bits can be the same or greater than the number of targets.


When the "noRegs" parameter passed into sym_multicast is true or 1, the following registers are not created, otherwise created:

Label Valid Register @ offset=64. This is an optional register.
[0] Valid. The label is valid.
Label Value Register @ offset=68. This is an optional register.
[wLabel-1:0]. The label value (i.e. the label ID).
Label Mask Register @ offset=72. This is an optional register.
[wMask-1:0]. The label mask.

SYM-2147
HW-CTF atui_axi
There can be two types of errors: (All internal errors can generate an interrupt if enabled)
  • Internally Generated Errors
    • RW Error - This means that a PAM entry indicated a target can only receive Writes and or Reads and the incoming transaction is the opposite.
    • Timeout Error - This means a transaction hasn't seen a completed response within a timeout to 2*timeout amount of cycles. In this case, the sequence number of the timed-out transaction will be poisoned.
    • Buffered Write - Error seen inside the response of a buffered write. Since the response has already been made an interrupt will be set and the context will be cleared.
    • Decode Error - Error occurs when there is not a match to the PAM table and decode errors are enabled. (If not enabled the packet will match the default entry)
    • Exclusive Error - Error occurs when there is an exclusive transaction that is split.
    • Power Domain Error - Error occurs when a transaction is sent to a target through a domain which is off as indicated by pamTargDomain and domainInt
  • Externally Generated Errors - Errors show up inside the packet.
    • Transport Error - Error seen in the transport. This mostly applies to poison and errors in the CSR network.
  • In addition, when a PMON exists inside this initiator, the PMON also generates an interrupt:
    • Done - Indicates that the global timer inside PMON reached the programmed limit.

RAL Programming

  1. Each error type has its own unique interrupt register using the naming convention CTL*IR.
  2. Set the CTL*IR field 'E' to 0x1 to enable interrupt generation for the error.
  3. The CTL*IR field 'I' will be set to a 1 when the interrupt triggers.
  4. Writing 0x1 to CTL*IR field 'I' will clear the interrupt.
  5. Please refer "Error Generation" section in below confluence page for behaviour
    HW-CTF Common Transaction Layer Microarchitecture Specification#ErrorGeneration
  6. At the system level the ATUI will only send a single interrupt out. If multiple ATUis exist there will have to be an interrupt block to accumulate these single interrupts. So at an ATUi level the verification can stop at making sure the single interrupt asserts/clears
  7. See Chapter 16 of the Symphony Architecture Spec to see some interrupt diagrams at a high level.

HW-CTF atut_axi



ATUT AXI used in a non-config network.

  • Exclusive Monitor Timeout Error - If the exclusive monitor does not receive a corresponding exclusive write before 2*timeout amount of cycles & the exclusive monitor is enabled, the entry will be cleared and an interrupt will be asserted.
  • Buffered Write - If a target receives a buffered write and receives an error response from the slave & enabled, an interrupt will be asserted.
  • Timeout Error - This means a transaction hasn't seen a completed response within a timeout to 2*timeout amount of cycles. In this case, the sequence number of the timed-out transaction will be poisoned.
  • In addition, when a PMON exists inside this target, the PMON also generates an interrupt:
    • Done - Indicates that the global timer inside PMON reached the programmed limit.
