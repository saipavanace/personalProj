Requirements

  • execute different tests to check mainly BW and Latency in DV simulation and produce Golden reference metrics.

  • Performance measurements will be done in the defined limtless config.

  • report results in Web page or Excel in every weekly regression.

  • Simulation will be based on FSYS environment (could be extended to IP level).

Testcases and list of Dynamic parameters

Dynamic parameters are the parameters controlling the test execution (like coherent/non-coherent, with/without allocation…). the Combination of these parameters will alow us to create different types of testcases that will let us:

  • to measure the performance of a dedicated configuration in best case like 1 initiator at a time.

  • to stress different/specific blocks in the Ncore (like buffers, arbitrers, switchs…).

Configuration

To be able to qualify the Ncore, 1 configuration will be created that will be the reference showing best performance of Ncore and helping to track performance metrics after each release. They will be no limititation in term of ressources (like skidbuffer sizes, FIFO size…) to avoid saturation issues. This configuration will consist on :

  • 1 CHI-B

  • 1 CHI-E

  • 1 ACE

  • 1 AXI with ProxyCache

  • 1 AXI without ProxyCache

  • 1 ACE-Lite

  • 1 ACE5-Lite

  • 1 ACE5-Lite DVM

  • 1 MultiPort AXI without ProxyCache

  • 1 MultiPort AXI with ProxyCache

  • 4 DMIs

  • 4 DCEs

  • 1 DIIs

  • Legato: 4N + Cross Bar Topology,switch buffer depth = 32
