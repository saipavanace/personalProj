Random FSYS configs

Currently we generate a small number of FSYS random configs and run dozens of tests per config. Add flow support to easily generate hundreds or thousands of FSYS random configs to run lint on and 0-1 simulations. This would help parameter coverage and tachl coverage efforts.

Block configs random instance support

Allow a block config to specific instead of an instance in the FSYS config, an instance_type (such as IOAIU), or instance_sub_type (such as IOAIU ACE-LITE-E), and the block config will randomly choose an instance based on instance_type and/or instance_sub_type. Start with IOAIU. This would help parameter coverage and tachl coverage efforts as well as bug hunting.

Random block configs

Enhance the FSYS_rand flow to be able to create a block config from a random FSYS config. Should take advantage of Block configs random instance support. This would help parameter coverage and tachl coverage efforts as well as bug hunting.

Improved signature parsing

Allow the delimiter :: to end the error line for signature binning. For example, “addr_mismatch:: expected=a00, actual=a05” would be binned as “addr_mismatch::”

Easy rerun

Make it very easy for users to rerun failures and copy/paste a rerun command.

Finer-grained timeouts.

See examples at IOAIU DV improvements

Add finer-grained timeouts to make Heart Beat debug much easier, especially for FSYS.

Better stimulus control

Create both more focused stimulus (i.e., realistic qos scenarios) and/or highly controllable transaction level directed stimulus.
