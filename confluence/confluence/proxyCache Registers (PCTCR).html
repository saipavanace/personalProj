PCTCR.UpdateDis

Setting XAIUPCTCR[UpdateDis] can be used to disable update commands to DCE. This bit affects update commands in general irrespective of the original allocating transaction is coherent or non-coherent.

Default setting: 1, write 0 in initialization sequence for all random traffic testcases to enable update request

ccp_switch_on_off

Test starts with PCTCR.LookupEn=0 and PCTCR.AllocEn=0,

→ Do random traffic. All go out to the bus, no CCP lookup.

→ Mid-simulation turn on LookupEn, do random traffic. No cache allocation

→ Mid-simulatio turn on AllocEn, do random traffic. Now CCP gets allocated.

→ Mid-simulation, take CCP snapshot and turn off AllocEn, and do random traffic.

(check that no allocations are happening into cache. In other words, cache is locked). Check the ncbu-cache model contents are intact, by matching current CCP snapshot with previous one

→ now initiate a cache-flush sequence to empty the cache.

→ Turn off LookupEn

→ Run random traffic, no cache lookup happens and all transactions go to bus.
