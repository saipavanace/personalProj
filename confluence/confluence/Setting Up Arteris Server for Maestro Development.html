Connect to Arteris VPN

In order to have access to Arteris resources (development server, Jira, Confluence, Jenkins, ect.) one should be connected to the Arteris VPN. Detailed instructions should be sent to a newcomer in the welcome e-mail.


Login to Arteris Development Server

Maestro is built on the gr6-3.arteris.com server.

To login to the Arteris development server use ssh:

$ ssh username@gr6-3.arteris.com
If prompted something like the following, type yes.
The authenticity of host 'gr6-3.arteris.com (192.168.10.175)' can't be established.
ECDSA key fingerprint is SHA256:(a bunch of stuff...).
Are you sure you want to continue connecting (yes/no)? yes

Then enter your UNIX/Linux environment password (from the welcome e-mail) and you should be logged in to the server (with your default shell, e.g. bash).

You can also use any suitable SSH client (e.g. PuTTY on Windows).


Configure Arteris Server for Maestro Build

Arteris development server has almost all the libraries required for Maestro build. However, a couple of things should be configured manually.

Setup Conan

Install Conan configuration for your user. It needs to be done only once.

$ conan config install /scratch3/maestro/conan/config/conan_config_arteris.tar.gz

Load Maestro Modules

In order to build and run Maestro, a server user needs to initialize the environment. Environment modules are used on the Arteris server for this purpose. You can load modules manually every time you log in using the commands below or you can add this commands to your shell init script (e.g. ~/.bashrc for bash):

# Loads environment variables for Maestro build.
module load maestro

# Loads Maestro license (to run Maestro).
module load license/arteris

# Loads the latest NodeJS required for tests.
module load node/lts

Please note, that any changes you make to .bashrc will be applied next time you launch terminal. If you want to apply them immediately, run the command below:

$ source ~/.bashrc
