The page will summarize the changes in the Tcl commands or syntax.

17 Jul 2020 save_project -force

Added -force option to save_project command. If -force option is present, save_project will overwrite the existing file. If not present, save_project will issue a warning and project file is not written. (MAES-506)


09 Jul 2020 Depreciated debug_generate_rtl and debug_run_derivations 

debug_generate_rtl has been replaced by gen_collateral.


debug_run_derivations functionality has been incorporated into the state machine. Depending on the state transition, the “ddg_checker” is automatically executed. There is no need for “debug_run_derivations” to call the “ddg_checker” explicitly.


07 Jul 2020 insert_adapter will create the adapter

Depreciated the need to create the adapter before inserting the adapter into the network. Now, the adapter will be created during adapter insertion.


Insert a width adapter on packet port $pp

set adapter [insert_adapter -name $adapterName -type width_adapter -port $pp]

The valid types are: width_adapter, clock_adapter, rate_adapter, pipe_adapter, fifo_adapter.

The $pp is a packet port. If the packet port is an input port, the adapter is inserted in front of the port. If packet port is an output port, the adapter is inserted after the port.

The $adapterName is the name of the adapter.

The return value is the name of the adapter created.



15 Jun 2020 Reduce Parameters in update_object

Depreciated

update_object -model routing -add [list ...] -method force


Replaced with:

update_object -model routing -add [list ...] -value_list [ method force ]


Depreciated

update_object -inclock $inclock_of_clockadapter -outclock $outclock_of_clockadapter


Replaced with:

update_object -value_list [list inclock $inclock_of_clockadapter outclock $outclock_of_clockadapter]


Depreciated

update_object -new_name $new_name_of_object


Replaced with:

update_object -value_list [list new_name $new_name_of_object]


Depreciated

update_object -execute $GENERATOR -method "execute()"


Replaced with:

update_object -value_list [list execute $GENERATOR method "execute()"]



08 Jun 2020 Multi-Flow Changes

Depreciated

multi_flow_edit command. It is replaced by update_object command.


To remove sockets from the destinations, the command is:

update_object -name $mf -type delete_destinations -value_list [list $socket1 $socket2 . . . ]



05 Jun 2020 Multi-Flow Changes

Depreciated

multi_flow_create and multi_flow_report commands are replaced by create_object, update_object, and get_objects commands.

New Usage

To create a multi-flow object under subsystem, the command is:

set mf [create_objects -name $name -type multi_flow -parent $subsystem]


To specify the source socket connection of the multi-flow: 

update_object -name $mf -type source -bind $srcSocket


To specify multiple socket destination connections of the multi-flow:

update_object -name $mf -type destinations -value_list [list $socket1 $socket2 . . . ]


To get the source of the mult-flow:

set src [get_objects -parent $mf -type socket -subtype source]


To get the destinations of the multi-flow:

set destinations [get_objects -parent $mf -type socket -subtype destinations]
