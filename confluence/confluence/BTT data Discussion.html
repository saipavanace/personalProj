  • Reads
    • Do not allocate BTT data for read hit with no dependencies. 
    • Allocate BTT data entry for read hit if the transaction has dependencies.
    • Allocate BTT data entry on all read misses
      • Need to wait for STR response (state and error info)
      • decouple SFI and AXI.R (assumption: NOC dependencies between AXI.R and SFI) 
    • Do the fill and then STRrsp as soon as the DTRreq and STRreq are received. Response data comes out of BTT data. (SNPs are not sent by DCE until it receives STRresp for the fill)

  • Writes
    • Do not allocate BTT data for write hit irrespective of dependencies
      • BTT Ctrl entry will need to be allocated in case of dependencies to issue proper response
        • should distinguish between multi line and same AXID 
    • Allocate BTT data for all write hit upgrade 
      • do not want to clobber the cache. 
    • Allocate BTT data for Write miss allocate partial 
      • Can avoid this if we can justify separation of byte-enable storage
    • Allocate BTT data for write miss allocate full 
      • Keep things simple minimal advantage of optimizing this case. 
    • Allocate BTT data for all Write miss no-allocate irrespective of dependencies
      • Should de-allocate BTT data ASAP (when DTW is sent)

  • Bypass path from SFI to AXI.R for transactions with no dependencies used when STR is received before DTR (might remove this if it simplifies the design)
