Discussion items

Things that are inconsistent with the Ncore software:

  • "More information" is often not applicable, the link should not show up.
  • First "Information" tab is about the project, even though you're not on the 'create project' view any more.
  • Issues - the project has both errors and warnings, should be seperately counted.
  • Issues does not include FlexNoC errors.
  • "Edit button" - inconsistent with any other software, and means that there are lots of every clicks for every edit.
    • Some fields don't adhere to the edit button clicking.
    • Can go from Generation to coherent subsystem when in edit mode, cannot go in the other direction.
    • The way the undo stack works relative to the edit button is inconsistent with most undo paradigms.
  • Some parameters are per port (per unit), and others are per logical agent
    • Suggestion:  Enter everything per pot, and have the GUI display the per-logical-unit values.
  • "TI" in the Ncore portion, vs. "NoC" in the FlexNOC portion.
  • Having both a "Name" and "RTL Prefix Name" for each element.
  • Some parameters are in the "Coherent Subsystem" tab, some are in the Generation" tab.
    • Proposal: remove the 'implementation parameters' page, and move the pipeline parameters to be with the rest of the unit parameters.
  • Some units are bits, some units are bytes.
