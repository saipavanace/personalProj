The models and testbench elements are configured using SystemC attributes and SystemC-CCI parameters (subsumed as properties in the following). These attributes and parameters can be configured using either a JSON or a YAML file. In both cases the keys of a hash of hashes form the hierarchical name of a property by concatenation of the keys using a '.'. So the following two examples in JSON format are equal:

{
    "ncore_sim": {
        "caiu0_drv": {
            "master": {
                "log_level": 6
            }
        }
    }
}
{
    "ncore_sim.caiu0_drv.master.log_level": 6
}

The same in YAML can be expressed as 

ncore_sim:
    caiu0_drv:
        master:
            log_level: 6

ncore_sim.caiu0_drv.master.log_level: 6

The YAML format has an advantage over JSON: it supports including other YAMl files. Taking the example above and splitting it into caiu.yaml

caiu0_drv:
        master:
            log_level: 6

and ncore_sim.yaml including caiu.yaml:

ncore_sim: !include caiu.yaml

results in the same setup as shown before.

The used infrastructure allows to use globbing and regular expressions to match hierarchical names.

The globbing supports the following meta symbols:

  • ‘*' any character except '.’ in any quantity

  • ‘**’ any character in any quantity

  • '?' any character exactly once

  • ‘[…]*’ the character class given in [] in any quantity

  • ‘[…]?’ the character class given in [] exactly one

A regular expression has always to begin with ‘^'. The hierarchical name is treated as a string. This means if there is a need to match the hierarchy separator it needs to be escaped ('\.').

As an example, the following 5 snippets achieve the same result, enabling the logging in caiu0_drv and its components:

{
    "ncore_sim": {
        "caiu0_drv": {
            "log_level": 6,
            "master": {
                "log_level": 6
            },
            "pe": {
                "log_level": 6
            },
            "parser": {
                "log_level": 6
            },
            "cache": {
                "log_level": 6
            }
        }
    }
}
{
    "ncore_sim": {
        "caiu0_drv**.log_level": 6
    }
}
ncore_sim:
  caiu0_drv:
    log_level: 6
    master:
      log_level: 6
    pe:
      log_level: 6
    parser:
      log_level: 6
    cache:
      log_level: 6
ncore_sim:
  caiu0_drv**.log_level: 6
ncore_sim.caiu0_drv**.log_level: 6

The last example as a regular expression it might look like

^ncore_sim\.caiu0_drv.*\.log_level$: 6
