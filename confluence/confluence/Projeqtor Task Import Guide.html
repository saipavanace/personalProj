It is possible to import tasks into Projeqtor from a .CSV (comma-separated value) file. Excel and other spreadsheet tools can export in this format.

  • You need to be an "Administrator" for the project for which you wish to import tasks.
  • Create an Excel sheet (or equivalent) with the following structure:
nameactivity typeprojectparent activitystatusresponsiblerequested durationplanning mode
taskname1taskprojectname1
(e.g. Ncore 3)		activityname1recordedpersonname1daycount1as soon as possible
taskname2taskprojectname2
(e.g. Ncore 3)		activityname2recordedpersonname2daycount2as soon as possible

NOTE: strings in bold should be used verbatim. Strings in italics should be customized task-by-task.

NOTE: this example does not include the allocation of a resource to the task. The example is to be extended. See below for a post-import solution.

  • Export the file as .CSV. For Excel users, this is in File -> Export -> Change File Type.

NOTE: Projeqtor demands a ";"-separated CSV file, i.e. each field separated by a semicolon. Some spreadsheet editors allow to choose the field delimiter upon export. Excel on the other hand silently relies on the locale settings of the country. The en-US locale default appears to be ",", i.e. each field separated by a comma. See this link https://www.itsupportguides.com/knowledge-base/office-2013/excel-20132016-how-to-change-csv-delimiter-character/ on how to change the delimiter temporarily for this export. It can be changed back right afterwards.

  • The file will look something like (also see attachment ncore3_ioaiu_schedule_to_upload.csv):

name;activity type;project;parent activity;status;responsible;requested duration;planning mode
ACE-LITE E stashing bring up;"Task";"Ncore 3";"DV IO AIU";"recorded";"David Clarino";"10";"as soon as possible"
ACE-LITE E misc support/DVM bring up;"Task";"Ncore 3";"DV IO AIU";"recorded";"David Clarino";"15";"as soon as possible"
Data width adaptation;"Task";"Ncore 3";"DV IO AIU";"recorded";"David Clarino";"5";"as soon as possible"
Configurations test plan;"Task";"Ncore 3";"DV IO AIU";"recorded";"David Clarino";"2";"as soon as possible"

NOTE: it is possible to hand-edit the file, including copy-pasting this snippet and editing it in a plain text editor. If your text contains semicolons or other special characters within a field, remember to enclose the whole field in double quotes "" or the file may be invalid. Excel takes care of this upon export.

  • In Projeqtor, go to the menu on the left. Under Tools, choose Import Data, then browse to pick the .CSV.

NOTE: this command is only available to project administrators, see first bullet.

NOTE: Projeqtor will throw an error, on a line-by-line basis, if there are problems with the CSV. However, if the data is formally correct, Projeqtor will not ask for further confirmation before adding the tasks to the project.

  • The example above creates a task within a project and under an activity of that project. The task is assigned a responsible person and a duration in days. However, no resource is allocated for the execution of the task. To do that, after the import, navigate to the task (e.g. in the Planning view), scroll down, and under the Assignment tab, either add a new resource or change the "assigned (d)" field of the task responsible to read the same count of days as the task duration. Save.

NOTE: overwriting tasks with a second import is untested and appears dangerous. Avoid until tested on a mockup project.
