This is one of the JS functions returned by vlogGen() which can be found in the following path and file:
hw-lib/js/lib_utils.js

Base type

vlogExpr

Composition

None

Description

This is a factory function that returns an object that represents a bit vector along with functions to perform various operations on the signal / bit-vector. Every bit-vector has a start, end, and name property. In addition, because the object returned is an extended vlogExpr object, it also has width and expr properties with it.

Usage

The factory function can be called using the following call signature:

vlogSignal( width, name,  options )

where width is a positive integer that represents the width of the signal / bit-vector in bits, name is a string that becomes the name of the signal, and options is an optional parameter with the following properties:

  1. end: Specifies the end offset of the signal. If not specified, the end offset defaults to 0.

Interface

This object returned by the vlogSignal( ) factory function provides the following .

Data

The returned object will have the following data-properties in additions to properties of the base type object (vlogExpr):

  1. start. type: number. Represents the start offset of the signal.
  2. end. type: number. Represents the end offset of the signal.
  3. name. type: string. Holds the computed Verilog expression string.

Methods

The returned object will provide the following methods: 

slice ( start, end )

Return type: vlogSignal. Returns a bit-vector resulting from the slice operation on the vlogSignal object on which this method is called. If start and end exceed the boundaries of the vector, an error will be thrown. start and end are indices of the slice to be extracted.

lower ( width, end )

Return type: vlogSignal. Returns lower or least-significant width bits starting from index end. end is optional and defaults to zero if not specified.

upper (width, end )

Return type: vlogSignal. Returns upper or most-significant width bits starting from the bit with index start. start is optional and defaults to the most-significant bit of the bit-vector on which this is called. 

trunc ( n )

Return type: vlogSignal. Returns a truncated ( to n bits ) vector of the original vector or net on which this method is called.

bit ( i )

Return type: vlogSignal. Returns a single bit from the original vector or net on which this method is called.

msb ( )

Return type: vlogSignal. Returns the most significant bit of the vector.

lsb ( )

Return type: vlogSignal. Returns the least significant bit of the vector.


split( chunkWidth )

Return type: JS Array of vlogSignal objects. Divides the bit vector uniformly into chunks of size chunkWidth and returns each chunk in the array.

slicer ( { start, end } )  (Relative slice)

Return type: vlogSignal. Similar to the slice method above but with two important differences:

  1. Call signature: A single parameter object with keys: start and end offsets. This is to simplify method calls with results from other libraries that return a slice object ( { start, end } ).
  2. Offset addition: Automatically adds the end index of the bit-vector / net, on which this method is called, to the start and end parameters before slicing. This allows us to treat the bit-vector as though it's end index was zero, and hence allowing passing a slice object without modification or shifting. Hence the name "Relative slice".

shift ( op, n, extendWidth )

Return type: vlogSignal. Returns a vlogExpr object resulting from shifting the vlogSignal object by a constant. Parameter op specifies the type of shift operator: "<<", ">>", "<<<", ">>>". Parameter n is the shift amount (number). By default the result of shift operation is assumed to be the same width as that of the signal that is being shifted. The last parameter, extendWidth can be optionally used to specify by how much to extend the width of the result.

toPkdArray( arrayDef, options )

Return type: vlogPkdArray. Returns a vlogPkdArray object whose dimensions / structure is defined as per the conventions for array definition mentioned in Support for packed data-structures in TACHL.

toPkdStruct( structDef, options )

Return type: vlogPkdStruct. Returns a vlogPkdStruct object whose dimensions / structure is defined as per the conventions for struct definition mentioned in Support for packed data-structures in TACHL.

declare ( type )

Return type: string. Returns the Verilog declaration of the signal. type parameter is optional and a string that can take only two legal values: "wire", "reg".
