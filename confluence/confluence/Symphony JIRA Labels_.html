How to Label Symphony JIRAS to determine RTL vs MAESTRO issue:

  1. Label = “illegalConfig” – Maestro Bug – Illegal “rtl.json” which includes the case where Maestro to generate say read data interleaving tests, however it do not have the proper json param set to true.
  2. Label = “modifiedConfig” – RTL Bug – The configuration needs to be modified to pass. This is when let’s say a 2x6 Multicast test that has narrows enabled and the test isn't specifically testing narrows. So you turn narrows off in the config and it now passes. This label will be added. (Also a JIRA needs to be attached making note of the narrow bug).


Design team need to handle “illegalConfig” bugs as follows,

  1. Open new MAES-* bug and add Label = "symphony"
  2. Resolve SYM-* bug as follows,
    1. Label = “illegalConfig”
    2. Resolution = “Won’t Fix”
    3. Linked issue = “clones” and fill MAES bug number.
