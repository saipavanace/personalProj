The Ncore dev build (Also called the ARL build) allows you to create and generate Ncore systems that are beyond the per-version parameter limits.  To use it, do the following:

 

Installing & Running the dev build

  • Download a 'dev' release at http://dev.arteris.com/releases/conductor/
  • Untar the file

tar zxf conductor-1.5.0-ir.18-373+dev.tgz1

  • Set FLEXNOC_PATH to the FlexNoC version that goes with this release

setenv FLEXNOC_HOME /engr/dev/tools/FlexNoC/limited/FlexNoC-3.1.1-SP7/

  • Set variables to point to the license server (via a convenient script)

source /engr/dev/tools/script/conductor.csh

  • Run conductor

./conductor &

 

 

Using the dev build

WARNING The dev build lets you go to transport or to generate when there are errors.  There's nothing to say that transport won't crash, that generation won't crash, or that the generated RTL will be full of errors.

The dev build works the same way as any other conductor build, with the following exceptions:

  • To go to the transport perspective when there are validation error, hold Ctrl while clicking on the Transport perspective tab.
  • To generate even when there are validation errors, hold Ctrl while clicking "Generate".
