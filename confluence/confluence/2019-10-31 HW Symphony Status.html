Design Progress

  • Wrote first TCL script to implement ATUs that have a narrower ATP interface than native interface.
    • Found issues in compile and sent to AXI ATU designer to fix.
  • Created a test bench for PMON blocks in an AXI to AXI environment.
    • Has a APB interface use to read and write PMON blocks.
  • Debugged data width adapter issues with seagate serial packets.Â 
  • Fixed issues with ATUs ignoring _SKIP_Â on pma interfaces.
    • Exposed issue where ATUs were not using all new interfaces internally.
    • Started cleaning this issue up.
  • Did some synthesis experiments with logic_tree primitives to see the effect of different coding styles.
  • Did some optimization experiments on the ATUs to reduce size.
    • Initial results look good, but there were some issues with synthesis runs, so they need to be redone.
  • Made decision to only modify multicast target lists using user bits.
    • Makes verification a lot easier, because verification doesn't need to know the structure of the tree.
    • Simplifies information needed from Maestro to setup up test bench.
      • Only needs to provide which mask bit corresponds to which target.
      • When an interrupt is generated because of an error on a buffered response, only need to verify an interrupt was generated, not that it came from the right multicast block.
  • Ran into a test bench issue where splitting has been turned off (by the "customer",) but in socket can produce larger packet than out socket can handle.
    • Test bench needs to be modified to not send large request to targets that can't handle it.
    • Maestro needs to properly program PAM tables so test bench can identify this situation.
    • When ARIA is used to create test bench, the scripting will print out a warning to user explaining they have turned off splitting in a situation that it may be needed and it's up to their logic to prevent the issue from happening or turn splitting back on.
  • When discussing how to setup timing for async bridges led to a discussion of how to handle pressure across an async interface.
    • decision to implement a separate async interface for pressure vs. packets.
