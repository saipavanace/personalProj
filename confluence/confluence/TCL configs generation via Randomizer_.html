1. Make sure you already have working area setup.
    1. “setup_sym”
    2. “source SORUCEME”
  2. “cd $HOME”
  3. “mkdir <some_dir_name>”
  4. “cd some_dir_name”
  5. “$WORK_TOP/tcl_gen/randomizer.tcl 3” // to generate 3 new configs.
  6. “cd testbench”
  7. “$WORK_TOP/scripts/add_tcl_cfg axi_axi” // if you are trying to create axi_axi config.
  8. Rest of the job is done by above (in bullet 7) script.
  9. Now you push your new configs in the hw-sym repo if you are satisfied.

Saad push a shell script (at $WORK_TOP/scripts/add_tcl_cfg) which will help to automate multiple checks as follows,

  1. Check for $WORK_TOP is set or not.
  2. Check of $MAES_BUILD is set or not.
  3. Run “run_maestro” command with defined $MAES_BUILD build number and make sure it passes before moving forward.
  4. Run JOSN file checker ($WORK_TOP/scirpts/chk_json_params.js) make sure it passes before moving forward.
  5. Run Verilator and make it passes before moving forward.
  6. Then move the TCL script in $WORK_TOP/tcl_cfg/maestro_4x4/axi_axi (or axi_apb or apb_apb or apb_axi or nl) folders defined in command line argv of add_tcl_cfg.
