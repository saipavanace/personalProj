SW generated CSR description requirements

Software must include in the block ID in the a description field of the block the CSR is associated with. That description must make it out to the IP-XACT description when the CSR is converted to IP-XACT. This is documented here: OBSOLETE-DELETE HW-SYM Symphony Block Common Characteristics

SW generated CSR description requirements for interrupt blocks

Software must include in a description field a map of register bit to the block ID of the block generating the interrupt that is associated with the bit. That description must make it out to the IP-XACT description when the CSR is converted to IP-XACT. This is documented here: OBSOLETE-DELETE HW-SYM Symphony Block Common Characteristics

PMON parameter requirements

Software must provide the following parameter in the gen_wrapper parameters for the top of a design:

pmonConnectivity[
{atuiID: integer, reqLinkProbeID: integer, respLinkProbeID: integer, statsBlockID: integer},
{},
{}...
]

If reqLinkProbeID is specified as -1, then that specifies that the request Probe associated with the ATUI does not exist. Same rules apply for respLinkProbeID, and statsBlockID.


Interrrupt parameter requirements

Software must provide the following parameter in the gen_wrapper parameters for the top of the design:

Example 1: Generic design with multilevel interrupts.

"interruptMapping": [ 

  { "name" : "EXT_INT_TOP",

    "ID" : integer (block ID),
    "statusRegister" : "INTVR",

    "addrBlockName" : "<IPXACT_name>_BlockID",

    "statusBits" : {

        "0": { "name" : "EXT_INT",

              "statusRegister": "INTVR",

              "ID" : integer (block ID),

              "statusBits" : {
                   "0": { "maskBit": 0,
                             "maskRegister": "INTMR",

               "sources": [ Size: Fixed 32 entries, Array of Block IDs sending interrupt signals to this block, -1 = No block is associated with bit position ] 

                   }

              }

        },
    }

  .

  .

  .

  { "name" : "EXT_INT",

    "ID" : integer (block ID),

    "statusRegister": "INTVR",

    "addrBlockName" : "<IPXACT_name>_BlockID

       "statusBits" : {

                  "0": { "maskBit": 0,
                           "maskRegister": "INTMR",

             "sources": [ Size: Fixed 32 entries, Array of Block IDs sending interrupt signals to this block, -1 = No block is associated with bit position ] 

                          }
       }

  },
}

]

Each "bitMap" will have 32 entries because the size of a register is fixed to be 32 by design team. if a given bit location in register doesn't have block ID associated with it then set it to -1.
