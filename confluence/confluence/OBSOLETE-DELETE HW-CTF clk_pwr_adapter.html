module_name

clk_pwr_adapter

JavaScript parameters

nodeId: A non-negative integer,

maxOutstandingReqs: A positive integer greater than 1. Determines the width of the counter used to keep track of number of pending requests (requests that haven't received responses.

async: A boolean. True or False. Set this to true if clocks in inClkInterface and outClkInterface are asynchronous with respect to each other.

aFifoDepth: A positive integer greater than 1. Must be a power of 2 if async is set to true. Sets the depth of the pipes in the async adapters inside.

jitterOn: A boolean. True or False. Only for simulation. True causes synchronizer cells to jitter one cycle.

respType: A string that can take only one of the following values: "presto", "ncore". Setting to "ncore" strips away some of the logic not required due to the absence of the respSInterface, and adds mapping of the subfields in H_ndp_data in the gen_err_resp tachl component.

hFifoDepth: A positive integer that is used to set the depths of any pipes inside gen_err_resp.

mstrArbLck: A boolean. Master arbiter lock. Default false. True means arbiter locks. False implies arbiter doesn't lock.

lckStyleVld: A boolean. Setting to true makes arbiter lock on valid, and setting to false locks on ready. Default false.

numPri: A positive integer. Number of priorities in packet definition.

arbType: A string that specifies the master arbiter type. Legal values: "arb_rr2", "arb_wrr_rr1", "arb_wrr_rr2", "arb_pri_rr1", "arb_pri_rr2",

arbRdyAware: When true, the master arbiter uses the ready and valid to make a request. When false, it relied just on valid.

weights: An array of positive integers and of length = number of VCs. The elements specify the weights for the master arbiter.

weightsProg: A Boolean value to be set to true if weights are required to be programmable.

protectionStyle: protectionStyle

Interfaces

inClkInterface: A slave interface of type InterfaceCLK for domain 1.

outClkInterface: A slave interface of type InterfaceCLK for domain 2.

reqMInterface: A master interface of type InterfaceATP. This is an ATP interface for the request path.

reqSInterface: A slave interface of type InterfaceATP. This is an ATP interface for the request path.

respMInterface: A master interface of type InterfaceATP. This is an ATP interface for the response path.

respSInterface: A slave interface of type InterfaceATP. This is an ATP interface for the response path.

inPmaControlInterface: A slave interface of type InterfacePMAControl for domain 1.

outPmaControlInterface: A slave interface of type InterfacePMAControl for domain 2.

inProtInterface: (Optional interface). A master interface of type InterfacePROT from domain 1.

outProtInterface: (Optional interface). A master interface of type InterfacePROT from domain 2.
