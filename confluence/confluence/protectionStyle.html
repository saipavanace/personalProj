This parameter is used to communicate the style of protection to be implemented in a block.

Common Parameter Definition:

protectionStyle : null or {
  protection : "ecc", "parity" or "none",
  scrub : true or false,
  useCorrected : true or false,
  protWidth : any positive integer // Maximum width parity or ecc calculated across.
}

Discussion

This is a common object that many library elements use.

If protectionStyle is defined, there will be an InterfacePROT associated with the register or memory taking in this parameter.

Based on the protectionStyle, the InterfacePROT must have the following signals:

There are two sets of signals that the interface can be either or (exclusive or): err_par, err_sb, err_db, syndrome and index XOR uncorr and corr. 

Software should only see the uncorr and corr set of signals.

A single InterfacePROT can carry errors from multiple sources and so can have all the signals of a set included. For Instance, if a block has both ecc and parity protection on it's state, it can have err_par, err_sb, err_db, syndrome and index on the InterfacePROT coming out of the block.
