Module Name:

pma_mux

Features:

  • Combines multiple PMA control interfaces into one. Allows a pma_slave to control multiple blocks.
  • Enables pipe-lining of PMA Control interfaces.

Parameters:

Javascript:

{
  "interfaces" : {
                    "outInterface" : {InterfacePMAControl},
                    "inInterfaces" : [{InterfacePMAControl},,,], // array of PMA control interfaces

                                  }

}

I/O:

u.nInterfaces(interfaces);

Modules Used:

logic_tree

Description:

The PMA Control interface contains four signals: goto_sleep, asleep, busy, and abort. The outInterface has upstream connections which will contain the logical OR of all busy signals and  the logical OR of all abort signals of the inInterfaces. In addition the sym_pma_mux will propagate the outInterfaces's goto_sleep and asleep signals to all inInterfaces.
