AN-251

Requested from MobilEye and answered that we would not implement, but in recent onsite they seem to still be expecting this to be delivered, even hinting that it was part of the standalone SOW

Latest request from MobilEye is just to fully implement user bits in the models so they can use it to pass QoS info through the network to downstream logic.

The problem is that this involves changing all ncore units, most do not pass user bits. Also may be affected by flexnoc issue FN-1762

Not something that will be in 7/12 release.

AN-265

No graceful way to terminate, proposed a print statement.

If we put an assertion in to stop simulation, it would be optimized out by Platform Architect, so would require terminate statement.

Not something that will be in 7/12 release.

AN-280

If allocate requested but all ways busy, then RTL demotes to non-allocate. Model does not match RTL.

Fixing the model is complicated. Currently model assumes allocate always, so all read returns will be written to the cache. Changing this would require careful searching code for all return paths affected by the read. Concerns about missing something due to assumptions in code.

It is expected that this condition should happen rarely and therefore behavior difference should not affect performance, except for pathological targeted test case where software is continually hitting same row.

Might consider a print statement when this occurs, or counter of number of times it occurs in a sim.

Not something that will be in 7/12 release.

Finalize what is in 7/12 release

RTL 7/12 release will contain all features in final release, though there will be verification pending.

Model 7/12 release will not contain any of the above 3, will just add Multiline burst. We may be able to supply a drop earlier, if it is finished earlier.
