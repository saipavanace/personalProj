Release Notes

  • CONC-2914, CONC-2915

When upgrading a design from Ncore 1.6.x version to Ncore 2.x version, some automatic changes in the connectivity table of the CCTI specification will result in UEXE messages in the CCTI architecture topology due to the removal of routes associated with the previous connectivity.
After reviewing those messages and potentially updating the topology, you can remove the remaining UEXE messages related to route changes in the architecture topology.

  • 2.0 Upgrade

The wId of AXI4 DMI interfaces can change when upgrading a project from 1.6 to 2.0 release.
