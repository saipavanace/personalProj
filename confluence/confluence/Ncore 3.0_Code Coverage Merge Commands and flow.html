1. This page is intended to give a way to perform code coverage merge. It doesn't mean that this is the only way or the best way. There are many other ways to perform code coverage merge. Please see the attached Questa Verification Management power point for details.
  2. The merge in general is performed by the command below:
    1. vcover merge merged.ucdb input0.ucdb input1.ucdb ....
  3. For merge over many instances, ie. OTT entries, into one single unit:
    1. vcover merge -du block_name -recursive total_block.ucdb block.ucdb full_sys.ucdb
    2. The example above merges all the instances in full sys and block level into one single unit, namely, total_block.ucdb
    3. Now you need to merge back total_block.ucdb into block level
      1. vcover merge -install /tb_top/block my_total_block.ucdb total_block.ucdb (added the block level test bench hierarchical path)
      2. vcover merge merged.ucdb existing_block.ucdb my_total_block.ucdb
  4. For merge from block to full sys:
    1. vcover merge -strip 1 block.ucdb merged_block.ucdb
    2. vcover merge -install /tb_top/full_sys f_block.ucdb block.ucdb
    3. vcover merge full_block_merge.ucdb f_block.ucdb full_sys.ucdb

If there is no license available, the above script waits for one to be available, and then it runs covercheck "qverify command" for the "timeout" set for each configuration under the <arg2> directory. Currently timeout is set for 30 minutes per configuration. 

When the above script finishes, there will be in the <arg1> dir

→ exclusion file created by covercheck tool: merged_dce_hw_cfg_3_exclude.do

→ new ucdb with the above exclusion file applied: dce_hw_cfg_3_merged.ucdb

→ dir with all the logs for the qverify command: dce_hw_cfg_3_coverage


dce_hw_cfg_3_merged.ucdb is provided to the designer to add additional exclusions.
