Maestro Term

TCL

Meaning

(Packet)Route

packet_route

A path that traffic can take between a source Port and a destination Port. The path is a sequence of physical resources that packets will traverse to reach the destination, e.g. Switch Ports. Some traversed resources, like LinkSegments and Adapters, may be implicitly (but unambiguously) defined by the explicitly defined resources.

Not all pair of Ports need to have a (Packet)Route between them.

In the current implementation, paths between a given source Port and a given destination Port are unique.

LinkSegment

link_segment

A direct connection between module Ports, of any width, comprising only signals but no intermediate modules of any type.

Link

link

An overall connection between module Ports, comprising one or more LinkSegments and zero or more Adapters in a given sequence.

A Link has exactly one source and one destination, with no forking.

Port

(not widely exposed yet)

An interface of a module, having a width of one or more signals, following a certain protocol, and performing a specific function.

Flow

flow

A traffic specification provided by the user, having two endpoints that are user-visible (two Sockets), and possibly annotated with properties (e.g. read or write traffic, burstiness) and QoS requirements (e.g. bandwidth, latency, security…)

Socket

socket

A type of Port that follows a complex protocol with bidirectional communication, typically an industry standard (e.g. the various ARM AMBA flavours, OCP, etc.).

Adapter

clock_adapter, width_adapter, pipe_adapter, …

A module with an input Port and an output Port, that lets incoming traffic through while performing a function (clock adaptation, width adjustment, buffering, retiming…). It is instantiated between LinkSegments as part of a Link.

Some Adapters may be splittable in two front and back halves, e.g. ClockAdapters may comprise two submodules, one per clock domain.

Switch

switch

A module with one or more input Ports and one or more output Ports, that can transfer traffic coming into an input Port onto a specified output Port. The connectivity of input to output Ports may or may not be complete depending on needs. The Switch performs arbitration in case of conflicts and may provide buffering.
