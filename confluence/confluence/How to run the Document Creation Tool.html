Go somewhere, anywhere and run the script:

$WORK_TOP/../../hw-lib/js/createParmDoc.js -w $WORK_TOP/../.. -o outDoc.txt

And the document has been created in outputted to outDoc.txt.

Before you push your changes you need verify that you blocks is not breaking two other scripts. Here are instructions on how to run the tools:

Parameter Checking Tool:

TACHL to CPR checking tool

Parameter Coverage Tool:

Parameter Coverage Tool Instructions

With the parameter coverage tool, you need to find a config that is actually using the block you are modifying. The two examples in the instructions are the seagate design and the enflame design. If the block you are modifying isn't in one of those two areas, you need to split a json from a config that does use your block. The split command is:

$WORK_TOP/../../hw-lib/js/create_hierarchy.js -i $WORK_TOP/gen_cfg/<some_config_that_uses_your_block>/tb.json -o `pwd`/split_json_yours

NOTE: You only need to run this command once. You may need more than one config for the blocks you are modifying.

Then the command to run updateParmCvg.js is:

$WORK_TOP/../../hw-lib/js/updateParmCvg.js -w $WORK_TOP/../.. -d ./data_base -j ./split_json_yours

NOTE: When you modify the CPRs with respect to coverage, you need to delete the previous data base files. Using the above examples you need to run the command:

rm -rf ./data_base/*

Before running the updateParmCvg.js command.
