Goals

  • To understand the requirements in Perfmon and other logic to support histogram calculations

Discussion items

TimeItemWhoNotes
60 min

  • The histogram data is expected to be computed on-the-fly for Functional usage and not so much or just for performance monitoring or debug. The goal is to provide the software the ability to control QoS dynamically.
  • The histogram data will include calculations such as - histogram of latencies of read and write requests, back-pressure frequencies, average and maybe individual latency calculations.
  • The accuracy of the latency measurements needs to be cycle-by-cycle accuracy and with a counter with upto 11-bits. About 8 bins maybe sufficient.
  • These features would also enable software monitoring of network/mesh health. However, it was discussed that implementing something like adaptive routing doesn't make sense typically for on-chip networks because the latencies involved are way less than the time taken by software to respond and reconfigure routing. Right now source-routing should suffice.
  • We should be supporting in APB/config network, the ability to read all the counter bins by pausing them before start of read and resuming only after the last bin is read. Burst reads would be a good idea to support on the config network not only to speed up reading all the bins but to speed up boot or wakeup re-configuration.
  • Also need to support capability to reset all bin counters in the same clock with a single event.
  • Latency measurement implies a sort of bidirectional snooper at the ATU, that can watch requests and responses on the request-response pair of links.
  • In addition to monitoring network health, some of Perfmon or Debug logic could also be used to support safety mechanisms in functional mode - such as validating that a sequence of events occurred. This would require cross-triggering capability and a programmable state-machine.
  • Resiliency - No resiliency needed on observable network (Perfmon or Debug).
  • All these features are not expected to be in R1. But could be targeted for post-R1.
